# <a name="building-on-the-framework"></a>프레임워크를 기반으로 구축

MFC 프레임 워크를 사용 하 여 응용 프로그램을 구성할 때의 역할은 응용 프로그램 관련 소스 코드를 제공 하 고 응답 하는 메시지와 명령을 정의 하 여 구성 요소를 연결 하는 것입니다. C + + 언어 및 표준 c + + 기술을 사용 하 여 클래스 라이브러리에서 제공 하는 응용 프로그램 관련 클래스를 파생 시키고 기본 클래스의 동작을 재정의 하 고 보강 합니다.

관련 항목에서 다음 표에서는 일반적으로 수행 하는 작업의 일반적인 순서와 프레임 워크의 책임에 대해 설명 합니다.

- [프레임 워크를 사용 하 여 응용 프로그램을 빌드하기 위한 시퀀스](sequence-of-operations-for-building-mfc-applications.md)

- [OLE 응용 프로그램을 만들기 위한 작업 시퀀스](sequence-of-operations-for-creating-ole-applications.md)

- [ActiveX 컨트롤을 만드는 작업 시퀀스](sequence-of-operations-for-creating-activex-controls.md)

- [데이터베이스 응용 프로그램을 만들기 위한 작업 시퀀스](sequence-of-operations-for-creating-database-applications.md)

대부분의 경우에는 이러한 테이블을 MFC 응용 프로그램을 만들기 위한 일련의 단계로 따라갈 수 있습니다. 단, 일부 단계는 다른 옵션입니다. 예를 들어 대부분의 응용 프로그램은 사용할 수 있는 여러 형식에서 하나의 뷰 클래스 형식을 사용 합니다.
