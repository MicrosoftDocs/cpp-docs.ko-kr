## <a name="remarks"></a>설명

`CDockablePane`는 다음과 같은 기능을 구현 합니다.

- 주 프레임 창에 창을 도킹 합니다.

- 창을 자동 숨기기 모드로 전환 합니다.

- 탭 창에 창을 연결 합니다.

- 미니 프레임 창에서 창을 부동 합니다.

- 미니 프레임 창에서 부동 창의 다른 창에 창을 도킹 합니다.

- 창의 크기를 조정 합니다.

- 도킹 창의 상태를 로드 하 고 저장 합니다.

    > [!NOTE]
    >  상태 정보는 Windows 레지스트리에 저장 됩니다.

- 캡션을 사용 하거나 사용 하지 않고 창을 만듭니다. 캡션에는 텍스트 레이블이 있을 수 있으며 그라데이션 색으로 채워질 수 있습니다.

응용 프로그램에서 도킹 창을 사용 하려면 클래스에서 창 클래스를 파생 시킵니다 `CDockablePane` . 파생 된 개체를 주 프레임 창 개체나 창의 인스턴스를 제어 하는 window 개체에 포함 합니다. 그런 다음 주 프레임 창에서 WM_CREATE 메시지를 처리할 때 [CDockablePane:: Create](#create) 메서드 또는 [CDockablePane:: createex](#createex) 메서드를 호출 합니다. 마지막으로 [Cbasepane:: EnableDocking](../../mfc/reference/cbasepane-class.md#enabledocking), [cbasepane::D ockpane](../../mfc/reference/cbasepane-class.md#dockpane)또는 [CDockablePane:: AttachToTabWnd](#attachtotabwnd)를 호출 하 여 pane 개체를 설정 합니다.

## <a name="cdockablepaneattachtotabwnd"></a><a name="attachtotabwnd"></a>CDockablePane:: AttachToTabWnd

현재 창을 대상 창에 연결 하 여 탭 창을 만듭니다.

```
virtual CDockablePane* AttachToTabWnd(
    CDockablePane* pTabControlBarAttachTo,
    AFX_DOCK_METHOD dockMethod,
    BOOL bSetActive= TRUE,
    CDockablePane** ppTabbedControlBar = NULL);
```

### <a name="parameters"></a>매개 변수

*pTabControlBarAttachTo*<br/>
[in, out] 현재 창이 연결 되는 대상 창을 지정 합니다. 대상 창은 도킹 가능한 창 이어야 합니다.

*dockMethod*<br/>
진행 도킹 방법을 지정 합니다.

*bSetActive*<br/>
진행 연결 작업 후 탭 창을 활성화 하려면 TRUE로 설정 합니다. 그렇지 않으면 FALSE입니다.

*ppTabbedControlBar*<br/>
제한이 연결 작업에서 발생 하는 탭 창을 포함 합니다.

### <a name="return-value"></a>Return Value

탭 창이 아닌 경우 현재 창에 대 한 포인터입니다. 그렇지 않으면 연결 작업에서 발생 하는 탭 창에 대 한 포인터입니다. 현재 창을 연결할 수 없거나 오류가 발생 한 경우 반환 값은 NULL입니다.

### <a name="remarks"></a>설명

이 메서드를 사용 하 여 도킹 가능한 창 하나를 다른 창에 연결 하면 다음 작업이 수행 됩니다.

1. 프레임 워크는 대상 창이 일반 도킹 창 인지 또는 [CBaseTabbedPane](../../mfc/reference/cbasetabbedpane-class.md)에서 파생 된 *pTabControlBarAttachTo* 를 확인 합니다.

1. 대상 창이 탭 창이 면 프레임 워크에서 탭으로 현재 창을 추가 합니다.

1. 대상 창이 일반 도킹 창이 면 프레임 워크에서 탭 창을 만듭니다.

   - 프레임 워크는를 호출 `pTabControlBarAttachTo->CreateTabbedPane` 합니다. 새 탭 창의 스타일은 멤버에 따라 다릅니다 `m_pTabbedControlBarRTC` . 기본적으로이 멤버는 [CTabbedPane](../../mfc/reference/ctabbedpane-class.md)의 runtime 클래스로 설정 됩니다. AFX_CBRS_OUTLOOK_TABS 스타일을 [CDockablePane:: Create](#create) 메서드에 *dwTabbedStyle* 매개 변수로 전달 하는 경우 Runtime 클래스 개체는 [CMFCOutlookBar](../../mfc/reference/cmfcoutlookbar-class.md)의 runtime 클래스로 설정 됩니다. 언제 든 지이 멤버를 변경 하 여 새 창의 스타일을 변경할 수 있습니다.

   - 이 메서드가 탭 창을 만들 때 프레임 워크는 새 탭 창에 대 한 포인터를 사용 하 여 포인터를 *pTabControlBarAttachTo* 로 대체 합니다. 즉, 창이 다중 미니 프레임 창에서 도킹 되거나 부동 됩니다.

   - 프레임 워크는 탭 창에 첫 번째 탭으로 *pTabControlBarAttachTo* 창을 추가 합니다. 그런 다음 프레임 워크는 현재 창을 두 번째 탭으로 추가 합니다.

1. 현재 창이에서 파생 되는 경우 `CBaseTabbedPane` 모든 탭이 *pTabControlBarAttachTo* 로 이동 하 고 현재 창이 소멸 됩니다. 따라서 메서드가 반환 될 때 현재 창에 대 한 포인터가 유효 하지 않을 수 있으므로이 메서드를 호출할 때는 주의 해야 합니다.

도킹 레이아웃을 빌드할 때 한 창을 다른 창에 연결 하는 경우 `dockMethod` DM_SHOW으로 설정 합니다.

다른 창을 연결 하기 전에 첫 번째 창을 도킹 해야 합니다.

진행 사각형의 가장자리에서 떨어진 거리 (픽셀)입니다. 도킹을 사용 하려면 포인터가 있어야 합니다.

### <a name="return-value"></a>Return Value

다음 상태 값 중 하나입니다.

|AFX_CS_STATUS 값|의미|
|---------------------------|-------------|
|CS_NOTHING|포인터가 도크 사이트를 벗어났습니다. 프레임 워크는 창을 도킹 하지 않습니다.|
|CS_DOCK_IMMEDIATELY|포인터는 고정 사이트에서 직접 실행 모드로 배치 됩니다. 창에는 DT_IMMEDIATE 도킹 모드가 사용 됩니다. 프레임 워크는 창을 즉시 도킹 합니다.|
|CS_DELAY_DOCK|포인터는 다른 도킹 창이 나 주 프레임의 가장자리에 해당 하는 도크 사이트 위에 있습니다. 프레임 워크는 지연 된 후 창을 도킹 합니다. 이 지연에 대 한 자세한 내용은 설명 부분을 참조 하세요.|
|CS_DELAY_DOCK_TO_TAB|포인터가 탭 창에 도킹 되도록 하는 도크 사이트 위에 있습니다. 이는 포인터가 다른 도킹 창의 캡션 위에 있거나 탭 창의 탭 영역에 있을 때 발생 합니다.|

### <a name="remarks"></a>설명

프레임 워크는 부동 창의 도킹을 처리 하기 위해이 메서드를 호출 합니다.

DT_IMMEDIATE 도킹 모드를 사용 하는 부동 도구 모음이 나 도킹 창의 경우 프레임 워크는 도킹 명령을 지연 하 여 도킹이 발생 하기 전에 사용자가 부모 프레임의 클라이언트 영역에서 창을 이동할 수 있도록 합니다. 지연 시간은 밀리초 단위로 측정 되며 [CDockingManager:: m_nTimeOutBeforeToolBarDock](../../mfc/reference/cdockingmanager-class.md#m_ntimeoutbeforetoolbardock) 데이터 멤버에 의해 제어 됩니다. [CDockingManager:: m_nTimeOutBeforeToolBarDock](../../mfc/reference/cdockingmanager-class.md#m_ntimeoutbeforetoolbardock) 의 기본값은 200입니다. 이 동작은 Microsoft Word 2007의 도킹 동작을 에뮬레이트합니다.

지연 된 도킹 상태 (CS_DELAY_DOCK 및 CS_DELAY_DOCK_TO_TAB)의 경우 프레임 워크는 사용자가 마우스 단추를 놓을 때까지 도킹을 수행 하지 않습니다. 창에서 DT_STANDARD 도킹 모드를 사용 하는 경우 프레임 워크는 예상 도킹 위치에 사각형을 표시 합니다. 창에서 DT_SMART 도킹 모드를 사용 하는 경우 프레임 워크는 프로젝션 된 도킹 위치에 스마트 도킹 표식과 반투명 사각형을 표시 합니다. 창에 대 한 도킹 모드를 지정 하려면 [Cbasepane:: SetDockingMode](../../mfc/reference/cbasepane-class.md#setdockingmode) 메서드를 호출 합니다. 스마트 도킹에 대 한 자세한 내용은 [CDockingManager:: GetSmartDockingParams](../../mfc/reference/cdockingmanager-class.md#getsmartdockingparams)를 참조 하세요.

## <a name="cdockablepaneundockpane"></a><a name="undockpane"></a>CDockablePane:: UndockPane

주 프레임 창 또는 미니 프레임 창 컨테이너에서 창을 도킹 해제 합니다.

```

```

### <a name="parameters"></a>매개 변수

*bDelay*<br/>
진행 도킹 레이아웃 계산을 지연 시키려면 TRUE입니다. 도킹 레이아웃을 즉시 다시 계산 하려면 FALSE로 설정 합니다.

### <a name="remarks"></a>설명

주 프레임 창이 나 다중 미니 프레임 창 컨테이너 (다른 창과 단일 미니 프레임 창에서 부동 창)에서 창을 도킹 해제 하려면이 메서드를 호출 합니다.

[CDockingManager](../../mfc/reference/cdockingmanager-class.md)에서 수행 되지 않는 외부 작업을 수행 하기 전에 창의 도킹을 해제 해야 합니다. 예를 들어 한 위치에서 다른 위치로 프로그래밍 방식으로 이동 하려면 창의 도킹을 해제 해야 합니다.

프레임 워크는 소멸 되기 전에 창을 자동으로 도킹 해제 합니다.
