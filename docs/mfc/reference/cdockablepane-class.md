|[CDockablePane::m_bHideInAutoHideMode](#m_bhideinautohidemode)|창이 자동 숨기기 모드에 있을 때 창의 동작을 결정합니다.|

## <a name="remarks"></a>설명

`CDockablePane`다음 기능을 구현합니다.

- 창을 주 프레임 창에 도킹합니다.

- 창을 자동 숨기기 모드로 전환합니다.

- 탭된 창에 창을 연결합니다.

- 미니프레임 창에 창을 부동시.

- 미니프레임 창에 떠 있는 다른 창에 창을 도킹합니다.

- 창 크기 조정.

- 도킹 창에 대한 상태 로드 및 저장

    > [!NOTE]
    >  상태 정보는 Windows 레지스트리에 저장됩니다.

- 캡션이 있거나 없는 창 만들기 캡션에는 텍스트 레이블이 있을 수 있으며 그라데이션 색상으로 채울 수 있습니다.

- 창의 내용을 표시하는 동안 창 끌기

- 드래그 사각형을 표시하는 동안 창을 드래그합니다.

응용 프로그램에서 도킹 창을 사용하려면 클래스에서 창 클래스를 `CDockablePane` 파생합니다. 파생 된 개체를 주 프레임 창 개체 또는 창의 인스턴스를 제어하는 창 개체에 포함합니다. 그런 다음 메인 프레임 창에서 WM_CREATE 메시지를 처리할 때 [CDockablePane::Create](#create) 메서드 또는 [CDockablePane:CreateEx](#createex) 메서드를 호출합니다. 마지막으로 [CBasePane::EnableDocking,](../../mfc/reference/cbasepane-class.md#enabledocking) [CBasePane::DockPane](../../mfc/reference/cbasepane-class.md#dockpane)또는 [CDockablePane::AttachToTabWnd를](#attachtotabwnd)호출하여 창 개체를 설정합니다.

## <a name="customization-tips"></a>사용자 지정 팁

다음 팁은 `CDockablePane` 개체에 적용됩니다.

- [CDockablePane::AttachToTabWnd](#attachtotabwnd) 두 개의 탭되지 않은 도킹 가능한 창에 대해 탭된 창에 대한 포인터가 *ppTabbedControlBar* 매개 변수에 반환됩니다. 이 매개 변수를 사용하여 탭창에 탭을 계속 추가할 수 있습니다.

- [CDockablePane::AttachToTabWnd에](#attachtotabwnd) 의해 생성 되는 탭 된 창의 `CDockablePane` *종류는 pTabControlBarAttachTo* 매개 변수의 개체에 의해 결정 됩니다. [CDockablePane::SetTabbedPaneRTC를](#settabbedpanertc) 호출하여 탭된 창의 종류를 `CDockablePane` 설정하여 만들 수 있습니다. 기본 형식은 `dwTabbedStyle` [CDockablePane::Create를](#create) 처음 만들 때 `CDockablePane`에 의해 결정됩니다. *dwTabbedStyle이* AFX_CBRS_OUTLOOK_TABS 기본 형식은 [CMFCOutlookBar 클래스입니다.](../../mfc/reference/cmfcoutlookbar-class.md) *dwTabbedStyleAFX_CBRS_REGULAR_TABS* 기본 형식은 [CTabbedPane 클래스입니다.](../../mfc/reference/ctabbedpane-class.md)

- 도킹 가능한 창을 다른 창에 도킹하려면 [CDockablePane::DockToWindow](#docktowindow) 메서드를 호출합니다. 이 메서드를 호출하기 전에 원래 창을 어딘가에 도킹해야 합니다.

- 멤버 변수 [CDockablePane::m_bHideInAutoHideMode](#m_bhideinautohidemode) [CDockablePane::ShowPane를](#showpane)호출할 때 도킹 가능한 창이 자동 숨기기 모드에서 어떻게 실행되는지 제어합니다. 이 멤버 변수가 TRUE로 설정된 경우 도킹 가능한 창과 자동 숨기기 버튼이 숨겨집니다. 그렇지 않으면, 그들은 안팎으로 미끄러집니다.

- [CDockablePane::m_bDisableAnimation](#m_bdisableanimation) 멤버 변수를 TRUE로 설정하여 자동 숨기기 애니메이션을 비활성화할 수 있습니다.

## <a name="cdockablepaneattachtotabwnd"></a><a name="attachtotabwnd"></a>CDockablePane::첨부토탭

현재 창을 대상 창에 연결하여 탭된 창을 만듭니다.

```
virtual CDockablePane* AttachToTabWnd(
    CDockablePane* pTabControlBarAttachTo,
    AFX_DOCK_METHOD dockMethod,
    BOOL bSetActive= TRUE,
    CDockablePane** ppTabbedControlBar = NULL);
```

### <a name="parameters"></a>매개 변수

*pTab컨트롤바첨부토*<br/>
【인, 아웃】 현재 창이 첨부하는 대상 창을 지정합니다. 대상 창은 도킹 가능한 창이어야 합니다.

*dockMethod*<br/>
【인】 도킹 방법을 지정합니다.

*bSetActive*<br/>
【인】 TRUE 는 부착 작업 후 탭 창을 활성화하는 데; 그렇지 않으면 false입니다.

*ppTabbed컨트롤바*<br/>
【아웃】 연결 작업의 결과로 발생하는 탭창이 포함되어 있습니다.

### <a name="return-value"></a>Return Value

탭된 창이 아닌 경우 현재 창에 대한 포인터입니다. 그렇지 않으면 연결 작업에서 발생하는 탭창에 대한 포인터입니다. 현재 창을 연결할 수 없거나 오류가 발생하는 경우 반환 값은 NULL입니다.

### <a name="remarks"></a>설명

이 메서드를 사용하여 도킹 가능한 창 하나가 다른 창에 연결되면 다음과 같은 현상이 발생합니다.

1. 프레임워크는 대상 창 *pTabControlBarAttachTo가* 일반 도킹 창인지 아니면 [CBaseTabbedPane](../../mfc/reference/cbasetabbedpane-class.md)에서 파생되는지 여부를 확인합니다.

1. 대상 창이 탭된 창인 경우 프레임워크는 현재 창을 탭으로 추가합니다.

1. 대상 창이 일반 도킹 창인 경우 프레임워크는 탭된 창을 만듭니다.

   - 프레임워크는 `pTabControlBarAttachTo->CreateTabbedPane`을 호출합니다. 새 탭창의 스타일은 멤버에 `m_pTabbedControlBarRTC` 따라 다릅니다. 기본적으로 이 멤버는 [CTabbedPane](../../mfc/reference/ctabbedpane-class.md)의 런타임 클래스로 설정됩니다. AFX_CBRS_OUTLOOK_TABS 스타일을 *dwTabbedStyle* 매개 변수로 [CDockablePane::Create](#create) 메서드에 전달하는 경우 런타임 클래스 개체는 [CMFCOutlookBar의](../../mfc/reference/cmfcoutlookbar-class.md)런타임 클래스로 설정됩니다. 언제든지 이 멤버를 변경하여 새 창의 스타일을 변경할 수 있습니다.

   - 이 메서드가 탭된 창을 만들 때 프레임 워크는 *pTabControlBarAttachTo* (창이 도킹 되거나 다중 미니프레임 창에 부동 하는 경우)에 대 한 포인터를 새 탭 된 창에 대 한 포인터로 대체 합니다.

   - 프레임워크는 탭된 창에 *pTabControlBarAttachTo* 창을 첫 번째 탭으로 추가합니다. 그런 다음 프레임워크는 현재 창을 두 번째 탭으로 추가합니다.

1. 현재 창에서 `CBaseTabbedPane`파생 된 경우 모든 *탭이 pTabControlBarAttachTo로* 이동 하 고 현재 창이 소멸 됩니다. 따라서 메서드가 반환될 때 현재 창에 대한 포인터가 유효하지 않을 수 있으므로 이 메서드를 호출할 때주의해야 합니다.

도킹 레이아웃을 작성할 때 한 창을 다른 `dockMethod` 창에 연결하는 경우 DM_SHOW 설정합니다.

다른 창을 연결하기 전에 첫 번째 창을 도킹해야 합니다.

## <a name="cdockablepanegetdockingstatus"></a><a name="getdockingstatus"></a>CDockablePane::GetDocking상태

```
    int nSensitivity);
```

### <a name="parameters"></a>매개 변수

*pt*<br/>
【인】 화면 좌표에서 포인터의 위치입니다.

*n감도*<br/>
【인】 포인터가 도킹을 사용하도록 설정하려면 사각형 가장자리에서 멀리 떨어진 거리(픽셀 단위)입니다.

### <a name="return-value"></a>Return Value

다음 상태 값 중 하나입니다.

|AFX_CS_STATUS 값|의미|
|---------------------------|-------------|
|CS_NOTHING|포인터가 도크 사이트 위에 있지 않습니다. 프레임워크가 창을 도킹하지 않습니다.|
|CS_DOCK_IMMEDIATELY|포인터는 즉시 모드에서 도크 사이트 위에 DT_IMMEDIATE 있습니다. 프레임워크는 창을 즉시 도킹합니다.|
|CS_DELAY_DOCK|포인터가 다른 도킹 창이거나 주 프레임의 가장자리인 도크 사이트 위에 있습니다. 프레임워크는 지연 후 창을 도킹합니다. 이 지연에 대한 자세한 내용은 비고 섹션을 참조하십시오.|
|CS_DELAY_DOCK_TO_TAB|포인터는 탭된 창에 창을 도킹하도록 하는 도크 사이트 위에 있습니다. 포인터가 다른 도킹 창의 캡션 위에 있거나 탭된 창의 탭 영역 위에 있을 때 발생합니다.|

### <a name="remarks"></a>설명

프레임워크는 부동 창의 도킹을 처리하기 위해 이 메서드를 호출합니다.

DT_IMMEDIATE 도킹 모드를 사용하는 부동 도구 모음 또는 도킹 창의 경우 프레임워크는 도킹이 발생하기 전에 사용자가 부모 프레임의 클라이언트 영역 밖으로 창을 이동할 수 있도록 도크 명령을 지연시갑니다. 지연 시간은 밀리초 단위로 측정되며 [CDockingManager::m_nTimeOutBeforeToolBarDock](../../mfc/reference/cdockingmanager-class.md#m_ntimeoutbeforetoolbardock) 데이터 멤버에 의해 제어됩니다. [CDockingManager::m_nTimeOutBeforeToolBarDock](../../mfc/reference/cdockingmanager-class.md#m_ntimeoutbeforetoolbardock) 기본값은 200입니다. 이 동작은 Microsoft Word 2007의 도킹 동작을 에뮬레이트합니다.

지연된 도킹 상태(CS_DELAY_DOCK 및 CS_DELAY_DOCK_TO_TAB)의 경우 사용자가 마우스 단추를 해제할 때까지 프레임워크가 도킹을 수행하지 않습니다. 창이 DT_STANDARD 도킹 모드를 사용하는 경우 프레임워크는 투영된 도킹 위치에 사각형을 표시합니다. 창이 DT_SMART 도킹 모드를 사용하는 경우 프레임워크는 투영된 도킹 위치에 스마트 도킹 마커와 반투명 사각형을 표시합니다. 창에 대한 도킹 모드를 지정하려면 [CBasePane::SetDockingMode 메서드를](../../mfc/reference/cbasepane-class.md#setdockingmode) 호출합니다. 스마트 도킹에 대한 자세한 내용은 [CDockingManager::GetSmartDockingParams](../../mfc/reference/cdockingmanager-class.md#getsmartdockingparams)을 참조하십시오.

주 프레임 창 또는 미니프레임 창 컨테이너에서 창을 도킹 해제합니다.

```
virtual void UndockPane(BOOL bDelay = FALSE);
```

### <a name="parameters"></a>매개 변수

*bDelay*<br/>
【인】 도킹 레이아웃 계산을 지연하는 TRUE; FALSE를 사용하여 도킹 레이아웃을 즉시 다시 계산합니다.

### <a name="remarks"></a>설명

이 메서드를 호출하여 주 프레임 창 또는 다중 미니프레임 창 컨테이너(다른 창이 있는 단일 미니프레임 창에 떠 있는 창)에서 창을 도킹 해제합니다.

[CDockingManager](../../mfc/reference/cdockingmanager-class.md)에서 수행하지 않는 외부 작업을 수행하기 전에 창의 도킹을 해제해야 합니다. 예를 들어 창을 프로그래밍 방식으로 이동하려면 창을 한 위치에서 다른 위치로 이동해제해야 합니다.

프레임워크는 파기되기 전에 창의 도킹을 자동으로 해제합니다.
