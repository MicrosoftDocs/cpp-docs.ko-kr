# <a name="clistctrl-class"></a>CListCtrl 클래스

각각 이미지 목록의 아이콘과 레이블로 구성되는 항목 컬렉션을 표시하는 "목록 뷰 컨트롤"의 기능을 캡슐화합니다.

## <a name="syntax"></a>구문

```
class CListCtrl : public CWnd
```

## <a name="remarks"></a>설명

아이콘과 레이블 외에도 각 항목에는 아이콘과 레이블 의 오른쪽에 있는 열에 정보가 표시될 수 있습니다. 이 컨트롤(및 `CListCtrl` 따라서 클래스)은 Windows 95/98 및 Windows NT 버전 3.51 이상에서 실행되는 프로그램에서만 사용할 수 있습니다.

다음은 클래스에 대한 간략한 `CListCtrl` 개요입니다. 자세한 개념적 설명은 [CListCtrl](../../mfc/using-clistctrl.md) 및 [컨트롤](../../mfc/controls-mfc.md)사용을 참조하십시오.

## <a name="views"></a>보기

목록 보기 컨트롤은 "보기"라고 하는 네 가지 방법으로 내용을 표시할 수 있습니다.

- 아이콘 보기

   각 항목은 전체 크기의 아이콘(32 x 32픽셀)으로 표시되며 그 아래에 레이블이 표시됩니다. 사용자는 목록 보기 창의 모든 위치로 항목을 끌 수 있습니다.

- 작은 아이콘 보기

   각 항목은 오른쪽에 레이블이 있는 작은 아이콘(16 x 16픽셀)으로 표시됩니다. 사용자는 목록 보기 창의 모든 위치로 항목을 끌 수 있습니다.

- 목록 보기

   각 항목은 오른쪽에 레이블이 있는 작은 아이콘으로 나타납니다. 항목은 열로 정렬되며 목록 보기 창의 어떤 위치로도 끌 수 없습니다.

- 보고서 뷰

   각 항목은 오른쪽 의 열에 정렬 된 추가 정보와 함께 자신의 줄에 나타납니다. 맨 왼쪽 열에는 작은 아이콘과 레이블이 포함되며 후속 열에는 응용 프로그램에서 지정한 하위 항목이 포함됩니다. 포함된 헤더 컨트롤(클래스 [CHeaderCtrl)은](../../mfc/reference/cheaderctrl-class.md)이러한 열을 구현합니다. 보고서 보기의 헤더 컨트롤 및 열에 대한 자세한 내용은 [CListCtrl 사용: 컨트롤에 열 추가(보고서 보기)를](../../mfc/adding-columns-to-the-control-report-view.md)참조하십시오.

컨트롤의 현재 목록 뷰 스타일에 따라 현재 뷰가 결정됩니다. 이러한 스타일과 사용법에 대한 자세한 내용은 [CListCtrl 사용: 목록 제어 스타일 변경](../../mfc/changing-list-control-styles.md)을 참조하십시오.

## <a name="extended-styles"></a>확장 스타일

클래스는 `CListCtrl` 표준 목록 스타일 외에도 확장된 스타일의 큰 집합을 지원하여 풍부한 기능을 제공합니다. 이 기능의 몇 가지 예는 다음과 같습니다.

- 가리키기 선택

   활성화된 경우 커서가 일정 기간 동안 항목 위에 남아 있을 때 항목을 자동으로 선택할 수 있습니다.

- 가상 목록 보기

   활성화된 경우 컨트롤이 최대 DWORD 항목을 지원할 수 있습니다. 이는 응용 프로그램에 항목 데이터 관리의 오버헤드를 배치하여 가능합니다. 항목 선택 및 포커스 정보를 제외하고 모든 항목 정보는 응용 프로그램에서 관리해야 합니다. 자세한 내용은 [CListCtrl 사용: 가상 목록 컨트롤](../../mfc/virtual-list-controls.md)을 참조하십시오.

- 1회 및 2회 클릭 활성화

   활성화하면 핫 트래킹(항목 텍스트의 자동 강조 표시)과 강조 표시된 항목의 1~2회 클릭 활성화를 허용합니다.

- 열 정렬 끌어서 놓기

   활성화된 경우 목록 보기 컨트롤에서 열을 끌어서 놓기 순서를 다시 정렬할 수 있습니다. 보고서 보기에서만 사용할 수 있습니다.

이러한 새 확장 스타일 사용에 대한 자세한 내용은 [CListCtrl 사용: 목록 제어 스타일 변경을](../../mfc/changing-list-control-styles.md)참조하십시오.

## <a name="items-and-subitems"></a>항목 및 하위 항목

목록 보기 컨트롤의 각 항목은 아이콘(이미지 목록), 레이블, 현재 상태 및 응용 프로그램 정의 값("항목 데이터"라고 함)으로 구성됩니다. 하나 이상의 하위 항목을 각 항목과 연결할 수도 있습니다. "하위 항목"은 보고서 보기에서 항목 아이콘 및 레이블 오른쪽의 열에 표시할 수 있는 문자열입니다. 목록 보기 컨트롤의 모든 항목에는 동일한 수의 하위 항목이 있어야 합니다.

클래스는 `CListCtrl` 이러한 항목을 삽입, 삭제, 찾기 및 수정하기 위한 몇 가지 기능을 제공합니다. 자세한 내용은 [CListCtrl::GetItem,](#getitem) [CListCtrl::InsertItem](#insertitem)및 [CListCtrl::FindItem:](#finditem) [컨트롤에 항목 추가](../adding-items-to-the-control.md)및 [스크롤, 정렬, 정렬 및 목록 컨트롤에서 찾기](../scrolling-arranging-sorting-and-finding-in-list-controls.md)를 참조하십시오.

기본적으로 목록 보기 컨트롤은 항목의 아이콘과 텍스트 속성을 저장합니다. 그러나 이러한 항목 유형 외에도 `CListCtrl` 클래스는 "콜백 항목"을 지원합니다. "콜백 항목"은 컨트롤이 아닌 응용 프로그램이 텍스트, 아이콘 또는 둘 다를 저장하는 목록 보기 항목입니다. 콜백 마스크는 응용 프로그램에서 제공하는 항목 특성(텍스트 및/또는 아이콘)을 지정하는 데 사용됩니다. 응용 프로그램에서 콜백 항목을 사용하는 경우 필요에 따라 텍스트 및/또는 아이콘 특성을 제공할 수 있어야 합니다. 콜백 항목은 응용 프로그램에서 이 정보 중 일부를 이미 유지 관리하는 경우에 유용합니다. 자세한 내용은 [CListCtrl: 콜백 항목 및 콜백 마스크 를 참조하십시오.](../callback-items-and-the-callback-mask.md)

## <a name="image-lists"></a>이미지 목록

목록 보기 항목에 대한 아이콘, 헤더 항목 이미지 및 응용 프로그램 정의 상태는 목록 보기 컨트롤을 만들고 할당하는 여러 이미지 목록(클래스 [CImageList에서](cimagelist-class.md)구현됨)에 포함됩니다. 각 목록 보기 컨트롤에는 최대 4가지 유형의 이미지 목록이 있을 수 있습니다.

- 큰 아이콘

   전체 크기의 아이콘에 대한 아이콘 보기에서 사용됩니다.

- 작은 아이콘

   아이콘 보기에 사용된 작은 버전의 아이콘에 대한 작은 아이콘, 목록 및 보고서 보기에 사용됩니다.

- 응용 프로그램 정의 상태

   항목아이콘 옆에 표시되는 상태 이미지를 포함하여 응용 프로그램 정의 상태를 나타냅니다.

- 헤더 항목

   각 헤더 제어 항목에 나타나는 작은 이미지에 대한 보고서 보기에 사용됩니다.

*iCount*<br/>

### <a name="example"></a>예제

```cpp
        // Reverse the order of the columns in the list view control
        // (i.e. make the first column the last, the last column
        // the first, and so on...).
        CHeaderCtrl* pHeaderCtrl = m_myListCtrl.GetHeaderCtrl();

        if (pHeaderCtrl != NULL)
        {
            int  nColumnCount = pHeaderCtrl->GetItemCount();
            LPINT pnOrder = (LPINT) malloc(nColumnCount*sizeof(int));
            ASSERT(pnOrder != NULL);
            m_myListCtrl.GetColumnOrderArray(pnOrder, nColumnCount);

            int i, j, nTemp;
            for (i = 0, j = nColumnCount-1; i < j; i++, j--)
            {
                nTemp = pnOrder[i];
                pnOrder[i] = pnOrder[j];
                pnOrder[j] = nTemp;
            }

            m_myListCtrl.SetColumnOrderArray(nColumnCount, pnOrder);
            free(pnOrder);
        }
```

## <a name="clistctrlgetitemrect"></a><a name="getitemrect"></a>CListCtrl::GetItemRect

현재 뷰에서 항목의 전체 또는 일부에 대한 경계 사각형을 검색합니다.

```

    int nItem,
    LPRECT lpRect,
    UINT nCode) const;
```

### <a name="parameters"></a>매개 변수

*nItem*<br/>

*Lprect*<br/>

*nCode*<br/>
경계 사각형을 검색할 목록 뷰 항목의 일부입니다. 다음 값 중 하나일 수 있습니다.

- LVIR_BOUNDS 아이콘과 레이블을 포함하여 전체 항목의 경계 사각형을 반환합니다.

- LVIR_ICON 아이콘 또는 작은 아이콘의 경계 사각형을 반환합니다.

- LVIR_LABEL 항목 텍스트의 경계 사각형을 반환합니다.

성공하는 경우 0이 아니고, 그렇지 않으면 0입니다.

### <a name="example"></a>예제

```cpp
// OnClick is the handler for the NM_CLICK notification
void CListCtrlDlg::OnClick(NMHDR* pNMHDR, LRESULT* pResult)
{
    UNREFERENCED_PARAMETER(pResult);
    LPNMITEMACTIVATE pia = (LPNMITEMACTIVATE)pNMHDR;

    // Get the current mouse location and convert it to client
    // coordinates.
    CPoint pos( ::GetMessagePos() );
    ScreenToClient(&pos);

    // Get indexes of the first and last visible items in
    // the listview control.
    int index = m_myListCtrl.GetTopIndex();
    int last_visible_index = index + m_myListCtrl.GetCountPerPage();
    if (last_visible_index > m_myListCtrl.GetItemCount())
        last_visible_index = m_myListCtrl.GetItemCount();

    // Loop until number visible items has been reached.
    while (index <= last_visible_index)
    {
        // Get the bounding rectangle of an item. If the mouse
        // location is within the bounding rectangle of the item,
        // you know you have found the item that was being clicked.
        CRect r;
        m_myListCtrl.GetItemRect(index, &r, LVIR_BOUNDS);
        if (r.PtInRect(pia->ptAction))
        {
            UINT flag = LVIS_SELECTED | LVIS_FOCUSED;
            m_myListCtrl.SetItemState(index, flag, flag);
            break;
        }

        // Get the next item in listview control.
        index++;
    }
}
```

## <a name="clistctrlhittest"></a><a name="hittest"></a>CListCtrl::히트 테스트

지정된 위치에 있는 목록 보기 항목(있는 경우)을 결정합니다.

```
int HitTest(LVHITTESTINFO* pHitTestInfo) const;

int HitTest(
    CPoint pt,
    UINT* pFlags = NULL) const;
```

### <a name="parameters"></a>매개 변수

*pHitTestInfo*<br/>
테스트를 적중할 수 있는 위치를 포함하고 적중 테스트 결과에 대한 정보를 수신하는 `LVHITTESTINFO` 구조의 주소입니다.

*pt*<br/>
테스트할 점입니다.

*pFlags*<br/>
테스트 결과에 대한 정보를 수신하는 정수에 대한 포인터입니다. Windows SDK에서 `flags` [LVHITTESTINFO](/windows/win32/api/commctrl/ns-commctrl-lvhittestinfo) 구조의 멤버에 대한 설명을 참조하십시오.

### <a name="return-value"></a>Return Value

*pHitTestInfo에*의해 지정된 위치에 있는 항목의 인덱스( 있는 경우) 또는 -1 그렇지 않은 경우.

### <a name="remarks"></a>설명

구조대 `flag` 구성원의 LVHT_ABOVE, LVHT_BELOW, LVHT_TOLEFT 및 LVHT_TORIGHT 값을 사용하여 목록 보기 컨트롤의 내용을 스크롤할지 여부를 결정할 수 있습니다. 예를 들어 위치가 클라이언트 영역의 위와 왼쪽에 있는 경우 이러한 플래그 중 두 개를 결합할 수 있습니다.

구조대 `flag` 원의 LVHT_ONITEM 값을 테스트하여 지정된 위치가 목록 보기 항목 위에 있는지 확인할 수 있습니다. 이 값은 구조체 `flag` 멤버의 LVHT_ONITEMICON, LVHT_ONITEMLABEL 및 LVHT_ONITEMSTATEICON 값에 대한 비트-OR 작업입니다.

### <a name="example"></a>예제

```cpp
void CListCtrlDlg::OnRClick(NMHDR* pNMHDR, LRESULT* pResult)
{
    LPNMITEMACTIVATE pia = (LPNMITEMACTIVATE)pNMHDR;
    CPoint point(pia->ptAction);

    // Select the item the user clicked on.
    UINT uFlags;
    int nItem = m_myListCtrl.HitTest(point, &uFlags);

    if (uFlags & LVHT_ONITEMLABEL)
    {
        m_myListCtrl.SetItem(nItem, 0, LVIF_STATE, NULL, 0, LVIS_SELECTED,
            LVIS_SELECTED, 0);
    }

    *pResult = 0;
}
```

### <a name="example"></a>예제

```cpp

        int nColumnCount = m_myListCtrl.GetHeaderCtrl()->GetItemCount();

        // Insert 10 items in the list view control.
        for (int i = 0; i < 10; i++)
        {
            strText.Format(TEXT("item %d"), i);

            // Insert the item, select every other item.
            m_myListCtrl.InsertItem(LVIF_TEXT | LVIF_STATE, i, strText,
                (i % 2) == 0 ? LVIS_SELECTED : 0, LVIS_SELECTED, 0, 0);

            // Initialize the text of the subitems.
            for (int j = 1; j < nColumnCount; j++)
            {
                strText.Format(TEXT("sub-item %d %d"), i, j);
                m_myListCtrl.SetItemText(i, j, strText);
            }
        }
```

```cpp

    int iCount = m_listCtrl.GetItemCount();
    UINT nId = 0;
    UINT nIndex = 0;
    for (int iIndexOriginal = 0; iIndexOriginal < iCount; iIndexOriginal++)
    {
```

### <a name="example"></a>예제

```cpp
    // Remove all working areas.
    m_myListCtrl.SetWorkAreas(0, NULL);
```

## <a name="clistctrlsortitems"></a><a name="sortitems"></a>CListCtrl::정렬 항목

응용 프로그램 정의 비교 함수를 사용하여 목록 보기 항목을 정렬합니다.

```
BOOL SortItems(
    PFNLVCOMPARE pfnCompare,
    DWORD_PTR dwData);
```

### <a name="parameters"></a>매개 변수

*pfnCompare*<br/>
【인】 응용 프로그램 정의 비교 함수의 주소입니다.

정렬 작업은 두 목록 항목의 상대 순서를 결정해야 할 때마다 비교 함수를 호출합니다. 비교 함수는 클래스의 정적 멤버이거나 클래스의 멤버가 아닌 독립 실행형 함수여야 합니다.

*dwData*<br/>
【인】 비교 함수에 전달되는 응용 프로그램 정의 값입니다.

### <a name="return-value"></a>Return Value

TRUE 메서드가 성공하면 그렇지 않으면 거짓.

### <a name="remarks"></a>설명

이 메서드는 각 항목의 인덱스를 변경하여 새 시퀀스를 반영합니다.

비교 *함수인 pfnCompare는*다음과 같은 형태를 가지고 있습니다.

```
int CALLBACK CompareFunc(LPARAM lParam1,
    LPARAM lParam2,
    LPARAM lParamSort);
```

비교 함수는 첫 번째 항목이 두 번째 항목 앞에 오는 경우 음수 값을 반환해야 하며, 첫 번째 항목이 두 번째 항목을 따라야 하는 경우 양수 값, 두 항목이 같으면 0을 반환해야 합니다.

*lParam1* 매개 변수는 비교되는 첫 번째 항목과 연결된 32비트 값이며 *lParam2* 매개 변수는 두 번째 항목과 연결된 값입니다. 이러한 값은 목록에 삽입될 때 항목의 [LVITEM](/windows/win32/api/commctrl/ns-commctrl-lvitemw) 구조의 *lParam* 멤버에 지정된 값입니다. *lParamSort* 매개 변수는 *dwData* 값과 동일합니다.

이 메서드는 Windows SDK에 설명 된 [LVM_SORTITEMS](/windows/win32/Controls/lvm-sortitems) 메시지를 보냅니다.

### <a name="example"></a>예제

다음은 *lParam* 값으로 정렬되는 항목을 만드는 간단한 비교 함수입니다.

```cpp
// Sort items by associated lParam
int CALLBACK CListCtrlDlg::MyCompareProc(LPARAM lParam1, LPARAM lParam2,
    LPARAM lParamSort)
{
    UNREFERENCED_PARAMETER(lParamSort);
    return (int)(lParam1 - lParam2);
}
```

```cpp
// Sort the items by passing in the comparison function.
void CListCtrlDlg::Sort()
{
    m_myListCtrl.SortItems(&CListCtrlDlg::MyCompareProc, 0);
}
```

## <a name="clistctrlsortitemsex"></a><a name="sortitemsex"></a>CListCtrl::정렬 항목 Ex

응용 프로그램 정의 비교 함수를 사용하여 현재 목록 보기 컨트롤의 항목을 정렬합니다.

```
BOOL SortItemsEx(
    PFNLVCOMPARE pfnCompare,
    DWORD_PTR dwData);
```

### <a name="parameters"></a>매개 변수

|매개 변수|Description|
|---------------|-----------------|
|*pfnCompare*|【인】 응용 프로그램 정의 비교 함수의 주소입니다.<br /><br /> 정렬 작업은 두 목록 항목의 상대 순서를 결정해야 할 때마다 비교 함수를 호출합니다. 비교 함수는 클래스의 정적 멤버이거나 클래스의 멤버가 아닌 독립 실행형 함수여야 합니다.|
|*dwData*|【인】 응용 프로그램 정의 값이 비교 함수에 전달되었습니다.|

### <a name="return-value"></a>Return Value

이 메서드가 성공하면 TRUE입니다. 그렇지 않으면 false입니다.

### <a name="remarks"></a>설명

이 메서드는 각 항목의 인덱스를 변경하여 새 시퀀스를 반영합니다.

비교 *함수인 pfnCompare는*다음과 같은 형태를 가지고 있습니다.

```
int CALLBACK CompareFunc(LPARAM lParam1,
    LPARAM lParam2,
    LPARAM lParamSort);
```

이 메시지는 비교 함수에 전달되는 정보 유형을 제외한 [LVM_SORTITEMS](/windows/win32/Controls/lvm-sortitems)것과 같습니다. [LVM_SORTITEMS](/windows/win32/Controls/lvm-sortitems) *lParam1* 및 *lParam2는* 비교할 항목의 값입니다. [LVM_SORTITEMSEX](/windows/win32/Controls/lvm-sortitemsex) *lParam1은* 비교할 첫 번째 항목의 현재 인덱스이고 *lParam2는* 두 번째 항목의 현재 인덱스입니다. [LVM_GETITEMTEXT](/windows/win32/Controls/lvm-getitemtext) 메시지를 보내 항목에 대한 자세한 정보를 검색할 수 있습니다.

비교 함수는 첫 번째 항목이 두 번째 항목 앞에 오는 경우 음수 값을 반환해야 하며, 첫 번째 항목이 두 번째 항목을 따라야 하는 경우 양수 값, 두 항목이 같으면 0을 반환해야 합니다.

> [!NOTE]
> 정렬 프로세스 중에 목록 보기 내용이 불안정합니다. 콜백 함수가 [LVM_GETITEM](/windows/win32/Controls/lvm-getitem)이외의 목록 보기 컨트롤에 메시지를 보내는 경우 결과를 예측할 수 없습니다.

이 메서드는 Windows SDK에 설명 된 [LVM_SORTITEMSEX](/windows/win32/Controls/lvm-sortitemsex) 메시지를 보냅니다.

### <a name="example"></a>예제

다음 코드 예제에서는 현재 `m_listCtrl`목록 보기 컨트롤에 액세스 하는 데 사용 되는 변수를 정의 합니다. 이 변수는 다음 예제에서 사용됩니다.

```cpp
public:
    // Variable used to access the list control.
    CListCtrl m_listCtrl;
```

### <a name="example"></a>예제

다음 코드 예제는 `SortItemEx` 메서드. 이 코드 예제의 이전 섹션에서는 보고서 보기에 "ClientID" 및 "Grade"라는 두 개의 열을 표시하는 목록 보기 컨트롤을 만들었습니다. 다음 코드 예제에서는 "Grade" 열의 값을 사용하여 테이블을 정렬합니다.

```cpp
// The ListCompareFunc() method is a global function used by SortItemEx().
int CALLBACK ListCompareFunc(
                             LPARAM lParam1,
                             LPARAM lParam2,
                             LPARAM lParamSort)
{
    CListCtrl* pListCtrl = (CListCtrl*) lParamSort;
    CString    strItem1 = pListCtrl->GetItemText(static_cast<int>(lParam1), 1);
    CString    strItem2 = pListCtrl->GetItemText(static_cast<int>(lParam2), 1)
    int x1 = _tstoi(strItem1.GetBuffer());
    int x2 = _tstoi(strItem2.GetBuffer());
    int result = 0;
    if ((x1 - x2) < 0)
        result = -1;
    else if ((x1 - x2) == 0)
        result = 0;
    else
        result = 1;

    return result;
}

void CCListCtrl_s2Dlg::OnBnClickedButton1()
{
    // SortItemsEx
    m_listCtrl.SortItemsEx( ListCompareFunc, (LPARAM)&m_listCtrl );
}
```

## <a name="clistctrlsubitemhittest"></a><a name="subitemhittest"></a>CListCtrl::하위 항목 히트 테스트

```

```

### <a name="parameters"></a>매개 변수

*pInfo*<br/>
[LVHITTESTINFO](/windows/win32/api/commctrl/ns-commctrl-lvhittestinfo) 구조에 대한 포인터입니다.

### <a name="return-value"></a>Return Value

테스트 중인 항목 또는 하위 항목의 한 기반 인덱스(있는 경우) 또는 -1

### <a name="remarks"></a>설명

이 멤버 함수는 Windows SDK에 설명된 대로 Win32 매크로 [ListView_SubItemHitTest](/windows/win32/api/commctrl/nf-commctrl-listview_subitemhittest)동작을 구현합니다.

### <a name="example"></a>예제

```cpp
void CListCtrlDlg::OnDblClk(NMHDR* pNMHDR, LRESULT* pResult)
{
    UNREFERENCED_PARAMETER(pResult);
    LPNMITEMACTIVATE pia = (LPNMITEMACTIVATE)pNMHDR;
    LVHITTESTINFO lvhti;

    // Clear the subitem text the user clicked on.
    lvhti.pt = pia->ptAction;
    m_myListCtrl.SubItemHitTest(&lvhti);

    if (lvhti.flags & LVHT_ONITEMLABEL)
    {
        m_myListCtrl.SetItemText(lvhti.iItem, lvhti.iSubItem, NULL);
    }
}
```
