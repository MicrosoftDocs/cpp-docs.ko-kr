프레임 창에 대한 MDI 탭 그룹 기능을 활성화하거나 사용하지 않도록 설정합니다.

```
void EnableMDITabbedGroups(
    BOOL bEnable,
    const CMDITabInfo& params);
```

### <a name="parameters"></a>매개 변수

*bEnable*<br/>
【인】 TRUE이면 MDI 탭 그룹 기능이 활성화됩니다. FALSE인 경우 MDI 탭 그룹 기능이 비활성화됩니다.

*params*<br/>
【인】 프레임워크가 MDI 클라이언트 영역에서 생성된 자식 창에 적용되는 매개 변수를 지정합니다.

### <a name="remarks"></a>설명

이 방법을 사용하여 MDI 탭 그룹 기능을 사용하거나 사용하지 않도록 설정합니다. 이 기능을 사용하면 MDI 응용 프로그램에서 자식 창을 MDI 클라이언트 영역 내에서 수직 또는 수평으로 정렬된 탭창으로 표시할 수 있습니다. 탭 창의 그룹은 분할자로 구분됩니다. 사용자는 스플리터를 사용하여 탭된 그룹의 크기를 조정할 수 있습니다.

- 사용자는 다음을 수행할 수 있습니다.

- 그룹 간에 개별 탭을 드래그합니다.

- 개별 탭을 창 가장자리로 드래그하여 새 그룹을 만듭니다.

- 바로 가기 메뉴를 사용하여 탭을 이동하거나 새 그룹을 만듭니다.

- 응용 프로그램은 탭된 창의 현재 레이아웃과 현재 열려 있는 문서 목록을 저장할 수 있습니다.

*bEnable* set을 FALSE로 사용하여 이 메서드를 호출하면 *매개 변수가* 무시됩니다.

MDI 탭 그룹이 이미 활성화되어 있더라도 이 메서드를 다시 호출하여 자식 창에 대한 설정을 수정할 수 있습니다. *bEnable* set을 TRUE로 사용하여 메서드를 호출하고 `CMDITabInfo` *params* 매개 변수에 의해 지정된 개체의 멤버를 수정합니다.

MDI 탭 그룹을 사용하는 방법에 대한 자세한 내용은 [MDI 탭 그룹](../../mfc/mdi-tabbed-groups.md)을 참조하십시오.

## <a name="cmdiframewndexsavemdistate"></a><a name="savemdistate"></a>CMDIFrameWndEx::SaveMDIState

MDI 탭 그룹의 현재 레이아웃과 이전에 열린 문서 목록을 저장합니다.

```
virtual BOOL SaveMDIState(LPCTSTR lpszProfileName);
```

### <a name="parameters"></a>매개 변수

*lpszProfileName*<br/>
【인】 프로파일 이름을 지정합니다.

### <a name="return-value"></a>Return Value

저장이 성공하면 TRUE입니다. 저장에 실패한 경우 FALSE입니다.

### <a name="remarks"></a>설명

MDI 탭 및 그룹 및 열린 문서 목록을 로드하거나 저장하려면 다음을 수행합니다.

- 주 `SaveMDIState` 프레임이 닫혀 있을 때 호출

- 주 프레임이 생성될 때 [CMDIFrameWndEx::LoadMDIState를](#loadmdistate) 호출합니다. 이 호출에 권장되는 위치는 주 프레임이 처음으로 표시되기 전입니다.

- 전에 `CWinAppEx::EnableLoadWindowPlacement(FALSE);` 전화`pMainFrame->LoadFrame (IDR_MAINFRAME);`

- 레지스트리에 `LoadMDIState` 저장된 위치에 주 프레임을 표시하려면 호출합니다. `CWinAppEx::ReloadWindowPlacement(pMainFrame)`

- 응용 프로그램이 파일로 저장되지 않은 문서를 표시하는 경우 - 파생 클래스에서 재정의합니다. `GetDocumentName` `CMDIChildWndEx` 반환된 문자열은 레지스트리에 문서 식별자로 저장됩니다. 자세한 내용은 [CMDIChildWndEx::GetDocumentName](../../mfc/reference/cmdichildwndex-class.md#getdocumentname)을 참조하십시오.

- [CMDIFrameWndEx::CreateDocumentWindow를](#createdocumentwindow) 재정의하여 레지스트리에서 로드될 때 문서를 올바르게 만듭니다. 매개 `CreateDocumentWindow` 변수는 이전에 `GetDocumentName` 반환된 문자열입니다.
