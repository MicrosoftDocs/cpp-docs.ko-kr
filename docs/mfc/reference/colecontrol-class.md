## <a name="remarks"></a>설명

파생 된 `CWnd`,이 클래스는 Windows 창 개체의 모든 기능 및 특정 기능을 추가 OLE 이벤트 발생 등 메서드 및 속성을 지 원하는 기능을 상속 합니다.

OLE 컨트롤 OLE 컨테이너 응용 프로그램에 삽입할 수 및 이벤트 발생 및 노출 메서드 및 속성을 컨테이너의 양방향 시스템을 사용 하 여 컨테이너를 사용 하 여 통신 합니다. 참고 표준 OLE 컨테이너에만 OLE 컨트롤의 기본 기능을 지원 합니다. OLE 컨트롤의 확장된 기능을 지원할 수 없는 합니다. 이벤트 발생 이벤트는 컨트롤에서 수행 하는 특정 작업의 결과로 컨테이너에 전송 되 면 발생 합니다. 따라서 컨테이너 멤버 함수는 유사한 속성과 메서드 집합을 노출된 하 고 c + + 클래스의 데이터 멤버를 사용 하 여 컨트롤을 사용 하 여 통신 합니다. 이 방법을 통해 개발자가 컨트롤의 모양을 제어 하 고 특정 작업이 발생 하면 컨테이너를 알릴 수 있습니다.

## <a name="windowless-controls"></a>창 없는 컨트롤

OLE 컨트롤 창 없이 사용 되는 내부 활성화 될 수 있습니다. 창 없는 컨트롤 가지 중요 한 이점이 있습니다.

- 투명 한 사각형이 아닌 창 없는 컨트롤 수 있습니다.

- 창 없는 컨트롤 개체의 인스턴스 크기와 만든 시간을 줄이기

컨트롤을 창이 필요는 없습니다. 서비스 창에서 제공 하는 단일 공유 창 (일반적으로 컨테이너의) 및 약간의 디스패치 코드를 통해 쉽게 제공할 수 있습니다. 창 개체에는 불필요 한 complication 대부분입니다.

창 없는 활성화를 사용 하면 (갖지 않으므로 창)는 컨테이너는 컨트롤의 창에서 제공은 서비스를 제공 하는 일을 담당 합니다. 예를 들어 컨트롤이 키보드 포커스를 쿼리하거나, 마우스 캡처를 쿼리 또는 장치 컨텍스트를 가져와야 하는 경우 이러한 작업은 컨테이너에 의해 관리 됩니다. `COleControl` 컨테이너에서 이러한 작업을 호출 하는 작업 창 없는 멤버 함수입니다.

컨테이너를 위임 하는 입력 메시지를 컨트롤의 창 없는 활성화를 사용 하는 경우 `IOleInPlaceObjectWindowless` 인터페이스 (의 확장인 [위해서는](/windows/desktop/api/oleidl/nn-oleidl-ioleinplaceobject) 창 없는 지원에 대 한). `COleControl`조정 마우스 좌표를 적절히 후이 인터페이스의 구현에서 컨트롤의 메시지 맵을 통해 이러한 메시지를 발송 합니다. 해당 항목을 메시지 맵에 추가 하 여 일반적인 창 메시지와 마찬가지로 이러한 메시지를 처리할 수 있습니다.

창 없는 컨트롤에서 항상 사용 해야 합니다 `COleControl` 멤버 함수는 해당 하는 대신 `CWnd` 멤버 함수 또는 해당 관련된 Windows API 함수입니다.

OLE 컨트롤 개체 활성화 되기 있지만 비활성-활성 전환 하는 데 필요한 작업 시간을 이동 및 전환의 속도 중단 하는 경우에 창을 만들 수도 있습니다. 문제가 발생 하는 경우가 있습니다: 예를 들어 입력란의 표는 것이 좋습니다. 때 열을 위아래로 커서, 각 컨트롤 이어야 합니다 전체 활성화 및 비활성화 합니다. 활성/비활성 전환의 속도 스크롤 속도가 저하 직접 될 합니다.

OLE 컨트롤 프레임 워크를 개발에 대 한 자세한 내용은 문서를 참조 하세요 [MFC ActiveX 컨트롤](../../mfc/mfc-activex-controls.md) 하 고 [개요: MFC ActiveX 컨트롤 프로그램을 만드는](../../mfc/reference/mfc-activex-control-wizard.md)합니다. OLE 컨트롤, 창 및 깜박임 컨트롤을 포함 하 여 최적화에 대 한 내용은 [MFC ActiveX 컨트롤: 최적화](../../mfc/mfc-activex-controls-optimization.md)합니다.

##  <a name="getactivationpolicy"></a>  COleControl::GetActivationPolicy

지 원하는 컨트롤의 기본 활성화 동작을 변경 하 여 `IPointerInactive` 인터페이스입니다.

```
virtual DWORD GetActivationPolicy();
```

### <a name="return-value"></a>반환 값

POINTERINACTIVE 열거형에서 플래그의 조합입니다. 가능한 플래그는:

POINTERINACTIVE_ACTIVATEONENTRY 개체 내부에서 마우스 마우스 이동 작업 중 입력 하면 활성화 해야 합니다.

POINTERINACTIVE_DEACTIVATEONLEAVE 마우스 마우스 중 개체를 벗어나면 개체를 비활성화 해야 작업을 이동 합니다.

POINTERINACTIVE_ACTIVATEONDRAG 개체 끌기 중 위로 마우스를 끌 때 활성화 될 수 하 고 작업을 삭제 해야 합니다.

### <a name="remarks"></a>설명

경우는 `IPointerInactive` 인터페이스를 사용 하면 컨테이너 WM_SETCURSOR 및 WM_MOUSEMOVE 메시지를 위임 합니다. `COleControl`조정 마우스 좌표를 적절히 후이 인터페이스의 구현에서 컨트롤의 메시지 맵을 통해 이러한 메시지를 발송 합니다.

컨테이너를 지 원하는 비활성 개체 위에 마우스 포인터가 있으면 WM_SETCURSOR 또는 WM_MOUSEMOVE 메시지를 수신 하는 때마다 `IPointerInactive`를 호출 해야 `GetActivationPolicy` 인터페이스에 POINTERINACTIVE 열거형에서 플래그를 반환 합니다.

해당 항목을 메시지 맵에 추가 하 여 일반적인 창 메시지와 마찬가지로 이러한 메시지를 처리할 수 있습니다. 처리기를 사용 하지 않습니다는 `m_hWnd` 멤버 변수 (또는 사용 하는 모든 멤버 함수) 첫 번째 하지 않고 해당 값이 NULL이 아닌 인지 확인 합니다.

수행할 여러 마우스 커서를 설정 및/또는와 같은 마우스 이동 이벤트를 발생 하는 데 개체 특수 한 시각적 피드백을 제공, POINTERINACTIVE_ACTIVATEONENTRY 플래그를 반환 고 활성 상태인 경우에 피드백을 그리는 해야 합니다. 컨테이너 전체 것을 즉시 정품 인증 하며 동일한 메시지에 대 한 호출을 트리거한 전달이 플래그를 반환 하는 개체, 경우 `GetActivationPolicy`합니다.

POINTERINACTIVE_ACTIVATEONENTRY와 POINTERINACTIVE_DEACTIVATEONLEAVE 플래그 반환 되는 경우 다음 개체만 활성화 됩니다 개체 위에 마우스가 있을 때. POINTERINACTIVE_ACTIVATEONENTRY 플래그 반환 되는지, 그런 다음 개체만 활성화 됩니다 때 마우스 개체를 처음 실행 되 면 합니다.

##  <a name="getcontrolflags"></a>  COleControl::GetControlFlags

제어 플래그 설정을 검색합니다.

```
virtual DWORD GetControlFlags();
```

### <a name="return-value"></a>반환 값

ControlFlags 열거의 플래그는 ORed 조합:

```
enum ControlFlags {
    fastBeginPaint = 0x0001,
    clipPaintDC = 0x0002,
    pointerInactive = 0x0004,
    noFlickerActivate = 0x0008,
    windowlessActivate = 0x0010,
    canOptimizeDraw = 0x0020,
    };
```

### <a name="remarks"></a>설명

기본적으로 `GetControlFlags` 반환 `fastBeginPaint | clipPaintDC`합니다.

`fastBeginPaint` 경우 설정 사용을 시작-그리기 함수에 맞게 대신 OLE 컨트롤의 [BeginPaint](/windows/desktop/api/winuser/nf-winuser-beginpaint) API (기본적으로 설정 됨).

`clipPaintDC` 그렇지 않은 경우 설정에 대 한 호출을 사용 하지 않도록 설정 `IntersectClipRect` 수행한 `COleControl` 및 속도가 약간 빨라집니다. 창 없는 활성화를 사용 하는 경우 플래그는 영향을 주지 않습니다.

`pointerInactive` 경우 설정, 제공 마우스 상호 작용 하 고 컨트롤을 사용 하 여 활성화 되지 `COleControl`의 구현을 `IPointerInactive` 인터페이스를 기본적으로 비활성화 됩니다.

`noFlickerActivate` 경우 설정, 추가 그리기 작업 및 깜빡임의 제거 합니다. 컨트롤의 비활성 및 활성 상태에서 자신을 동일 하 게 그릴 때 사용 합니다. 창 없는 활성화를 사용 하는 경우 플래그는 영향을 주지 않습니다.

`windowlessActivate` 경우 설정, 창 없는 활성화를 사용 하는 컨트롤을 나타냅니다.

`canOptimizeDraw` 경우 설정, 컨트롤 그리기 최적화를 수행 되는 컨테이너에서 지 원하는 경우를 나타냅니다.

에 대 한 자세한 내용은 `GetControlFlags` 다른 최적화 OLE 컨트롤의 내용과 [ActiveX 컨트롤: 최적화](../../mfc/mfc-activex-controls-optimization.md)합니다.

##  <a name="getdc"></a>  COleControl::GetDC

창 없는 개체에 대 한 get 화면 (또는 호환) 장치 컨텍스트에 해당 컨테이너에서를 제공합니다.

```
CDC* GetDC(
    LPCRECT lprcRect = NULL,
    DWORD dwFlags = OLEDC_PAINTBKGND);
```

### <a name="parameters"></a>매개 변수

*lprcRect*<br/>
사각형 창 없는 컨트롤에 대 한 포인터를 컨트롤의 클라이언트 좌표에서 다시 그릴 하려고 합니다. NULL은 전체 개체의 범위를 의미 합니다.

*dwFlags*<br/>
장치 컨텍스트의 그리기 특성입니다. 선택할 수 있습니다.

- OLEDC_NODRAW 나타냅니다 개체 모든 그리기 하지만 단순히 디스플레이 장치에 대 한 정보를 가져오려고 하는 데 장치 컨텍스트를 사용 하지 않습니다. 단순히 컨테이너 추가적인 처리 없이 창의 DC를 전달 해야 합니다.

- DC를 반환 하기 전에 컨테이너 배경을 칠하는 OLEDC_PAINTBKGND 요청 합니다. 개체는 투명 한 배경 사용 하 여 영역 다시 그리기에 대 한 DC를 요청 하는 경우이 플래그를 사용 해야 합니다.

- OLEDC_OFFSCREEN 개체 다음 화면에 복사 해야 하는 오프 스크린 비트맵으로 렌더링 하는 컨테이너를 알립니다. 개체는 그리기 작업을 수행 하려고 하는 깜박임 많이 생성 하는 경우이 플래그를 사용 해야 합니다. 컨테이너는 여부이 요청을 처리할 수 있습니다. 그러나이 플래그를 설정 하지 않으면 컨테이너 해야 다시 전달할는 화면 DC입니다. 이 따라 직접 화면 표시를 선택 하는 등의 작업을 수행 하는 개체 (통해는 **XOR** 작업).

### <a name="return-value"></a>반환 값

컨테이너에 대 한 디스플레이 장치 컨텍스트에 대 한 포인터 `CWnd` 클라이언트 영역의 경우이 고, 그렇지 않으면, 반환 값은 NULL입니다. 디스플레이 장치 컨텍스트는 컨테이너 창의 클라이언트 영역에 그릴 이후의 GDI 함수에서 사용할 수 있습니다.

### <a name="remarks"></a>설명

합니다 [ReleaseDC](#releasedc) 그리기 후 컨텍스트를 해제 하려면 멤버 함수를 호출 해야 합니다. 호출할 때 `GetDC`, 개체 자체 클라이언트 좌표에서에 그릴 하려는 사각형을 전달 합니다. `GetDC` 이러한 컨테이너 클라이언트 영역의 좌표를 변환합니다. 개체는 원하는 그리기 사각형의 크기는를 사용 하 여 검색할 수는 자체 클라이언트 영역 사각형 보다 큰 요청 하지 [GetClientRect](#getclientrect)합니다. 이 위치에 의도 하지 않은 실수로 드로잉에서 개체를 방지 합니다.

##  <a name="onqueryhitpoint"></a>  COleControl::OnQueryHitPoint

컨테이너에 대 한 응답에서 프레임 워크에서 호출 `IViewObjectEx::QueryHitPoint` 요청 합니다.

```
virtual BOOL OnQueryHitPoint(
    DWORD dwAspect,
    LPCRECT pRectBounds,
    POINT ptlLoc,
    LONG lCloseHint,
    DWORD* pHitResult);
```

### <a name="parameters"></a>매개 변수

*dwAspect*<br/>
개체는 표시 하는 방법을 지정 합니다. 유효한 값은 열거형에서 가져온 [DVASPECT](/windows/desktop/api/wtypes/ne-wtypes-tagdvaspect) 또는 DVASPECT2 합니다.

*pRectBounds*<br/>
에 대 한 포인터를 `RECT` 구조 OLE 컨트롤 클라이언트 영역의 경계 사각형을 지정 합니다.

*ptlLoc*<br/>
에 대 한 포인터를 `POINT` 구조를 적중 검사할 지점을 지정 합니다. 지점은 OLE 클라이언트 영역 좌표로 지정 됩니다.

*lCloseHint*<br/>
"닫기" 지점에 적중에 대 한 검사를 정의 하는 거리입니다.

*pHitResult*<br/>
적중 쿼리의 결과에 대 한 포인터입니다. 다음 값 중 하나입니다.

- HITRESULT_OUTSIDE *ptlLoc* OLE 외부 개체를 닫지 됩니다.

- HITRESULT_TRANSPARENT *ptlLoc* 이미지 근접 하지 않지만 OLE 개체의 범위 안에 있습니다. 예를 들어, 투명 한 원 가운데에 있는 점을 HITRESULT_TRANSPARENT 수 있습니다.

- HITRESULT_CLOSE *ptlLoc* 내부 또는 외부 OLE 개체 이지만 가까이 개체 내에서 것으로 간주 됩니다. 작은, thin, 세부 개체는이 값을 사용할 수 있습니다. 점이 경계 외부에 있는 경우에 계속 수 하는 개체의 사각형 인접 하 게 (작은 개체에 도달 하는 것에 대 한 필요 됩니다).

- HITRESULT_HIT *ptlLoc* 이 개체의 이미지 내에 있습니다.

### <a name="return-value"></a>반환 값

0이 아닌 적중된 결과 반환 했습니다. 그렇지 않으면 0입니다. 적중은 OLE 컨트롤 표시 영역을 사용 하 여 중복 됩니다.

### <a name="remarks"></a>설명

지정 된 개체의 표시 사각형에 겹치는 지 여부를 쿼리 (지점에 도달). `QueryHitPoint` 사각형이 아닌 개체에 대 한 적중 테스트를 재정의할 수 있습니다.

##  <a name="onrenderdata"></a>  COleControl::OnRenderData

```

```

### <a name="parameters"></a>매개 변수

성공하면 0이 아니고, 그렇지 않으면 0입니다.

지정 된 형식을 사용 하 여 컨트롤 개체에 이전에 배치 하나를 [DelayRenderData](../../mfc/reference/coledatasource-class.md#delayrenderdata) 하거나 [DelayRenderFileData](../../mfc/reference/coledatasource-class.md#delayrenderfiledata) 지연 된 렌더링에 대 한 멤버 함수입니다. 이 함수의 기본 구현이 호출 `OnRenderFileData` 또는 `OnRenderGlobalData`각각 제공 되는 저장소 미디어 파일 또는 메모리 경우. 요청된 된 형식으로 CF_METAFILEPICT 또는 영구 속성 집합 형식인 경우 기본 구현은 적절 한 데이터를 렌더링 하 고 0이 아닌 값을 반환 합니다. 그렇지 않으면 0을 반환 하 고 아무 작업도 수행 하지.

요청 된 형식 및 미디어 데이터를 제공 하려면이 함수를 재정의 합니다. 데이터에 따라 대신이 함수의 다른 버전 중 하나를 재정의 하는 것이 좋습니다. 데이터가 작고 크기가 고정 되어 있으면 재정의 `OnRenderGlobalData`합니다. 데이터 파일에는 가변 크기의 경우 재정의 `OnRenderFileData`합니다.

자세한 내용은 참조는 `FORMATETC` 및 `STGMEDIUM` Windows SDK에는 구조입니다.

- HTGROWBOX 크기 상자에 있습니다.

- HTHSCROLL 가로 스크롤 막대의 합니다.

- HTLEFT에서 창의 왼쪽된 테두리입니다.

- 한 최대화에서 HTMAXBUTTON 단추입니다.

- HTMENU 메뉴 영역에서 합니다.

- 최소화 HTMINBUTTON 단추입니다.

- HTNOWHERE 화면 배경 기반 컴퓨터나 windows 사이 구분선입니다.

- 최소화 HTREDUCE 단추입니다.

- HTRIGHT에서 창의 오른쪽 테두리입니다.

- HTSIZE 크기 상자에 (HTGROWBOX 동일).

- 컨트롤의 HTSYSMENU 메뉴 또는 자식 창에서 닫기 단추.

- HTTOP에서 창의 위쪽 가로 테두리입니다.

- HTTOPLEFT의 창 테두리의 왼쪽 위 모퉁이.

### <a name="return-value"></a>반환 값

### <a name="remarks"></a>설명

일부 컨테이너에서 컨트롤 액세스할 수 없는 해당 앰비언트 속성에 초기 재정의 호출 하는 동안 `COleControl::DoPropExchange`합니다. 컨테이너를 호출 하는 경우이 대/소문자 [IPersistStreamInit::Load](/windows/desktop/api/ocidl/nf-ocidl-ipersiststreaminit-load) 하거나 [IPersistStorage::Load](/windows/desktop/api/objidl/nf-objidl-ipersiststorage-load) 호출 하기 전에 [IOleObject::SetClientSite](/windows/desktop/api/oleidl/nf-oleidl-ioleobject-setclientsite) (즉, 그렇지 않은 경우 인식 OLEMISC_SETCLIENTSITEFIRST 상태 비트).

##  <a name="windowproc"></a>  COleControl::WindowProc

에 대 한 Windows 절차를 제공 된 `COleControl` 개체입니다.

```
virtual LRESULT WindowProc(
    UINT message,
    WPARAM wParam,
    LPARAM lParam);
```

### <a name="parameters"></a>매개 변수

*message*<br/>
처리할 Windows 메시지를 지정 합니다.

*wParam*<br/>
메시지 처리에 사용 되는 추가 정보를 제공 합니다. 매개 변수 값을 메시지에 따라 달라 집니다.

*lParam*<br/>
메시지 처리에 사용 되는 추가 정보를 제공 합니다. 매개 변수 값을 메시지에 따라 달라 집니다.

### <a name="return-value"></a>반환 값

디스패치 된 메시지의 반환 값입니다.

### <a name="remarks"></a>설명

컨트롤의 메시지 맵을 통해 특정 메시지를 발송 하는 데이 함수를 호출 합니다.
