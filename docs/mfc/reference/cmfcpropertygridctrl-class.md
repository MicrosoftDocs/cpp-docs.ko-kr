## <a name="remarks"></a>설명

클래스는 `CMFCPropertyGridCtrl` [CMFCPropertyGridProperty](../../mfc/reference/cmfcpropertygridproperty-class.md) 클래스에서 파생된 편집 가능한 속성을 포함하는 속성 그리드 컨트롤을 표시합니다. 각 속성은 형식을 나타낼 수 있으며 하위 항목을 포함할 수 있습니다. 속성 그리드 컨트롤은 선택한 속성에 대한 설명을 표시할 수 있는 아래쪽의 재지정 영역을 지원합니다.

속성 그리드 컨트롤을 사용 `CMFCPropertyGridCtrl` 하려면 개체를 구성 하 고 [CMFCPropertyGridCtrl:만들기 메서드를](#create) 호출 합니다. [CMFCPropertyGridCtrl::AddProperty](#addproperty) 메서드를 사용하여 목록에 속성을 추가합니다.

## <a name="cmfcpropertygridctrladdproperty"></a><a name="addproperty"></a>CMFCProperty그리드Ctrl::추가 속성

```
int AddProperty(
    CMFCPropertyGridProperty* pProp,
    BOOL bRedraw=TRUE,
    BOOL bAdjustLayout=TRUE);
```

### <a name="parameters"></a>매개 변수

*pProp*<br/>
【인】 속성에 대한 포인터입니다.

*bRedraw*<br/>
【인】 TRUE는 즉시 속성을 다시 그릴; 그렇지 않으면 false입니다. 기본값은 TRUE입니다.

*bAdjust레이아웃*<br/>
【인】 TRUE는 속성의 텍스트와 값을 그리는 방법을 다시 계산한 다음 속성을 그립니다. FALSE는 기존 계산을 사용하여 속성을 그립니다. 기본값은 TRUE입니다.

이 메서드는 속성 그리드 컨트롤의 속성 목록 의 끝에 지정 된 속성에 대 한 포인터를 추가 합니다. 그리드 컨트롤이 소멸되기 전에 속성을 삭제하거나 범위를 벗어나지 않도록 하십시오. 속성 그리드 컨트롤이 완료되면 [CMFCPropertyGridCtrl::RemoveAll을](#removeall) 호출하여 추가된 모든 속성을 삭제합니다. 지정된 속성이 목록에 이미 추가된 경우 AddProperty 메서드가 실패합니다.

## <a name="cmfcpropertygridctrlhittest"></a><a name="hittest"></a>CMFCProperty그리드트rl::히트테스트

지정된 점이 항목에 있는 경우 속성 그리드 제어 항목에 해당하는 속성 개체에 대한 포인터를 검색합니다. 이 메서드는 점 포함 하는 속성 그리드 컨트롤의 영역도 나타냅니다.

```
CMFCPropertyGridProperty* HitTest(
    CPoint pt,
    CMFCPropertyGridProperty::ClickArea* pnArea=NULL,
    BOOL bPropsOnly=FALSE) const;
```

### <a name="parameters"></a>매개 변수

*pt*<br/>
【인】 클라이언트 좌표의 한 점입니다.

*pnArea*<br/>
【인, 아웃】 변수에 대한 `ClickArea` 포인터입니다. 이 메서드가 반환될 때 변수는 지정된 점을 포함하는 *속성 영역을* 나타냅니다. 속성 영역에 대한 자세한 내용은 비고를 참조하십시오.

*bProps전용*<br/>
【인】 속성 영역만 테스트하는 TRUE; FALSE는 지정된 점이 속성 영역에 없는 경우 *설명* 영역을 테스트합니다. 기본값은 FALSE입니다. 설명 영역에 대한 자세한 내용은 비고를 참조하십시오.

### <a name="return-value"></a>Return Value

*bPropsOnly* 매개 변수가 TRUE이고 지정된 점이 속성 영역에 있는 경우 반환 값은 해당 속성 개체에 대한 포인터입니다. 또한 *pnArea* 매개 변수는 지정된 점을 포함하는 특정 영역으로 설정됩니다. 그렇지 않으면 반환 값은 NULL이고 *pnArea* 매개 변수는 수정되지 않습니다.

*bPropsOnly* 매개 변수가 FALSE인 경우 반환 값은 항상 NULL입니다. 그러나 지정된 점이 설명 영역에 있으면 *pnArea* 매개 변수가 `CMFCPropertyGridProperty::ClickDescription`로 설정됩니다.

### <a name="remarks"></a>설명

속성 *영역이라는* 용어는 속성 그리드 제어 항목의 이름, 값 또는 확장 상자 영역 중 하나를 나타냅니다. *설명 영역은* 속성 그리드 컨트롤의 맨 아래에 있는 영역입니다. 속성 그리드 제어 항목을 클릭하면 설명 영역에 해당 속성에 대한 설명이 표시됩니다.

이 메서드는 *pnArea* 매개 변수가 가리키는 변수의 값을 설정합니다. 다음 표에는 가능한 값과 해당 영역이 나열되어 있습니다.

|값|영역|
|-----------|----------|
|`ClickArea::ClickExpandBox`|속성 확장 상자 컨트롤입니다.|
|`ClickArea::ClickName`|속성 이름입니다.|
|`ClickArea::ClickValue`|속성 값.|
|`CMFCPropertyGridProperty::ClickDescription`|속성 그리드 제어 설명 영역입니다.|

## <a name="cmfcpropertygridctrlsetgroupnamefullwidth"></a><a name="setgroupnamefullwidth"></a>CMFCProperty그리드Ctrl::집합그룹이름전체폭

현재 속성 그리드 컨트롤의 속성 그룹에 대한 범주 이름의 전체 너비를 표시할지 여부를 지정합니다.

```
void SetGroupNameFullWidth(
    BOOL bGroupNameFullWidth = TRUE,
    BOOL bRedraw = TRUE);
```

### <a name="parameters"></a>매개 변수

*b그룹네임전체폭*<br/>
【인】 TRUE를 사용하여 속성 이름 열의 너비에 관계없이 범주 이름의 전체 너비를 표시합니다. FALSE는 범주 이름의 너비를 속성 이름 열의 너비로 제한합니다. 기본값은 TRUE입니다.

*bRedraw*<br/>
【인】 TRUE는 속성 그리드 제어를 즉시 업데이트합니다. FALSE는 다음 다시 그리기 이벤트가 발생할 때 컨트롤을 업데이트합니다. 기본값은 TRUE입니다.

### <a name="remarks"></a>설명

속성 그리드 컨트롤은 큰식 *속성 이름* 열과 *속성 값* 열로 구성됩니다. 이름 열의 끝은 값 열의 시작이기도 합니다. 열크기를 조정하려면 열 사이의 테두리를 끕습니다.

용어 *그룹 이름* 및 *범주 이름은* 이 메서드에서 서로 바꿔 사용됩니다. 범주 이름은 관련 속성 및 값 집합을 이끄는 행에 표시됩니다. 이 메서드는 속성 이름 열의 너비표시 된 범주 이름의 너비도 지정 하는지 여부를 지정 합니다.

### <a name="parameters"></a>매개 변수
