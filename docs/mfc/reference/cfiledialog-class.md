## <a name="remarks"></a>설명

일반적인 파일 대화 상자를 사용하면 파일 선택 대화 상자를 구현할 수 있습니다(예: 파일 열기 및 **로 저장**)을 Windows 표준과 일치하는 방식으로 구현할 **수** 있습니다.

제공된 생성자와 마찬가지로 사용하거나 `CFileDialog` 사용자 고유의 대화 상자 클래스를 `CFileDialog` 파생시키고 필요에 맞게 생성기를 작성할 수 있습니다. 두 경우 모두 이러한 대화 상자는 [CCommonDialog 클래스에서](../../mfc/reference/ccommondialog-class.md)파생되므로 표준 MFC 대화 상자처럼 동작합니다. `CFileDialog`COMMDLG에 의존합니다. Windows에 포함된 DLL 파일입니다.

Windows Vista의 모양과 `CFileDialog` 기능은 이전 버전의 Windows버전과 다릅니다. 기본값은 `CFileDialog` 프로그램이 컴파일되고 Windows Vista 이상에서 실행되는 경우 코드 변경 없이 새 Windows Vista 또는 이후 스타일을 자동으로 사용합니다. 생성자의 *bVistaStyle* 매개 변수를 사용하여 이 자동 업데이트를 수동으로 재정의합니다. 자동 업데이트의 예외는 사용자 지정된 대화 상자입니다. 새 스타일로 변환되지 않습니다. 생성자에 대한 자세한 내용은 [CFileDialog::CFileDialog](#cfiledialog)를 참조하십시오.

> [!NOTE]
> 제어 ID 시스템은 을 사용할 `CFileDialog`때 Windows Vista 또는 이전 버전의 Windows에서 다릅니다. 이전 버전의 Windows에서 프로젝트를 포팅하려면 먼저 코드의 `CFileDialog` 컨트롤에 대한 모든 참조를 업데이트해야 합니다.

일부 `CFileDialog` 메서드는 Windows Vista 이상에서 지원되지 않습니다. 메서드가 지원되는지 여부에 대한 자세한 내용은 개별 메서드 항목을 확인합니다. 또한 다음과 같은 상속된 함수는 Windows Vista 또는 이후에서 지원되지 않습니다.

- [CDialog::OnInitDialog](../../mfc/reference/cdialog-class.md#oninitdialog)

- [시디로그::온셋글꼴](../../mfc/reference/cdialog-class.md#onsetfont)

클래스의 `CFileDialog` 창 메시지는 사용 중인 운영 체제에 따라 다릅니다. 예를 들어 Windows XP는 [CDialog::OnCancel](../../mfc/reference/cdialog-class.md#oncancel) 및 `CFileDialog` [CDialog:OnOK를](../../mfc/reference/cdialog-class.md#onok) 지원하지 않습니다. 그러나 Windows Vista 및 이후 운영 체제에서 이를 지원합니다. 생성되는 다양한 메시지와 수신되는 순서에 대한 자세한 내용은 [CFileDialog 샘플: 이벤트 순서 로깅](../../overview/visual-cpp-samples.md)을 참조하십시오.

개체를 `CFileDialog` 사용하려면 먼저 생성기를 사용하여 `CFileDialog` 개체를 만듭니다. 대화 상자를 생성한 후 [CFileDialog:m_ofn](#m_ofn) 구조의 값을 설정하거나 수정하여 대화 상자 컨트롤의 값이나 상태를 초기화할 수 있습니다. 구조는 `m_ofn` 형식입니다. `OPENFILENAME` 자세한 내용은 Windows SDK의 [OPENFILENAME](/windows/win32/api/commdlg/ns-commdlg-openfilenamew) 구조를 참조하십시오.

대화 상자 컨트롤을 초기화한 후 [CFileDialog::DoModal](#domodal) 메서드를 호출하여 사용자가 경로 및 파일 이름을 입력할 수 있도록 대화 상자를 표시합니다. `DoModal`사용자가 확인(IDOK) 또는 취소(IDCANCEL) 단추를 클릭했는지 여부를 반환합니다. IDOK를 `DoModal` `CFileDialog` 반환하는 경우 공용 멤버 기능 중 하나를 사용하여 사용자가 넣은 정보를 검색할 수 있습니다.

> [!NOTE]
> Windows Vista 또는 이후에서 [IFileDialog::SetFileType에](/windows/win32/api/shobjidl_core/nf-shobjidl_core-ifiledialog-setfiletypes) 대한 여러 호출로 인해 오류가 발생합니다. windows Vista `SetFileTypes` 또는 나중에 E_UNEXPECTED `CFileDialog` 반환 하는 모든 인스턴스에 대 한 두 번째 호출입니다. 일부 `CFileDialog` 메서드 함수는 을 호출합니다. `SetFileTypes` 예를 들어, `CFileDialog::DoModal` `CFileDialog` 생성 [ASSERT의](diagnostic-services.md#assert)동일한 인스턴스에 대 한 두 개의 호출.

`CFileDialog`에는 공유 위반, 파일 이름 유효성 검사 및 목록 확인 변경 알림을 사용자 지정처리할 수 있는 여러 보호된 멤버가 포함됩니다. 이러한 보호된 멤버는 기본 처리가 자동으로 수행되기 때문에 대부분의 응용 프로그램에서 사용할 필요가 없는 콜백 함수입니다. 이러한 함수에 대한 메시지 맵 항목은 표준 가상 함수이므로 필요하지 않습니다.

Windows [CommDlgExtendedError](/windows/win32/api/commdlg/nf-commdlg-commdlgextendederror) 함수를 사용하여 대화 상자 초기화 중에 오류가 발생했는지 확인하고 오류에 대해 자세히 알아볼 수 있습니다.

개체의 `CFileDialog` 소멸은 자동으로 처리됩니다. [CDialog::EndDialog를](../../mfc/reference/cdialog-class.md#enddialog)호출할 필요가 없습니다.

사용자가 여러 파일을 선택할 수 있도록 하려면 호출하기 전에 OFN_ALLOWMULTISELECT 플래그를 설정합니다. `DoModal` 반환된 여러 파일 이름 목록을 수용하려면 고유한 파일 이름 버퍼를 제공해야 합니다. 을 생성한 `m_ofn.lpstrFile` `CFileDialog`후 을 호출하기 `DoModal`전에 할당한 버퍼에 대한 포인터로 대체하여 이 작업을 수행합니다.

또한 `m_ofn.lpstrFile`을 가리키는 `m_ofn.nMaxFile` 버퍼의 문자 수를 사용하여 설정해야 합니다. 선택할 `n`파일의 최대 수를 설정하면 필수 버퍼 크기가 `n * (_MAX_PATH + 1) + 1`입니다. 버퍼에서 반환되는 첫 번째 항목은 파일이 선택된 폴더에 대한 경로입니다. Windows Vista 또는 이후 스타일 대화 상자의 경우 디렉터리 및 파일 이름 문자열은 null-종료되고 성 파일 이름 다음의 추가 null 문자가 있습니다. 이 형식을 사용하면 탐색기 스타일 대화 상자를 공백이 포함된 긴 파일 이름을 반환할 수 있습니다. 이전 스타일의 대화 상자의 경우 디렉터리 및 파일 이름 문자열은 공백으로 구분되며 함수는 공백이 있는 파일 이름에 대해 짧은 파일 이름을 사용합니다.

## <a name="cfiledialogcfiledialog"></a><a name="cfiledialog"></a>CFile 대화 상자::CFile 대화 상자

이 함수를 호출하여 표준 Windows 파일 대화 상자를 구성합니다.

```
explicit CFileDialog(
    BOOL bOpenFileDialog,
    LPCTSTR lpszDefExt = NULL,
    LPCTSTR lpszFileName = NULL,
    DWORD dwFlags = OFN_HIDEREADONLY | OFN_OVERWRITEPROMPT,
    LPCTSTR lpszFilter = NULL,
    CWnd* pParentWnd = NULL,
    DWORD dwSize = 0,
    BOOL bVistaStyle = TRUE);
```

### <a name="parameters"></a>매개 변수

*bOpenFile 디아로그*<br/>
【인】 만들 대화 상자 유형을 지정하는 매개 변수입니다. **파일 열기** 대화 상자를 구성하려면 TRUE로 설정합니다. **FALSE로** 설정하여 파일 저장 대화 상자를 생성합니다.

【인】 기본 파일 이름 확장명입니다. 사용자가 파일 이름 상자에 알려진 확장(사용자 컴퓨터에 연결이 있는 확장)을 포함하지 않으면 *lpszDefExt가* 지정한 확장명이 파일 이름에 자동으로 추가됩니다. 이 매개 변수가 NULL이면 확장이 추가되지 않습니다.

【인】 파일 이름 상자에 나타나는 초기 파일 이름입니다. NULL이면 초기 파일 이름이 나타나지 않습니다.

*dwFlags*<br/>
【인】 대화 상자를 사용자 지정하는 데 사용할 수 있는 하나 이상의 플래그의 조합입니다. 이러한 플래그에 대한 설명은 Windows SDK의 [OPENFILENAME](/windows/win32/api/commdlg/ns-commdlg-openfilenamew) 구조를 참조하십시오. 구조체 멤버를 `m_ofn.Flags` 수정하는 경우 변경 내용에서 비트OR 연산자(bitwise-OR 연산자)를 사용하여 기본 동작을 그대로 유지합니다.

*lpsz필터*<br/>
【인】 파일에 적용할 수 있는 필터를 지정하는 일련의 문자열 쌍입니다. 파일 필터를 지정하면 필터 조건과 일치하는 파일만 파일 목록에 나타납니다. 파일 필터를 사용하여 작업하는 방법에 대한 자세한 내용은 비고 섹션을 참조하십시오.

*pParentWnd*<br/>
【인】 파일 대화 상자의 부모 또는 소유자 창에 대한 포인터입니다.

*dwSize*<br/>
【인】 구조의 `OPENFILENAME` 크기입니다. 이 값은 운영 체제 버전에 따라 다릅니다. MFC는 이 매개 변수를 사용하여 만들 적절한 종류의 대화 상자를 결정했습니다. 기본 크기가 0이면 MFC 코드가 프로그램이 실행되는 운영 체제 버전에 따라 사용할 올바른 대화 상자 크기를 결정합니다.

*b비스타스타일*<br/>
【인】 **참고 사항** 이 매개 변수는 Visual Studio 2008 이상에서 사용할 수 있으며 Windows Vista 이상에서 실행 중인 경우에만 새 스타일의 대화 상자를 사용할 수 있습니다.

파일 대화 상자의 스타일을 지정하는 매개 변수입니다. 새 Vista 스타일 파일 대화 상자를 사용하려면 TRUE로 설정합니다. 그렇지 않으면 이전 스타일의 대화 상자가 사용됩니다. Vista에서 실행에 대한 자세한 내용은 비고 섹션을 참조하십시오.

### <a name="remarks"></a>설명

파일 **열기** 또는 **파일 저장** 대화 상자는 *bOpenFileDialog*의 값에 따라 생성됩니다.

## <a name="cfiledialogm_ofn"></a><a name="m_ofn"></a>CFile Dialog::m_ofn

`m_ofn`은 형식의 `OPENFILENAME`구조입니다. 이 구조의 데이터는 `CFileDialog`의 현재 상태를 나타냅니다.

### <a name="remarks"></a>설명

이 구조를 사용하여 **파일 열기** 또는 파일 저장 대화 상자의 모양을 생성한 후 [DoModal](#domodal) **메서드로** 표시하기 전에 초기화합니다. 예를 들어 대화 상자에 사용할 `m_ofn` *캡션의 lpstrTitle* 멤버를 설정할 수 있습니다.

Windows Vista 또는 이후 스타일인 [CFileDialog에서는](../../mfc/reference/cfiledialog-class.md) `m_ofn` 대화 상자의 상태를 항상 일치시키지 않습니다. 이전 버전의 Windows의 대화 상자와 동기화됩니다. Windows Vista 에서 `m_ofn` 구조 및 `CFileDialog` 상태를 동기화하는 데 대한 자세한 내용은 [CFileDialog:ApplyOFNToShellDialog](#applyofntoshelldialog) 및 [CFileDialog::UpdateOFNFromShellDialog를](#updateofnfromshelldialog) 참조하십시오.

Windows Vista 또는 이후 스타일 파일 대화 상자는 의 `CFileDialog`특정 멤버 및 플래그를 지원하지 않습니다. 따라서 이러한 효과는 영향을 주지 않습니다.

다음은 Windows Vista 또는 이후에서 지원되지 않는 멤버 목록입니다.

- `lpstrCustomFilter`

- `lpstrInitialDir`

- `lCustData`

- `lpfnHook`

- `lpTemplateName`

다음 플래그는 지원되지 않으므로 Windows Vista 또는 이후 스타일의 다음 `CFileDialog`스타일을 사용할 때는 영향을 주지 않습니다.

- OFN_ENABLEHOOK

- OFN_ENABLEINCLUDENOTIFY

- OFN_ENABLETEMPLATE

- OFN_ENABLETEMPLATEHANDLE

- OFN_EXPLORER

- OFN_EXTENSIONDIFFERENT

- OFN_HIDEREADONLY

- OFN_LONGNAMES - 효과적으로 항상 윈도우 비스타 이상에

- OFN_NOLONGNAMES - 효과적으로 항상 윈도우 비스타 이상에서 꺼져

- OFN_NONETWORKBUTTON - 효과적으로 항상 윈도우 비스타 이상에

- OFN_READONLY

- OFN_SHOWHELP

이 구조에 대한 자세한 내용은 Windows SDK의 [OPENFILENAME](/windows/win32/api/commdlg/ns-commdlg-openfilenamew) 구조를 참조하십시오.

```

```

### <a name="return-value"></a>Return Value

1 파일 이름이 유효한 파일 이름이 아닌 경우; 그렇지 않으면 0.

### <a name="remarks"></a>설명

이 기능을 사용하면 응용 프로그램별 이유로 파일 이름을 거부할 수 있습니다. 일반적으로 프레임워크는 파일 이름에 대한 기본 유효성 검사를 제공하고 잘못된 파일 이름을 입력하면 메시지 상자를 표시하므로 이 함수를 사용할 필요가 없습니다.

1이 반환되면 사용자가 다른 파일 이름을 입력할 수 있도록 대화 상자가 계속 표시됩니다. 대화 상자 절차는 반환이 0인 경우 대화 상자를 해제합니다. 다른 비영점 반환 값은 현재 예약되어 있으며 사용할 수 없습니다.

## <a name="cfiledialogupdateofnfromshelldialog"></a><a name="updateofnfromshelldialog"></a>CFile Dialog::업데이트OFNFrom셸디아로그

내부 `m_ofn` 개체의 현재 상태를 기반으로 [CFileDialog의](../../mfc/reference/cfiledialog-class.md) 데이터 구조를 업데이트합니다.

```cpp
void UpdateOFNFromShellDialog();
```

### <a name="remarks"></a>설명

Windows Vista 이전의 Windows 버전에서는 [멤버 OPENFILENAME](/windows/win32/api/commdlg/ns-commdlg-openfilenamea) 데이터 구조가 `CFileDialog`의 상태와 지속적으로 동기화되었습니다. [m_ofn](#m_ofn) 멤버 변수에 대한 모든 변경 사항은 대화 상자의 상태에 직접적인 영향을 미칩니다. 또한 대화 상자의 상태가 변경되면 m_ofn 멤버 변수가 즉시 업데이트되었습니다.

Windows Vista 이상에서는 `m_ofn` 데이터 구조가 자동으로 업데이트되지 않습니다. 멤버 변수의 데이터의 정확성을 `m_ofn` 보장하려면 데이터에 액세스하기 전에 `UpdateOFNFromShellDialog` 함수를 호출해야 합니다. Windows는 [IFileDialog::OnFileOK](/windows/win32/api/shobjidl_core/nf-shobjidl_core-ifiledialogevents-onfileok)를 처리하는 동안 이 함수를 자동으로 호출합니다.

Windows Vista 에서 또는 `CFileDialog` 나중에 클래스를 사용하는 방법에 대한 자세한 내용은 [CFileDialog 클래스를](../../mfc/reference/cfiledialog-class.md)참조하십시오.
