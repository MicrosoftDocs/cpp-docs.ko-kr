# <a name="cmfctoolbar-class"></a>CMFC툴바 클래스

클래스는 `CMFCToolBar` [CToolBar 클래스와](../../mfc/reference/ctoolbar-class.md)유사하지만 사용자 인터페이스 기능에 대한 추가 지원을 제공합니다. 여기에는 평면 도구 모음, 핫 이미지 포함 도구 모음, 큰 아이콘, 페이저 단추, 잠긴 도구 모음, rebar 컨트롤, 이미지 아래 텍스트, 배경 이미지, 탭 도구 모음이 포함됩니다. `CMFCToolBar` 클래스에는 기본 제공 사용자 지정 도구 모음과 메뉴, 메뉴 사이 끌어 놓기, 콤보 상자 단추, 편집 상자 단추, 색 선택 및 롤업 단추 도구 모음이 포함됩니다.

자세한 내용은 Visual Studio 설치의 **\\VC\\atlmfc\\src mfc** 폴더에 있는 소스 코드를 참조하십시오.

## <a name="syntax"></a>구문

```
class CMFCToolBar : public CMFCBaseToolBar
```

## <a name="remarks"></a>설명

개체를 `CMFCToolBar` 응용 프로그램에 통합하려면 다음 단계를 따르십시오.

1. 기본 `CMFCToolBar` 프레임 창에 개체를 추가합니다.

1. 주 프레임 창에 대한 WM_CREATE 메시지를 처리할 때 [CMFCToolBar::만들기](#create) 또는 [CMFCToolBar::CreateEx를](#createex) 호출하여 도구 모음을 만들고 스타일을 지정합니다.

1. [CBasePane::EnableDocking을](../../mfc/reference/cbasepane-class.md#enabledocking) 호출하여 도킹 스타일을 지정합니다.

콤보 상자 또는 드롭다운 도구 모음과 같은 특수 단추를 삽입하려면 상위 리소스에 더미 단추를 예약하고 [CMFCToolBar::ReplaceButton을](#replacebutton)사용하여 런타임에 더미 단추를 교체합니다. 자세한 내용은 [연습: 도구 모음에 컨트롤 넣기](../walkthrough-putting-controls-on-toolbars.md)를 참조하십시오.

`CMFCToolBar`는 MFC 라이브러리 클래스 [CMFCMenuBar 클래스,](../../mfc/reference/cmfcmenubar-class.md) [CMFCPopupMenuBar 클래스](../../mfc/reference/cmfcpopupmenubar-class.md)및 [CMFCDropDownToolBar 클래스의 기본 클래스입니다.](../../mfc/reference/cmfcdropdowntoolbar-class.md)

## <a name="cmfctoolbaraddbasiccommand"></a><a name="addbasiccommand"></a>CMFC툴바::추가 베이직 명령

사용자가 메뉴를 열 때 항상 표시되는 명령 목록에 메뉴 명령을 추가합니다.

```
static void __stdcall AddBasicCommand(UINT uiCmd);
```

### <a name="parameters"></a>매개 변수

*uiCmd*<br/>
【인】 추가할 명령을 지정합니다.

### <a name="remarks"></a>설명

메뉴를 열 때 기본 명령이 항상 표시됩니다. 이 메서드는 사용자가 최근에 사용한 명령을 보도록 선택할 때 의미가 있습니다.

[CMFCToolBar::SetBasicCommands 메서드를](#setbasiccommands) 사용하여 사용자가 메뉴를 열 때 항상 표시되는 명령 목록을 설정합니다. [CMFCToolBar::GetBasicCommands 메서드를](#getbasiccommands) 사용하여 응용 프로그램에서 사용하는 기본 명령 목록을 검색합니다.

## <a name="cmfctoolbaraddcommandusage"></a><a name="addcommandusage"></a>CMFC툴바::추가 명령사용법

지정된 명령과 연결된 카운터를 하나씩 증분합니다.

```
static void __stdcall AddCommandUsage(UINT uiCommand);
```

### <a name="parameters"></a>매개 변수

*uiCommand*<br/>
【인】 명령 카운터가 증분하도록 지정합니다.

### <a name="remarks"></a>설명

사용자가 메뉴 항목을 선택할 때 프레임워크는 이 메서드를 호출합니다.

프레임워크는 명령 카운터를 사용하여 최근에 사용한 메뉴 항목을 표시합니다.

이 메서드는 [CMFCCmdUsageCount::AddCmd](../../mfc/reference/cmfccmdusagecount-class.md#addcmd) 메서드를 사용 하 여 명령 카운터를 증가 합니다.

## <a name="cmfctoolbaraddtoolbarforimagecollection"></a><a name="addtoolbarforimagecollection"></a>CMFC툴바::추가도구바포이미지컬렉션

사용자 인터페이스 리소스의 이미지를 응용 프로그램의 이미지 컬렉션에 추가합니다.

```
static BOOL __stdcall AddToolBarForImageCollection(
    UINT uiResID,
    UINT uiBmpResID=0,
    UINT uiColdResID=0,
    UINT uiMenuResID=0,
    UINT uiDisabledResID=0,
    UINT uiMenuDisabledResID=0);
```

### <a name="parameters"></a>매개 변수

*uiResID*<br/>
【인】 로드할 이미지가 있는 도구 모음의 리소스 ID입니다.

*uiBmpResID*<br/>
【인】 도구 모음 이미지가 있는 비트맵의 리소스 ID입니다.

*uiColdResID*<br/>
【인】 "콜드" 도구 모음 이미지가 있는 비트맵의 리소스 ID입니다.

*uiMenuResID*<br/>
【인】 메뉴 이미지가 있는 비트맵의 리소스 ID입니다.

*ui장애인ResID*<br/>
【인】 비활성화된 도구 모음 이미지가 있는 비트맵의 리소스 ID입니다.

*uiMenu장애인ResID*<br/>
【인】 비활성화 된 메뉴 이미지와 비트 맵의 리소스 ID입니다.

### <a name="return-value"></a>Return Value

메서드가 성공하면 TRUE입니다. *uiResID* 또는 *uiBmpResID* 가 유효한 리소스를 지정하지 않거나 다른 오류가 발생하는 경우 FALSE입니다.

### <a name="remarks"></a>설명

이 메서드를 호출하여 도구 모음 이미지가 있는 비트맵을 로드하고 도구 모음 이미지 컬렉션에 추가합니다. 이 메서드는 임시 도구 모음 개체를 만들고 [CMFCToolBar::LoadToolBar](#loadtoolbar)를 호출합니다.

## <a name="cmfctoolbaradjustlayout"></a><a name="adjustlayout"></a>CMFC툴바::레이아웃 조정

도구 모음의 크기와 위치를 다시 계산합니다.

```
virtual void AdjustLayout();
```

### <a name="remarks"></a>설명

도구 모음이 만들어진 경우 이 메서드를 호출하여 크기와 위치를 다시 계산합니다.

프레임워크는 도구 모음의 레이아웃을 변경해야 할 때마다 이 메서드를 호출합니다. 예를 들어 사용자가 다른 컨트롤 모음을 이동하거나 응용 프로그램 창크기를 조정하거나 도구 모음을 사용자 지정할 때 레이아웃이 변경되어야 합니다.

이 메서드를 재정의하여 에서 파생된 클래스에서 `CMFCToolbar`고유한 동적 레이아웃을 제공합니다.

## <a name="cmfctoolbaradjustsize"></a><a name="adjustsize"></a>CMFC툴바::조정크기

도구 모음의 크기를 다시 계산합니다.

```
void AdjustSize();
```

### <a name="remarks"></a>설명

이 방법을 사용하면 도구 모음이 상위 프레임의 경계에 맞는지 확인합니다. 이 메서드는 도구 모음에 상위 프레임이 없는 경우 아무 작업을 수행하지 않습니다.

[CMFCToolBar::adjustLayout](#adjustlayout) 메서드는 도구 모음의 부모가 개체가 아닌 경우 크기를 `CMFCReBar` 다시 계산하기 위해 이 메서드를 호출합니다.

## <a name="cmfctoolbarallowchangetextlabels"></a><a name="allowchangetextlabels"></a>CMFCToolBar::허용변경텍스트레이블

도구 모음 단추의 이미지 아래에 텍스트 레이블을 표시할 수 있는지 여부를 지정합니다.

```
virtual BOOL AllowChangeTextLabels() const;
```

### <a name="return-value"></a>Return Value

TRUE 이미지 아래에 텍스트 레이블을 표시할 수 있는 경우 그렇지 않으면 거짓.

### <a name="remarks"></a>설명

이 메서드는 사용자 지정 대화 상자에서 호출하여 선택한 도구 모음에 대한 **도구 모음** 페이지에서 **텍스트 레이블 표시** 확인란을 사용하도록 설정할지 여부를 결정합니다.

기본 구현은 TRUE를 반환합니다.

## <a name="cmfctoolbarenablecustomizebutton"></a><a name="enablecustomizebutton"></a>CMFCToolBar::사용자 지정 버튼 사용

도구 모음에 나타나는 사용자 지정 단추를 활성화하거나 사용하지 않도록 설정합니다.

```
void EnableCustomizeButton(
    BOOL bEnable,
    int iCustomizeCmd,
    const CString& strCustomizeText,
    BOOL bQuickCustomize=TRUE);

void EnableCustomizeButton(
    BOOL bEnable,
    int iCustomizeCmd,
    UINT uiCustomizeTextResId,
    BOOL bQuickCustomize=TRUE);
```

### <a name="parameters"></a>매개 변수

*bEnable*<br/>
【인】 사용자 지정 단추를 활성화하거나 사용하지 않도록 설정합니다.

*i사용자 정의Cmd*<br/>
【인】 사용자 지정 단추의 명령 ID입니다.

*str사용자 정의 텍스트*<br/>
【인】 사용자 지정 단추의 텍스트 레이블입니다.

*uiizeTextResId*<br/>
【인】 사용자 지정 단추 레이블의 리소스 문자열 ID입니다.

*b빠른 사용자 정의*<br/>
【인】 단추에서 아래로 떨어지는 메뉴에서 단추 추가 또는 제거 옵션을 **활성화하거나** 사용하지 않도록 설정합니다.

### <a name="remarks"></a>설명

*iCustomizeCmd가* -1이면 여러 도구 모음 단추가 도구 모음 영역에 맞지 않을 때 프레임워크에 사용자 지정 버튼이 표시됩니다. 단추에는 더 많은 단추가 있음을 나타내는 이중 왼쪽 을 가리키는 화살표 또는 갈매기가 표시됩니다.

*iCustomizeCmd가* 유효한 명령 ID를 지정하고 *bEnable가* TRUE인 경우 사용자 지정 버튼이 항상 표시됩니다. 단추에는 작은 아래쪽 화살표가 있으며 명령이 포함된 메뉴가 열립니다. 이 명령은 *strCustomizeText*에 의해 지정된 텍스트 레이블을 사용합니다. *bQuickCustomize도* TRUE이면 메뉴에 **버튼 추가 또는 제거** 옵션이 표시됩니다.

프레임워크는 *iCustomizeCmd에*의해 지정된 항목 앞에 도구 모음 영역에 맞지 않는 단추를 메뉴에 동적으로 추가합니다. 갈매기가 아래쪽 화살표 옆에 표시됩니다.

모든 도구 모음에서 단추 및 이미지의 크기를 지정합니다.

```
static void __stdcall SetSizes(
    SIZE sizeButton,
    SIZE sizeImage);
```

### <a name="parameters"></a>매개 변수

*크기 버튼*<br/>
【인】 도구 모음 단추의 크기(픽셀)입니다.

*크기이미지*<br/>
【인】 도구 모음 단추 이미지의 크기(픽셀)입니다.

### <a name="remarks"></a>설명

도구 모음 단추의 기본 크기는 23x22픽셀입니다. 도구 모음 단추 이미지의 기본 크기는 16x15 픽셀입니다.

[CMFCToolBar::GetImageSize](#getimagesize) 메서드를 호출하여 도구 모음 단추 이미지의 크기를 검색합니다. [CMFCToolBar::GetButtonSize](#getbuttonsize) 메서드를 호출하여 도구 모음 단추의 크기를 검색합니다.

## <a name="cmfctoolbarsettoolbarbtntext"></a><a name="settoolbarbtntext"></a>CMFC툴바::셋툴바브엔텍스트

도구 모음에 있는 단추의 속성을 지정합니다.

```
void SetToolBarBtnText(
    UINT nBtnIndex,
    LPCTSTR szText=NULL,
    BOOL bShowText=TRUE,
    BOOL bShowImage=TRUE);
```

### <a name="parameters"></a>매개 변수

*nBtn인덱스*<br/>
【인】 도구 모음 단추 목록의 도구 모음 단추의 0기반 인덱스입니다.

*szText*<br/>
【인】 도구 모음 단추의 텍스트 레이블을 지정합니다.

*b쇼텍스트*<br/>
【인】 이 매개변수가 TRUE이면 프레임워크에 텍스트 레이블이 표시됩니다. 그렇지 않으면 프레임워크에서 텍스트 레이블을 숨깁니다.

*b쇼이미지*<br/>
【인】 이 매개 변수가 TRUE이면 프레임워크에 도구 모음 단추 이미지가 표시됩니다. 그렇지 않으면 프레임워크는 도구 모음 단추 이미지를 숨깁니다.

### <a name="remarks"></a>설명

기본적으로 프레임워크는 도구 모음 단추의 이미지를 보여 주지만 도구 모음 단추의 텍스트 레이블은 표시되지 않습니다.

디버그 빌드에서 *nBtnIndex가* 유효한 도구 모음 단추를 참조하지 않거나 도구 모음 단추가 구분기호인 경우 이 메서드는 어설션 오류를 생성합니다.

## <a name="cmfctoolbarsettworowswithsibling"></a><a name="settworowswithsibling"></a>CMFC툴바::셋투로우와 형제

도구 모음과 해당 형제를 별도의 행에 배치합니다.

```
void SetTwoRowsWithSibling();
```

### <a name="remarks"></a>설명

프레임워크는 사용자가 두 행의 단추 표시 단추를 클릭할 때 이 **메서드를 호출합니다.**

[CMFCToolBar::SetSiblingToolBar](#setsiblingtoolbar) 메서드를 호출하여 **한 행의 단추 표시** 또는 두 행 **단추의 단추 표시를** 활성화합니다. 이 도구 모음에 대해 [CMFCToolBar::SetSiblingToolBar를](#setsiblingtoolbar) 호출하면 형제 도구 모음이 별도의 행으로 이동됩니다. 그렇지 않으면 이 도구 모음이 별도의 행으로 이동됩니다.

프레임워크는 사용자가 한 행의 단추 표시 단추를 클릭할 때 [CMFCToolBar::SetOneRowWithSibling](#setonerowwithsibling) **메서드를 호출합니다.**

## <a name="cmfctoolbarsetuserimages"></a><a name="setuserimages"></a>CMFC툴바::셋유저 이미지

응용 프로그램에서 사용자 정의 이미지의 컬렉션을 설정합니다.

```
static BOOL SetUserImages(CMFCToolBarImages* pUserImages);
```

### <a name="parameters"></a>매개 변수

*pUser 이미지*<br/>
【인】 사용자 정의 이미지 컬렉션에 대한 포인터입니다.

### <a name="return-value"></a>Return Value

메서드가 성공하면 0이 아닙니다. 그렇지 않으면 지정된 `CMFCToolBarImages` 개체가 유효하지 않거나 도구 모음의 기본 이미지 크기와 다른 이미지 크기가 있는 경우 0입니다.

### <a name="remarks"></a>설명

프레임워크는 사용자 정의 이미지를 사용하여 사용자가 사용자 지정한 도구 모음 단추를 그립니다. *pUserImages에서* 지정한 이미지 목록은 응용 프로그램의 모든 도구 모음 간에 공유됩니다.

이 메서드는 지정된 `CMFCToolBarImages` 개체가 유효하지 않거나 도구 모음의 기본 이미지 크기와 다른 이미지 크기가 있는 경우 디버그 빌드에서 어설션 오류를 생성합니다.

OutlookDemo, ToolTipDemo 및 VisualStudioDemo 샘플은 이 메서드를 사용하여 사용자 정의 이미지의 전역 컬렉션을 설정합니다. 응용 프로그램의 작업 디렉토리에 있는 UserImages.bmp라는 파일을 로드합니다.

[CMFCToolBar::GetUserImages](#getuserimages) 메서드를 호출하여 응용 프로그램에서 사용자 정의 이미지 컬렉션을 검색합니다.

## <a name="cmfctoolbarstretchpane"></a><a name="stretchpane"></a>CMFC툴바::스트레치파인

도구 모음을 세로 또는 수평으로 늘이고 필요한 경우 단추의 위치를 조정합니다.

```
virtual CSize StretchPane(
    int nLength,
    BOOL bVert);
```

### <a name="parameters"></a>매개 변수

【인】 창을 늘릴 픽셀 단위로 양입니다.

*bVert*<br/>
【인】 TRUE이면 창을 세로로 늘입니다. FALSE인 경우 창을 가로로 늘입니다.

### <a name="return-value"></a>Return Value

도구 `CSize` 모음 클라이언트 영역의 크기를 지정하는 개체입니다.

### <a name="remarks"></a>설명

이 메서드는 [CMFCToolBar::WrapToolBar를](#wraptoolbar) 호출하여 확장된 도구 모음 내의 단추를 재배치합니다.

반환 값은 [CMFCToolBar::CalcSize를](#calcsize)호출하여 결정됩니다.

## <a name="cmfctoolbarwraptoolbar"></a><a name="wraptoolbar"></a>CMFC툴바::랩툴바

지정된 치수 내에서 도구 모음 단추의 위치를 지정합니다.

```
int WrapToolBar(
    int nWidth,
    int nHeight = 32767,
    CDC* pDC = NULL,
    int nColumnWidth = -1,
    int nRowHeight = -1);
```

### <a name="parameters"></a>매개 변수

*nWidth*<br/>
【인】 도구 모음의 최대 너비입니다.

*nHeight*<br/>
【인】 도구 모음의 최대 높이입니다. 도구 모음이 부동 인 경우 사용되지 않습니다.

### <a name="remarks"></a>설명
