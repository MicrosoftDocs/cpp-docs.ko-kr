# <a name="styles-used-by-mfc"></a>MFC에서 사용하는 스타일

해당 MFC 개체를 만들 때 창 또는 컨트롤 모양 및 동작을 지정 하려면 다음 스타일 플래그를 사용 합니다. 대부분의 경우 이러한 스타일은 클래스 `Create` 함수의 *dwstyle* 매개 변수에 설정 됩니다.

## <a name="button-styles"></a>단추 스타일

단추 스타일은 라디오 단추, 확인란 및 누름 단추 같은 [Cbutton 클래스](../../mfc/reference/cbutton-class.md) 개체에 적용 됩니다. [Cbutton:: Create](../../mfc/reference/cbutton-class.md#create)의 *dwstyle* 매개 변수에 스타일의 조합을 지정 합니다. Windows의 단추 스타일에 대 한 자세한 내용은 [단추 스타일 (Windows)](/windows/win32/Controls/button-styles)을 참조 하세요.

### <a name="button-types"></a>단추 유형

다음 표에서는 단추 유형을 나열 합니다. 필요에 따라 다음 중 하나를 선택할 수 있습니다. 단추 유형을 지정 하지 않으면 기본값은 BS_PUSHBUTTON입니다.

|형식|설명|
|----------|-----------------|
|BS_3STATE|세 가지 상태를 사용 하 여 확인란 단추를 만듭니다. BST_CHECKED, BST_INDETERMINATE 및 BST_UNCHECKED입니다. 단추를 클릭 하면 소유자 창에 BN_CLICKED 알림이 전송 되지만 단추의 상태는 변경 되지 않습니다. 기본적으로 연결 된 텍스트는 확인란의 오른쪽에 표시 됩니다. 확인란 왼쪽에 텍스트를 표시 하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용 합니다.|
|BS_AUTO3STATE|세 가지 상태를 사용 하 여 확인란 단추를 만듭니다. BST_CHECKED, BST_INDETERMINATE 및 BST_UNCHECKED입니다. 단추를 클릭 하 여 소유자 창에 BN_CLICKED 알림을 보내고 단추의 상태를 변경 합니다. 단추의 상태는 BST_CHECKED, BST_INDETERMINATE 및 BST_UNCHECKED 순서 대로 순환 됩니다. 기본적으로 연결 된 텍스트는 확인란의 오른쪽에 표시 됩니다. 확인란 왼쪽에 텍스트를 표시 하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용 합니다.|
|BS_AUTOCHECKBOX|두 가지 상태를 사용 하 여 확인란 단추를 만듭니다. BST_CHECKED 및 BST_UNCHECKED. 단추를 클릭 하 여 소유자 창에 BN_CLICKED 알림을 보내고 단추의 상태를 변경 합니다. 기본적으로 연결 된 텍스트는 확인란의 오른쪽에 표시 됩니다. 확인란 왼쪽에 텍스트를 표시 하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용 합니다.|
|BS_AUTORADIOBUTTON|두 가지 상태를 사용 하 여 라디오 단추를 만듭니다. BST_CHECKED 및 BST_UNCHECKED. 라디오 단추는 일반적으로 그룹에서 사용 되며, 각 그룹에는 한 번에 하나의 선택 된 옵션만 있습니다. 단추를 클릭 하 여 소유자 창에 BN_CLICKED 알림을 보내고, 클릭 한 라디오 단추의 상태를 BST_CHECKED로 설정 하 고, 단추 그룹에 있는 다른 모든 라디오 단추의 상태를 BST_UNCHECKED로 설정 합니다. 기본적으로 연결 된 텍스트는 라디오 단추 오른쪽에 표시 됩니다. 라디오 단추의 왼쪽에 텍스트를 표시 하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용 합니다.|
|BS_CHECKBOX|두 가지 상태를 사용 하 여 확인란 단추를 만듭니다. BST_CHECKED 및 BST_UNCHECKED. 단추를 클릭 하면 소유자 창에 BN_CLICKED 알림이 전송 되지만 단추의 상태는 변경 되지 않습니다. 기본적으로 연결 된 텍스트는 확인란의 오른쪽에 표시 됩니다. 확인란 왼쪽에 텍스트를 표시 하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용 합니다.|
|BS_COMMANDLINK|명령 링크 단추를 만듭니다. 명령 링크 단추는 주 텍스트의 왼쪽에 녹색 화살표를 표시 하 고 주 텍스트 아래에 메모를 표시 하는 Windows Vista에만 해당 하는 명령 단추입니다. [Cbutton:: SetNote](../../mfc/reference/cbutton-class.md#setnote)를 사용 하 여 메모 텍스트를 설정할 수 있습니다.|
|BS_DEFCOMMANDLINK|명령 링크 단추를 만듭니다. 명령 링크 단추는 주 텍스트의 왼쪽에 녹색 화살표를 표시 하 고 주 텍스트 아래에 메모를 표시 하는 Windows Vista에만 해당 하는 명령 단추입니다. [Cbutton:: SetNote](../../mfc/reference/cbutton-class.md#setnote)를 사용 하 여 메모 텍스트를 설정할 수 있습니다. 단추가 대화 상자에 있는 경우 ENTER 키를 누르면 단추에 입력 포커스가 없어도 대화 상자에 BN_CLICKED 알림이 전송 됩니다.|
|BS_DEFPUSHBUTTON|굵은 검정 테두리가 있는 명령 단추를 만듭니다. 단추가 대화 상자에 있는 경우 ENTER 키를 누르면 단추에 입력 포커스가 없어도 대화 상자에 BN_CLICKED 알림이 전송 됩니다.|
|BS_DEFSPLITBUTTON|분할 단추를 만듭니다. 분할 단추는 드롭다운 화살표 옆의 단추를 포함 하는 Windows Vista와 관련 된 명령 단추입니다. 단추를 클릭 하면 기본 명령이 실행 됩니다. 드롭다운 화살표를 클릭 하면 추가 명령 메뉴가 나타납니다. 분할 단추가 대화 상자에 있는 경우 ENTER 키를 누르면 단추에 입력 포커스가 없어도 대화 상자에 BN_CLICKED 알림이 전송 됩니다.|
|BS_GROUPBOX|다른 단추를 그룹화 할 수 있는 사각형을 만듭니다. 이 스타일과 연결 된 텍스트는 사각형의 왼쪽 위 모퉁이에 표시 됩니다.|
|BS_OWNERDRAW|소유자가 그린 단추를 만듭니다. 프레임 워크는 단추의 `DrawItem` 시각적 측면이 변경 될 때 메서드를 호출 합니다. 이 스타일은 클래스를 `CBitmapButton` 사용할 때 설정 해야 합니다.|
|BS_PUSHBUTTON|사용자가 단추를 클릭할 때 소유자 창에 BN_CLICKED 알림을 보내는 명령 단추를 만듭니다.|
|BS_RADIOBUTTON|두 가지 상태를 사용 하 여 라디오 단추를 만듭니다. BST_CHECKED 및 BST_UNCHECKED. 라디오 단추는 일반적으로 그룹에서 사용 되며, 각 그룹에는 한 번에 하나의 선택 된 옵션만 있습니다. 단추를 클릭 하면 소유자 창에 BN_CLICKED 알림이 전송 되지만 그룹에 있는 단추의 상태는 자동으로 변경 되지 않습니다. 기본적으로 연결 된 텍스트는 라디오 단추 오른쪽에 표시 됩니다. 라디오 단추의 왼쪽에 텍스트를 표시 하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용 합니다.|
|BS_SPLITBUTTON|분할 단추를 만듭니다. 분할 단추는 드롭다운 화살표 옆의 단추를 포함 하는 Windows Vista와 관련 된 명령 단추입니다. 단추를 클릭 하면 기본 명령이 실행 됩니다. 드롭다운 화살표를 클릭 하면 추가 명령 메뉴가 나타납니다.|
|BS_USERBUTTON|사용 되지 않지만 Windows의 16 비트 버전과의 호환성을 위해 제공 됩니다. Win32 기반 응용 프로그램은 BS_OWNERDRAW를 대신 사용 해야 합니다.|

### <a name="radio-button-and-check-box-styles"></a>라디오 단추 및 확인란 스타일

다음 표에서는 라디오 단추 및 확인란에 적용 되는 스타일을 보여 줍니다. 이러한 스타일은 다른 모든 단추 형식에서 무시 됩니다. 필요에 따라 다음 중 하나 이상을 선택할 수 있습니다.

|스타일|설명|
|-----------|-----------------|
|BS_LEFTTEXT|라디오 단추 또는 확인란 스타일과 함께 사용 하면 텍스트가 라디오 단추 또는 확인란의 왼쪽에 표시 됩니다.|
|BS_RIGHTBUTTON|라디오 단추 또는 확인란 스타일과 함께 사용 하면 텍스트가 라디오 단추 또는 확인란의 왼쪽에 표시 됩니다. 이 스타일은 BS_LEFTTEXT 스타일과 동일 합니다.|
|BS_PUSHLIKE|명령 단추 처럼 확인란 또는 라디오 단추를 표시 하 고 동작을 수행 합니다. 상태가 BST_CHECKED로 표시 되 고, 상태가 BST_INDETERMINATE 인 경우 흐리게 표시 되 고, 상태가 BST_UNCHECKED 일 때 해제 되 면 단추가 누름 상태로 표시 됩니다.|

### <a name="button-text-alignment-styles"></a>단추 텍스트 맞춤 스타일

다음 표에서는 가로 및 세로 텍스트 맞춤 옵션을 보여 줍니다. 필요에 따라 다음 중 하나를 선택할 수 있습니다.

|스타일|설명|
|-----------|-----------------|
|BS_LEFT|단추 사각형의 텍스트를 왼쪽에 맞춥니다. 그러나 단추가 BS_RIGHTBUTTON 스타일을 포함 하지 않는 확인란 또는 라디오 단추인 경우 텍스트는 확인란 또는 라디오 단추 오른쪽에 왼쪽 맞춤 됩니다.|

|BS_BOTTOM|단추 사각형의 아래쪽에 텍스트를 배치 합니다.|

## <a name="combo-box-styles"></a>콤보 상자 스타일

다음 콤보 상자 스타일을 MFC에서 사용할 수 있습니다. Windows의 콤보 상자 스타일에 대 한 자세한 내용은 [콤보 상자 스타일 (Windows)](/windows/win32/Controls/combo-box-styles)을 참조 하세요.

|스타일|설명|
|-----------|-----------------|
|CBS_AUTOHSCROLL|사용자가 줄의 끝에 문자를 입력 하면 편집 컨트롤의 텍스트가 자동으로 오른쪽으로 스크롤됩니다. 이 스타일을 설정하지 않으면 사각형 경계에 포함되는 텍스트만 허용됩니다.|
|CBS_DISABLENOSCROLL|목록 상자에 스크롤할 충분 한 항목이 포함 되지 않은 경우 비활성화 된 세로 스크롤 막대가 표시 됩니다. 이 스타일을 사용하지 않으면 목록 상자에 충분한 항목이 포함되지 않은 경우 스크롤 막대가 숨겨집니다.|
|CBS_DROPDOWN|사용자가 편집 컨트롤 옆의 아이콘을 선택 하지 않은 경우 목록 상자가 표시 되지 않는다는 점을 제외 하 고 CBS_SIMPLE와 유사 합니다.|
|CBS_DROPDOWNLIST|CBS_DROPDOWN와 유사 합니다. 단, 편집 컨트롤은 목록 상자에서 현재 선택 항목을 표시 하는 정적 텍스트 항목으로 바뀝니다.|
|CBS_HASSTRINGS|소유자 그리기 콤보 상자에는 문자열로 구성 된 항목이 있습니다. 콤보 상자는 애플리케이션에서 `GetText` 멤버 함수를 사용하여 특정 항목에 대한 텍스트를 검색할 수 있도록 문자열에 대한 포인터 및 메모리를 유지합니다.|
|CBS_LOWERCASE|선택 필드와 목록의 모든 텍스트를 소문자로 변환 합니다.|
|CBS_NOINTEGRALHEIGHT|콤보 상자의 크기가 콤보 상자를 만들 때 응용 프로그램에서 지정 하는 크기와 정확히 일치 하도록 지정 합니다. 일반적으로 Windows에서는 콤보 상자에 특정 항목이 표시되지 않도록 콤보 상자의 크기를 지정합니다.|
|CBS_OEMCONVERT|콤보 상자 편집 컨트롤에 입력 한 텍스트는 ANSI 문자 집합에서 OEM 문자 집합으로 변환 된 다음 다시 ANSI로 변환 됩니다. 따라서 애플리케이션이 콤보 상자의 ANSI 문자열을 OEM 문자로 변환하기 위해 `AnsiToOem` Windows 함수를 호출할 때 문자가 적절히 변환됩니다. 이 스타일은 파일 이름을 포함 하 고 CBS_SIMPLE 또는 CBS_DROPDOWN 스타일을 사용 하 여 만든 콤보 상자에만 적용 되는 콤보 상자에 가장 유용 합니다.|
|CBS_OWNERDRAWFIXED|목록 상자의 소유자는 콘텐츠를 그릴 책임이 있습니다. 목록 상자에 있는 항목의 높이가 모두 같습니다.|
|CBS_OWNERDRAWVARIABLE|목록 상자의 소유자는 콘텐츠를 그릴 책임이 있습니다. 목록 상자의 항목은 높이가 가변적입니다.|
|CBS_SIMPLE|목록 상자가 항상 표시 됩니다. 목록 상자에서 현재 선택된 항목이 편집 컨트롤에 표시됩니다.|
|CBS_SORT|목록 상자에 입력 한 문자열을 자동으로 정렬 합니다.|
|CBS_UPPERCASE|선택 필드와 목록의 모든 텍스트를 대문자로 변환 합니다.|

## <a name="edit-styles"></a>스타일 편집

편집 스타일은 [CEdit 클래스](../../mfc/reference/cedit-class.md) 개체에 적용 됩니다. [CEdit:: Create](../../mfc/reference/cedit-class.md#create)의 *dwstyle* 매개 변수에 스타일 조합을 지정 합니다. Windows에서 컨트롤 스타일을 편집 하는 방법에 대 한 자세한 내용은 [컨트롤 스타일 편집 (Windows)](/windows/win32/Controls/edit-control-styles)을 참조 하세요.

|스타일|설명|
|-----------|-----------------|
|ES_AUTOHSCROLL|사용자가 줄의 끝에 문자를 입력할 때 텍스트를 자동으로 오른쪽으로 스크롤합니다. 사용자가 ENTER 키를 누를 때 컨트롤은 모든 텍스트를 위치 0으로 다시 스크롤합니다.|
|ES_AUTOVSCROLL|사용자가 마지막 줄에서 ENTER 키를 누르면 자동으로 한 페이지씩 텍스트를 스크롤합니다.|
|ES_CENTER|한 줄 또는 여러 줄 편집 컨트롤의 텍스트를 가운데에 맞춥니다.|
|ES_LEFT|한 줄 또는 여러 줄 편집 컨트롤의 텍스트를 왼쪽에 맞춥니다.|
|ES_LOWERCASE|편집 컨트롤에 입력 될 때 모든 문자를 소문자로 변환 합니다.|
|ES_MULTILINE|여러 줄 편집 컨트롤을 지정 합니다. 기본값은 한 줄입니다. ES_AUTOVSCROLL 스타일이 지정 된 경우 편집 컨트롤은 가능한 한 많은 줄을 표시 하 고 사용자가 ENTER 키를 누를 때 세로로 스크롤합니다. ES_AUTOVSCROLL가 지정 되지 않은 경우 편집 컨트롤은 줄을 더 이상 표시할 수 없을 때 ENTER 키를 누르면 가능한 한 많은 줄과 경고음을 표시 합니다. ES_AUTOHSCROLL 스타일이 지정 된 경우 캐럿이 컨트롤의 오른쪽 가장자리를 벗어나면 자동으로 여러 줄 편집 컨트롤이 가로로 스크롤됩니다. 새 줄을 시작 하려면 사용자가 ENTER 키를 눌러야 합니다. ES_AUTOHSCROLL를 지정 하지 않으면 필요한 경우 컨트롤이 다음 줄의 시작 부분으로 자동으로 줄 바꿈 합니다. ENTER 키를 누르면 새 줄도 시작 됩니다. Wordwrap의 위치는 창 크기에 따라 결정 됩니다. 창 크기가 변경 되 면 wordwrap 위치가 바뀌고 텍스트가 다시 표시 됩니다. 여러 줄 편집 컨트롤에는 스크롤 막대가 있을 수 있습니다. 스크롤 막대가 있는 편집 컨트롤은 자체 스크롤 막대 메시지를 처리 합니다. 위에서 설명한 대로 스크롤 막대가 없는 편집 컨트롤을 스크롤하여 부모 창에서 보낸 스크롤 메시지를 처리 합니다.|
|ES_NOHIDESEL|일반적으로 편집 컨트롤은 컨트롤이 입력 포커스를 잃었을 때 선택을 숨기고 컨트롤이 입력 포커스를 받을 때 선택 영역을 반전 합니다. ES_NOHIDESEL을 지정 하면이 기본 작업이 삭제 됩니다.|
|ES_NUMBER|편집 컨트롤에 숫자만 입력할 수 있습니다.|
|ES_OEMCONVERT|편집 컨트롤에 입력 한 텍스트는 ANSI 문자 집합에서 OEM 문자 집합으로 변환 된 다음 다시 ANSI로 변환 됩니다. 이렇게 하면 응용 프로그램에서 `AnsiToOem` Windows 함수를 호출 하 여 편집 컨트롤의 ANSI 문자열을 OEM 문자로 변환할 때 문자 변환이 적절 하 게 수행 됩니다. 이 스타일은 파일 이름이 포함 된 편집 컨트롤에 가장 유용 합니다.|
|ES_PASSWORD|편집 컨트롤에 입력 될 때 모든 __\*__ 문자를 별표 ()로 표시 합니다. 응용 프로그램은 `SetPasswordChar` 멤버 함수를 사용 하 여 표시 되는 문자를 변경할 수 있습니다.|
|ES_READONLY|사용자가 편집 컨트롤의 텍스트를 입력 하거나 편집할 수 없도록 합니다.|
|ES_RIGHT|한 줄 또는 여러 줄 편집 컨트롤의 텍스트를 오른쪽에 맞춥니다.|
|ES_UPPERCASE|편집 컨트롤에 입력 될 때 모든 문자를 대문자로 변환 합니다.|
|ES_WANTRETURN|대화 상자에서 여러 줄 편집 컨트롤에 텍스트를 입력 하는 동안 사용자가 ENTER 키를 누를 때 캐리지 리턴을 삽입 하도록 지정 합니다. 이 스타일을 사용 하지 않는 경우 ENTER 키를 누르면 대화 상자 기본 누름 단추를 누르는 것과 동일한 효과가 있습니다. 이 스타일은 한 줄 편집 컨트롤에는 영향을 주지 않습니다.|

## <a name="frame-window-styles-mfc"></a>프레임 창 스타일

프레임 창 스타일은 [CFrameWnd 클래스](../../mfc/reference/cframewnd-class.md) 개체에 적용 됩니다. [CFrameWnd:: Create](../../mfc/reference/cframewnd-class.md#create)의 *dwstyle* 매개 변수에 스타일 조합을 지정 합니다.

|FWS_PREFIXTITLE|프레임 창 제목의 응용 프로그램 이름 앞에 문서 이름을 표시 합니다. 예를 들면 "Document-WordPad"입니다. 응용 프로그램 마법사의 문서 템플릿 문자열 탭에 표시 되는 문자열을 지정할 수 있습니다. 이 옵션을 해제 해야 하는 `CWnd::PreCreateWindow` 경우 멤버 함수를 재정의 합니다.|
|FWS_SNAPTOBARS|프레임 창에 도킹 되지 않고 부동 창에 있을 때 컨트롤 막대를 둘러싸는 프레임 창의 크기를 제어 합니다. 이 스타일은 컨트롤 막대에 맞게 창의 크기를 조정 합니다.|

## <a name="list-box-styles"></a>목록 상자 스타일

목록 상자 스타일은 [CListBox 클래스](../../mfc/reference/clistbox-class.md) 개체에 적용 됩니다. [CListBox:: Create](../../mfc/reference/clistbox-class.md#create)의 *dwstyle* 매개 변수에 스타일 조합을 지정 합니다. Windows의 목록 상자 스타일에 대 한 자세한 내용은 [목록 상자 스타일 (Windows)](/windows/win32/Controls/list-box-styles)을 참조 하세요.

|스타일|설명|
|-----------|-----------------|
|LBS_DISABLENOSCROLL|목록 상자에 스크롤할 충분 한 항목이 포함 되지 않은 경우 비활성화 된 세로 스크롤 막대가 표시 됩니다. 이 스타일을 사용하지 않으면 목록 상자에 충분한 항목이 포함되지 않은 경우 스크롤 막대가 숨겨집니다.|
|LBS_EXTENDEDSEL|사용자는 SHIFT 키와 마우스 또는 특수 키 조합을 사용 하 여 여러 항목을 선택할 수 있습니다.|
|LBS_HASSTRINGS|문자열로 구성 된 항목을 포함 하는 소유자 그리기 목록 상자를 지정 합니다. 목록 상자는 응용 프로그램에서 멤버 함수를 `GetText` 사용 하 여 특정 항목에 대 한 텍스트를 검색할 수 있도록 문자열에 대 한 메모리 및 포인터를 유지 합니다.|
|LBS_MULTICOLUMN|가로로 스크롤되는 여러 열로 된 목록 상자를 지정 합니다. 멤버 `SetColumnWidth` 함수는 열의 너비를 설정 합니다.|
|LBS_MULTIPLESEL|문자열 선택은 사용자가 문자열을 클릭 하거나 두 번 클릭할 때마다 전환 됩니다. 원하는 수의 문자열을 선택할 수 있습니다.|
|LBS_NODATA|비 데이터 목록 상자를 지정 합니다. 목록 상자의 항목 수가 1000를 초과 하는 경우이 스타일을 지정 합니다. 데이터 없음 목록 상자에는 LBS_OWNERDRAWFIXED 스타일도 있어야 하지만 LBS_SORT 또는 LBS_HASSTRINGS 스타일을 포함 하면 안 됩니다.<br/><br/> 데이터 없는 목록 상자는 항목에 대 한 문자열 또는 비트맵 데이터를 포함 하지 않는다는 점만 제외 하 고 소유자가 그린 목록 상자와 비슷합니다. 항목을 추가, 삽입 또는 삭제 하는 명령은 항상 지정 된 항목 데이터를 무시 합니다. 목록 상자 내에서 문자열을 찾기 위한 요청은 항상 실패 합니다. 시스템은 항목을 그려야 할 때 소유자 창에 WM_DRAWITEM 메시지를 보냅니다. WM_DRAWITEM 메시지와 함께 전달 `DRAWITEMSTRUCT` 되는 구조체의 itemID 멤버가 그릴 항목의 줄 번호를 지정 합니다. 데이터 없음 목록 상자는 WM_DELETEITEM 메시지를 보내지 않습니다.|
|LBS_NOINTEGRALHEIGHT|목록 상자의 크기는 목록 상자를 만들 때 응용 프로그램에서 지정 하는 크기와 동일 합니다. 일반적으로 Windows는 목록 상자에 부분 항목을 표시 하지 않도록 목록 상자 크기를 조정 합니다.|
|LBS_NOREDRAW|변경 내용이 적용 되 면 목록 상자 표시는 업데이트 되지 않습니다. 이 스타일은 WM_SETREDRAW 메시지를 전송 하 여 언제 든 지 변경할 수 있습니다.|
|LBS_NOSEL|목록 상자에 표시할 수 있지만 선택할 수 없는 항목을 포함 하도록 지정 합니다.|
|LBS_NOTIFY|사용자가 문자열을 클릭 하거나 두 번 클릭할 때마다 부모 창에서 입력 메시지를 받습니다.|
|LBS_OWNERDRAWFIXED|목록 상자의 소유자는 콘텐츠를 그릴 책임이 있습니다. 목록 상자에 있는 항목의 높이가 같습니다.|
|LBS_OWNERDRAWVARIABLE|목록 상자의 소유자는 콘텐츠를 그릴 책임이 있습니다. 목록 상자의 항목은 높이가 가변적입니다.|
|LBS_SORT|목록 상자의 문자열은 사전순으로 정렬 됩니다.|
|LBS_STANDARD|목록 상자의 문자열은 사전순으로 정렬 되 고, 사용자가 문자열을 클릭 하거나 두 번 클릭할 때마다 부모 창에서 입력 메시지를 받습니다. 목록 상자에는 모든 면에 테두리가 있습니다.|
|LBS_USETABSTOPS|목록 상자에서 해당 문자열을 그릴 때 탭 문자를 인식 하 고 확장할 수 있습니다. 기본 탭 위치는 32 대화 상자 단위입니다. 대화 상자 단위는 가로 또는 세로 거리입니다. 하나의 가로 대화 상자 단위는 현재 대화 상자 기본 너비 단위의 4 번째와 같습니다. 대화 상자 기본 단위는 현재 시스템 글꼴의 높이와 너비를 기준으로 계산 됩니다. Windows `GetDialogBaseUnits` 함수는 현재 대화 상자 기본 단위 (픽셀)를 반환 합니다. 이 스타일은 LBS_OWNERDRAWFIXED와 함께 사용 하면 안 됩니다.|
|LBS_WANTKEYBOARDINPUT|목록 상자 소유자는 입력 포커스가 있는 동안 사용자가 키를 누를 때마다 WM_VKEYTOITEM 또는 WM_CHARTOITEM 메시지를 받습니다. 이를 통해 응용 프로그램은 키보드 입력에서 특수 한 처리를 수행할 수 있습니다.|

## <a name="message-box-styles"></a>메시지 상자 스타일

메시지 상자 스타일은 [AfxMessageBox](../../mfc/reference/cstring-formatting-and-message-box-display.md#afxmessagebox) 항목에 적용 됩니다. 의`AfxMessageBox` *nType* 매개 변수에 스타일 조합을 지정 합니다. Windows의 메시지 상자 스타일에 대 한 자세한 내용은 [MessageBox 함수 (Windows)](/windows/win32/api/winuser/nf-winuser-messagebox)를 참조 하세요.

다음 메시지 상자 스타일을 사용할 수 있습니다.

### <a name="message-box-types"></a>메시지 상자 유형

|스타일|설명|
|-----------|-----------------|
|MB_ABORTRETRYIGNORE|메시지 상자에는 세 가지 누름 단추 있습니다. 중단, 다시 시도 및 무시 합니다.|
|MB_OK|메시지 상자에는 한 개의 누름이 포함 됩니다. OK.|
|MB_OKCANCEL|메시지 상자에는 두 가지 누름 단추 있습니다. 확인 하 고 취소 합니다.|
|MB_RETRYCANCEL|메시지 상자에는 두 가지 누름 단추 있습니다. 다시 시도 하 고 취소 합니다.|
|MB_YESNO|메시지 상자에는 두 가지 누름 단추 있습니다. 그렇기도 하고 그렇지 않기도 하고.|
|MB_YESNOCANCEL|메시지 상자에는 세 가지 누름 단추 있습니다. 예, 아니요 및 취소.|

### <a name="message-box-modality"></a>메시지 상자 모달

|스타일|설명|
|-----------|-----------------|
|MB_APPLMODAL|현재 창에서 작업을 계속 하기 전에 사용자가 메시지 상자에 응답 해야 합니다. 그러나 사용자는 다른 응용 프로그램의 창으로 이동 하 여 해당 창에서 작업할 수 있습니다. MB_SYSTEMMODAL 및 MB_TASKMODAL가 모두 지정 되지 않은 경우 기본값은 MB_APPLMODAL입니다.|
|MB_SYSTEMMODAL|모든 애플리케이션에는 사용자가 messagebox에 응답할 때까지 일시 중단 됩니다. 시스템 모달 메시지 상자는 즉각적인 주의가 필요한 치명적인 오류를 사용자에 게 알리는 데 사용 되며, 자주 사용 해야 합니다.|
|MB_TASKMODAL|MB_APPLMODAL와 유사 하지만 Microsoft Foundation class 응용 프로그램 내에서 유용 하지 않습니다. 이 플래그는 사용 가능한 창 핸들이 없는 호출 응용 프로그램이 나 라이브러리에 대해 예약 되어 있습니다.|

|MB_ICONINFORMATION|원에 "I"로 구성 된 아이콘이 메시지 상자에 나타납니다.|
|MB_ICONQUESTION|메시지 상자에 물음표 아이콘이 표시 됩니다.|
|MB_ICONSTOP|메시지 상자에 정지 서명 아이콘이 표시 됩니다.|

## <a name="scroll-bar-styles"></a>스크롤 막대 스타일

스크롤 막대 스타일은 [Cscrollbar 클래스](../../mfc/reference/cscrollbar-class.md) 개체에 적용 됩니다. [Cscrollbar:: Create](../../mfc/reference/cscrollbar-class.md#create)의 *dwstyle* 매개 변수에 스타일의 조합을 지정 합니다. 창의 스크롤 막대 컨트롤 스타일에 대 한 자세한 내용은 [Scroll Bar 컨트롤 스타일 (Windows)](/windows/win32/Controls/scroll-bar-control-styles)을 참조 하세요.

|스타일|설명|
|-----------|-----------------|
|SBS_BOTTOMALIGN|SBS_HORZ 스타일과 함께 사용 됩니다. 스크롤 막대의 아래쪽 가장자리를 `Create` 멤버 함수에 지정 된 사각형의 아래쪽 가장자리에 맞춥니다. 스크롤 막대는 시스템 스크롤 막대의 기본 높이를 가집니다.|
|SBS_HORZ|가로 스크롤 막대를 지정 합니다. SBS_BOTTOMALIGN 또는 SBS_TOPALIGN 스타일이 모두 지정 되지 않은 경우 스크롤 막대에는 `Create` 멤버 함수에 지정 된 높이, 너비 및 위치가 있습니다.|
|SBS_LEFTALIGN|SBS_VERT 스타일과 함께 사용 됩니다. 스크롤 막대의 왼쪽 가장자리가 `Create` 멤버 함수에 지정 된 사각형의 왼쪽 가장자리에 맞춰집니다. 스크롤 막대에는 시스템 스크롤 막대의 기본 너비가 있습니다.|
|SBS_RIGHTALIGN|SBS_VERT 스타일과 함께 사용 됩니다. 스크롤 막대의 오른쪽 가장자리가 `Create` 멤버 함수에 지정 된 사각형의 오른쪽 가장자리에 맞춰집니다. 스크롤 막대에는 시스템 스크롤 막대의 기본 너비가 있습니다.|
|SBS_SIZEBOX|크기 상자를 지정 합니다. SBS_SIZEBOXBOTTOMRIGHTALIGN 또는 SBS_SIZEBOXTOPLEFTALIGN 스타일을 모두 지정 하지 않으면 크기 상자에 `Create` 멤버 함수에 지정 된 높이, 너비 및 위치가 있습니다.|
|SBS_SIZEBOXBOTTOMRIGHTALIGN|SBS_SIZEBOX 스타일과 함께 사용 됩니다. 크기 상자의 오른쪽 아래 모퉁이는 `Create` 멤버 함수에 지정 된 사각형의 오른쪽 아래 모퉁이에 맞춰집니다. 크기 상자에는 시스템 크기 상자의 기본 크기가 있습니다.|
|SBS_SIZEBOXTOPLEFTALIGN|SBS_SIZEBOX 스타일과 함께 사용 됩니다. 크기 상자의 왼쪽 위 모퉁이는 `Create` 멤버 함수에 지정 된 사각형의 왼쪽 위 모퉁이에 맞춰집니다. 크기 상자에는 시스템 크기 상자의 기본 크기가 있습니다.|
|SBS_SIZEGRIP|SBS_SIZEBOX와 동일 하지만 볼록한 가장자리와 동일 합니다.|
|SBS_TOPALIGN|SBS_HORZ 스타일과 함께 사용 됩니다. 스크롤 막대의 위쪽 가장자리를 `Create` 멤버 함수에 지정 된 사각형의 위쪽 가장자리에 맞춥니다. 스크롤 막대는 시스템 스크롤 막대의 기본 높이를 가집니다.|
|SBS_VERT|세로 스크롤 막대를 지정 합니다. SBS_RIGHTALIGN 또는 SBS_LEFTALIGN 스타일이 모두 지정 되지 않은 경우 스크롤 막대에는 `Create` 멤버 함수에 지정 된 높이, 너비 및 위치가 있습니다.|

## <a name="static-styles"></a>정적 스타일

정적 스타일은 [Cstatic 클래스](../../mfc/reference/cstatic-class.md) 개체에 적용 됩니다. [Cstatic:: Create](../../mfc/reference/cstatic-class.md#create)의 *dwstyle* 매개 변수에 스타일의 조합을 지정 합니다. Windows의 정적 컨트롤 스타일에 대 한 자세한 내용은 [정적 컨트롤 스타일 (windows)](/windows/win32/Controls/static-control-styles)을 참조 하세요.

|스타일|설명|
|-----------|-----------------|
|SS_BITMAP|비트맵을 정적 컨트롤에 표시 하도록 지정 합니다. 지정 된 텍스트는 리소스 파일의 다른 곳에 정의 된 비트맵 (파일 이름 아님)의 이름입니다. 스타일은 nWidth 및 Nwidth 매개 변수를 무시 합니다. 컨트롤은 비트맵에 맞게 자동으로 크기를 조정 합니다.|
|SS_BLACKFRAME|창 프레임과 동일한 색을 사용 하 여 그린 프레임의 상자를 지정 합니다. 기본값은 검정입니다.|
|SS_BLACKRECT|창 프레임을 그리는 데 사용 되는 색으로 채워진 사각형을 지정 합니다. 기본값은 검정입니다.|
|SS_CENTER|단순 사각형을 지정 하 고 사각형의 가운데에 지정 된 텍스트를 표시 합니다. 텍스트를 표시 하기 전에 텍스트의 서식을 지정 합니다. 줄의 끝을 지나서 확장 되는 단어는 자동으로 다음으로 가운데 맞춤 된 선의 시작 부분으로 래핑됩니다.|
|SS_CENTERIMAGE|비트맵 또는 아이콘이 정적 컨트롤의 클라이언트 영역 보다 작은 경우 클라이언트 영역의 나머지 부분이 비트맵 또는 아이콘의 왼쪽 위 모퉁이에 있는 픽셀의 색으로 채워져 있음을 지정 합니다. 정적 컨트롤이 한 줄의 텍스트를 포함 하는 경우 텍스트는 컨트롤의 클라이언트 영역에서 세로 방향으로 가운데에 배치 됩니다.|
|SS_ENDELLIPSIS|또는 SS_PATHELLIPSIS 특정 문자열의 일부를 줄임표 (필요한 경우)로 바꿔서 결과가 지정 된 사각형에 맞춰집니다. <br /><br />SS_END_ELLIPSIS를 지정 하 여 문자열의 끝에 있는 문자를 바꾸거나 SS_PATHELLIPSIS를 지정 하 여 문자열의 중간에 있는 문자를 바꿀 수 있습니다. 문자열에 백슬래시 (\\) 문자가 포함 된 경우 SS_PATHELLIPSIS는 가능한 한 마지막 백슬래시 다음에 많은 텍스트를 유지 합니다.|
|SS_ENHMETAFILE|확장 메타 파일을 정적 컨트롤에 표시 하도록 지정 합니다. 지정 된 텍스트는 메타 파일의 이름입니다. 확장 메타 파일 정적 컨트롤의 크기는 고정 되어 있습니다. 메타 파일은 정적 컨트롤의 클라이언트 영역에 맞게 크기가 조정 됩니다.|
|SS_ETCHEDFRAME|EDGE_ETCHED 가장자리 스타일을 사용 하 여 정적 컨트롤의 프레임을 그립니다.|
|SS_ETCHEDHORZ|EDGE_ETCHED 가장자리 스타일을 사용 하 여 정적 컨트롤의 위쪽 및 아래쪽 가장자리를 그립니다.|
|SS_ETCHEDVERT|EDGE_ETCHED 가장자리 스타일을 사용 하 여 정적 컨트롤의 왼쪽 및 오른쪽 가장자리를 그립니다.|
|SS_GRAYFRAME|화면 배경 (바탕 화면)과 동일한 색을 사용 하 여 그린 프레임을 사용 하 여 상자를 지정 합니다. 기본값은 회색입니다.|
|SS_GRAYRECT|화면 배경을 채우는 데 사용 되는 색으로 채워진 사각형을 지정 합니다. 기본값은 회색입니다.|
|SS_ICON|대화 상자에 표시 되는 아이콘을 지정 합니다. 지정 된 텍스트는 리소스 파일의 다른 곳에 정의 된 아이콘 이름 (파일 이름이 아님)입니다. *Nwidth* 및 *nwidth* 매개 변수는 무시 됩니다. 아이콘 크기가 자동으로 조정 됩니다.|
|SS_LEFT|단순 사각형을 지정 하 고 사각형에 지정 된 텍스트 플러시를 표시 합니다. 텍스트를 표시 하기 전에 텍스트의 서식을 지정 합니다. 줄의 끝을 지나서 확장 하는 단어는 자동으로 다음 플러시 왼쪽 줄의 시작 부분으로 래핑됩니다.|
|SS_LEFTNOWORDWRAP|단순 사각형을 지정 하 고 사각형에 지정 된 텍스트 플러시를 표시 합니다. 탭이 확장 되지만 단어는 래핑되지 않습니다. 줄의 끝을 지나서 확장 하는 텍스트는 잘립니다.|
|SS_NOPREFIX|이 스타일을 지정 하지 않으면 Windows는 컨트롤의 텍스트에 있는 앰퍼샌드 (&) 문자를 액셀러레이터 접두사 문자로 해석 합니다. 이 경우 앰퍼샌드는 제거 되 고 문자열의 다음 문자에 밑줄이 그어집니다. 이 기능이 필요 하지 않은 텍스트를 정적 컨트롤이 포함 하는 경우에는 SS_NOPREFIX이 추가 될 수 있습니다. 이 정적 컨트롤 스타일은 정의 된 정적 컨트롤에 포함 될 수 있습니다. 비트 or 연산자를 사용 하 여 SS_NOPREFIX을 다른 스타일과 결합할 수 있습니다. 이는 일반적으로 대화 상자에서 정적 컨트롤에 앰퍼샌드를 포함할 수 있는 파일 이름 또는 기타 문자열을 표시 해야 하는 경우에 사용 됩니다.|
|SS_NOTIFY|사용자가 컨트롤을 클릭 하거나 두 번 클릭 하면 부모 창 STN_CLICKED, STN_DBLCLK, STN_DISABLE 및 STN_ENABLE 알림 메시지를 보냅니다.|
|SS_OWNERDRAW|정적 컨트롤의 소유자가 컨트롤의 그리기를 담당 하도록 지정 합니다. 컨트롤을 그려야 할 때마다 소유자 창에 WM_DRAWITEM 메시지가 수신 됩니다.|
|SS_REALSIZEIMAGE|정적 아이콘 또는 비트맵 컨트롤 (즉, SS_ICON 또는 SS_BITMAP 스타일이 있는 정적 컨트롤)의 크기가 로드 되거나 그려질 때 크기가 조정 되지 않도록 합니다. 아이콘 또는 비트맵이 대상 영역 보다 큰 경우 이미지는 잘립니다.|
|SS_RIGHT|단순 사각형을 지정 하 고 사각형에 지정 된 텍스트 플러시 오른쪽을 표시 합니다. 텍스트를 표시 하기 전에 텍스트의 서식을 지정 합니다. 줄의 끝을 지나서 확장 하는 단어는 자동으로 다음 들여쓰기 오른쪽 줄의 시작 부분으로 래핑됩니다.|
|SS_RIGHTJUST|컨트롤의 크기를 조정할 때 SS_BITMAP 또는 SS_ICON 스타일을 사용 하 여 정적 컨트롤의 오른쪽 아래 모퉁이가 고정 되도록 지정 합니다. 새 비트맵 또는 아이콘에 맞게 위쪽 및 왼쪽 면만 조정 됩니다.|
|SS_SIMPLE|사각형에 단순 사각형을 지정 하 고 한 줄의 텍스트 플러시를 표시 합니다. 텍스트 줄은 어떤 방식으로든 축약 하거나 변경할 수 없습니다. 컨트롤의 부모 창이 나 대화 상자는 WM_CTLCOLOR 메시지를 처리 하지 않아야 합니다.|
|SS_SUNKEN|정적 컨트롤 주위에 반 오목 테두리를 그립니다.|
|SS_USERITEM|사용자 정의 항목을 지정 합니다.|
|SS_WHITEFRAME|창 배경과 동일한 색을 사용 하 여 그린 프레임을 사용 하 여 상자를 지정 합니다. 기본값은 흰색입니다.|
|SS_WHITERECT|창 배경을 채우는 데 사용 되는 색으로 채워진 사각형을 지정 합니다. 기본값은 흰색입니다.|
|SS_WORDELLIPSIS|에 맞지 않고 줄임표를 추가 하는 텍스트를 자릅니다.|

## <a name="window-styles"></a>창 스타일

창 스타일은 [CWnd 클래스](../../mfc/reference/cwnd-class.md) 개체에 적용 됩니다. [Cwnd:: Create](../../mfc/reference/cwnd-class.md#create) 또는 [Cwnd:: Createex](../../mfc/reference/cwnd-class.md#createex)의 *dwstyle* 매개 변수에 스타일 조합을 지정 합니다. Windows의 창 스타일에 대 한 자세한 내용은 [창 스타일 (windows)](/windows/win32/winmsg/window-styles)을 참조 하세요.

|스타일|설명|
|-----------|-----------------|
|WS_BORDER|테두리가 있는 창을 만듭니다.|
|WS_CAPTION|제목 표시줄이 있는 창을 만듭니다 (WS_BORDER 스타일을 나타냄). WS_DLGFRAME 스타일과 함께 사용할 수 없습니다.|
|WS_CHILD|자식 창을 만듭니다. WS_POPUP 스타일과 함께 사용할 수 없습니다.|
|WS_CHILDWINDOW|WS_CHILD 스타일과 동일 합니다.|
|WS_CLIPCHILDREN|부모 창에서 그릴 때 자식 창이 차지 하는 영역을 제외 합니다. 부모 창을 만들 때 사용 됩니다.|
|WS_CLIPSIBLINGS|서로 상대적인 자식 창을 자릅니다. 즉, 특정 자식 창에서 그리기 메시지를 받을 때 WS_CLIPSIBLINGS 스타일은 자식 창의 영역에서 다른 모든 겹쳐진 자식 창을 업데이트 합니다. WS_CLIPSIBLINGS을 지정 하지 않고 자식 창이 겹치면 자식 창의 클라이언트 영역 내에서 그릴 때 인접 한 자식 창의 클라이언트 영역 내에 그릴 수 있습니다. WS_CHILD 스타일에만 사용 합니다.|
|WS_DISABLED|처음에 사용 하지 않도록 설정 된 창을 만듭니다.|
|WS_DLGFRAME|제목이 없는 이중 테두리를 사용 하 여 창을 만듭니다.|
|WS_GROUP|사용자가 화살표 키를 사용 하 여 한 컨트롤에서 다음 컨트롤로 이동할 수 있는 컨트롤 그룹의 첫 번째 컨트롤을 지정 합니다. 첫 번째 컨트롤이 동일한 그룹에 속하는 경우 WS_GROUP style으로 정의 된 모든 컨트롤은 FALSE입니다. WS_GROUP 스타일을 사용 하는 다음 컨트롤은 다음 그룹을 시작 합니다. 즉, 한 그룹은 다음에 시작 하는 위치로 끝납니다.|
|WS_HSCROLL|가로 스크롤 막대가 있는 창을 만듭니다.|
|WS_ICONIC|처음에 최소화 되는 창을 만듭니다. WS_MINIMIZE 스타일과 동일 합니다.|
|WS_MAXIMIZE|최대 크기의 창을 만듭니다.|
|WS_MAXIMIZEBOX|최대화 단추가 있는 창을 만듭니다.|
|WS_MINIMIZE|처음에 최소화 되는 창을 만듭니다. WS_OVERLAPPED 스타일에만 사용 합니다.|
|WS_MINIMIZEBOX|최소화 단추가 있는 창을 만듭니다.|
|WS_OVERLAPPED|겹쳐진 창을 만듭니다. 겹쳐진 창에는 일반적으로 캡션과 테두리가 있습니다.|
|WS_OVERLAPPEDWINDOW|WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX 및 WS_MAXIMIZEBOX 스타일을 사용 하 여 겹친 창을 만듭니다.|
|WS_POPUP|팝업 창을 만듭니다. WS_CHILD 스타일과 함께 사용할 수 없습니다.|
|WS_POPUPWINDOW|WS_BORDER, WS_POPUP 및 WS_SYSMENU 스타일을 사용 하 여 팝업 창을 만듭니다. WS_CAPTION 스타일을 WS_POPUPWINDOW 스타일과 결합 하 여 컨트롤 메뉴가 표시 되도록 해야 합니다.|
|WS_SIZEBOX|크기 조정 테두리가 있는 창을 만듭니다. WS_THICKFRAME 스타일과 동일 합니다.|
|WS_SYSMENU|제목 표시줄에 컨트롤 메뉴 상자를 포함 하는 창을 만듭니다. 제목 표시줄이 있는 창에만 사용 됩니다.|
|WS_TABSTOP|사용자가 TAB 키를 사용 하 여 이동할 수 있는 컨트롤 수 중 하나를 지정 합니다. TAB 키를 누르면 사용자가 WS_TABSTOP 스타일로 지정 된 다음 컨트롤로 이동 합니다.|
|WS_THICKFRAME|창의 크기를 조정 하는 데 사용할 수 있는 두꺼운 프레임을 사용 하 여 창을 만듭니다.|
|WS_TILED|겹쳐진 창을 만듭니다. 겹쳐진 창에는 제목 표시줄과 테두리가 있습니다. WS_OVERLAPPED 스타일과 동일 합니다.|
|WS_TILEDWINDOW|WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX 및 WS_MAXIMIZEBOX 스타일을 사용 하 여 겹친 창을 만듭니다. WS_OVERLAPPEDWINDOW 스타일과 동일 합니다.|
|WS_VISIBLE|처음에 표시 되는 창을 만듭니다.|
|WS_VSCROLL|세로 스크롤 막대가 있는 창을 만듭니다.|

## <a name="extended-window-styles"></a>확장 창 스타일

확장 창 스타일은 [CWnd 클래스](../../mfc/reference/cwnd-class.md) 개체에 적용 됩니다. [CWnd:: CreateEx](../../mfc/reference/cwnd-class.md#createex)의 *dwexstyle* 매개 변수에 스타일의 조합을 지정 합니다. Windows의 확장 창 스타일에 대 한 자세한 내용은 [확장 창 스타일 (windows)](/windows/win32/winmsg/extended-window-styles)을 참조 하세요.

|스타일|설명|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|이 스타일을 사용 하 여 만든 창에서 끌어서 놓기 파일을 허용 하도록 지정 합니다.|
|WS_EX_APPWINDOW|창이 표시 될 때 작업 표시줄에 최상위 창을 강제로 적용 합니다.|
|WS_EX_CLIENTEDGE|창에 3D 모양 즉, 오목한 가장자리가 있는 테두리를 지정 합니다.|
|WS_EX_CONTEXTHELP|창의 제목 표시줄에 물음표를 포함 합니다. 사용자가 물음표를 클릭하면 커서가 포인터가 있는 물음표로 변경됩니다. 사용자가 자식 창을 클릭 하면 자식은 WM_HELP 메시지를 수신 합니다.|
|WS_EX_CONTROLPARENT|사용자가 TAB 키를 사용 하 여 창의 자식 창 간을 탐색할 수 있습니다.|
|WS_EX_DLGMODALFRAME|*Dwstyle* 매개 변수에서 WS_CAPTION style 플래그를 지정할 때 제목 표시줄을 사용 하 여 선택적으로 만들 수 있는 이중 테두리가 있는 창을 지정 합니다.|
|WS_EX_LAYERED|창이 [계층화 된 창](/windows/win32/winmsg/window-features)입니다. CS_OWNDC 또는 CS_CLASSDC의 [클래스 스타일이](/windows/win32/winmsg/about-window-classes) 창에 있는 경우에는이 스타일을 사용할 수 없습니다. 그러나 Microsoft Windows 8은 자식 창에 대 한 WS_EX_LAYERED 스타일을 지원 합니다. 여기서 이전 Windows 버전은 최상위 windows 에서만 지원 합니다.|
|WS_EX_LEFT|창의 왼쪽에 맞춰진 왼쪽 맞춤 속성을 제공 합니다. 이 값이 기본값입니다.|
|WS_EX_LEFTSCROLLBAR|클라이언트 영역의 왼쪽에 세로 스크롤 막대를 배치 합니다.|
|WS_EX_LTRREADING|왼쪽에서 오른쪽으로 읽는 순서 속성을 사용 하 여 창 텍스트를 표시 합니다. 이 값이 기본값입니다.|
|WS_EX_MDICHILD|MDI 자식 창을 만듭니다.|
|WS_EX_NOPARENTNOTIFY|이 스타일을 사용 하 여 만든 자식 창이 자식 창이 만들어지거나 제거 될 때 해당 부모 창에 WM_PARENTNOTIFY 메시지를 보내지 않도록 지정 합니다.|
|WS_EX_OVERLAPPEDWINDOW|WS_EX_CLIENTEDGE 및 WS_EX_WINDOWEDGE 스타일을 결합 합니다.|
|WS_EX_PALETTEWINDOW|WS_EX_WINDOWEDGE 및 WS_EX_TOPMOST 스타일을 결합 합니다.|
|WS_EX_RIGHT|창에 대 한 일반 오른쪽 맞춤 속성을 제공 합니다. 이는 창 클래스에 따라 다릅니다.|
|WS_EX_RIGHTSCROLLBAR|클라이언트 영역의 오른쪽에 세로 스크롤 막대 (있는 경우)를 배치 합니다. 이 값이 기본값입니다.|
|WS_EX_RTLREADING|오른쪽에서 왼쪽으로 읽는 순서 속성을 사용 하 여 창 텍스트를 표시 합니다.|
|WS_EX_STATICEDGE|사용자 입력을 허용 하지 않는 항목에 사용할 3 차원 테두리 스타일을 사용 하 여 창을 만듭니다.|
|WS_EX_TOOLWINDOW|부동 도구 모음으로 사용 하기 위한 창인 도구 창을 만듭니다. 도구 창에는 일반 제목 표시줄보다 짧은 제목 표시줄이 있으며 창 제목은 더 작은 글꼴을 사용하여 그려집니다. 사용자가 Alt+Tab을 누를 때 나타나는 작업 표시줄이나 창에는 도구 창이 나타나지 않습니다.|
|WS_EX_TOPMOST|이 스타일을 사용 하 여 만든 창이 맨 위 창이 아닌 모든 창 위에 배치 되도록 지정 하 고 창이 비활성화 된 경우에도 위에 둡니다. 애플리케이션은 `SetWindowPos` 멤버 함수를 사용하여 이 특성을 추가 또는 제거할 수 있습니다.|
|WS_EX_TRANSPARENT|이 스타일을 사용 하 여 만든 창이 투명 하도록 지정 합니다. 즉, 이 창 아래에 있는 모든 창은 이 창에 의해 가려지지 않습니다. 이 스타일을 사용 하 여 만든 창은 그 아래에 있는 모든 형제 창이 업데이트 된 후에만 WM_PAINT 메시지를 수신 합니다.|
|WS_EX_WINDOWEDGE|창에 가장자리가 볼록하게 지정 된 테두리를 지정 합니다.|
