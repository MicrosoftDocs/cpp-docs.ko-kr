# <a name="styles-used-by-mfc"></a>MFC에서 사용하는 스타일

다음 스타일 플래그를 사용하여 해당 MFC 개체를 만들 때 창 또는 컨트롤 모양 및 동작을 지정합니다. 대부분의 경우 이러한 스타일은 클래스 `Create` 함수의 *dwStyle* 매개 변수에 설정됩니다.

## <a name="button-styles"></a><a name="button-styles"></a>단추 스타일

단추 스타일은 라디오 단추, 확인란 및 푸시버튼과 같은 [CButton 클래스](../../mfc/reference/cbutton-class.md) 개체에 적용됩니다. [CButton::Create](../../mfc/reference/cbutton-class.md#create)의 *dwStyle* 매개변수에서 스타일 조합을 지정합니다. Windows의 단추 스타일에 대한 자세한 내용은 [단추 스타일(Windows)을](/windows/win32/Controls/button-styles)참조하십시오.

### <a name="button-types"></a>버튼 유형

다음 표에는 단추 유형이 나열되어 있습니다. 선택적으로 다음 중 하나를 선택할 수 있습니다. 단추 유형을 지정하지 않으면 기본값이 BS_PUSHBUTTON.

|Type|Description|
|----------|-----------------|
|BS_3STATE|BST_CHECKED, BST_INDETERMINATE 및 BST_UNCHECKED 세 가지 상태로 확인란 단추를 만듭니다. 단추를 클릭하면 소유자 창에 BN_CLICKED 알림이 전송되지만 단추의 상태는 변경되지 않습니다. 기본적으로 연관된 텍스트는 확인란의 오른쪽에 표시됩니다. 확인란의 왼쪽에 텍스트를 표시하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용합니다.|
|BS_AUTO3STATE|BST_CHECKED, BST_INDETERMINATE 및 BST_UNCHECKED 세 가지 상태로 확인란 단추를 만듭니다. 단추를 클릭하면 소유자 창에 BN_CLICKED 알림이 전송되고 단추 상태가 변경됩니다. 단추는 BST_CHECKED, BST_INDETERMINATE 및 BST_UNCHECKED 순서로 주기를 말합니다. 기본적으로 연관된 텍스트는 확인란의 오른쪽에 표시됩니다. 확인란의 왼쪽에 텍스트를 표시하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용합니다.|
|BS_AUTOCHECKBOX|BST_CHECKED BST_UNCHECKED 상태의 두 가지 상태로 확인란 단추를 만듭니다. 단추를 클릭하면 소유자 창에 BN_CLICKED 알림이 전송되고 단추 상태가 변경됩니다. 기본적으로 연관된 텍스트는 확인란의 오른쪽에 표시됩니다. 확인란의 왼쪽에 텍스트를 표시하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용합니다.|
|BS_AUTORADIOBUTTON|BST_CHECKED BST_UNCHECKED 상태의 두 가지 상태로 라디오 단추를 만듭니다. 라디오 버튼은 일반적으로 그룹에서 사용되며 각 그룹은 한 번에 최대 하나의 선택 된 옵션을 가합니다. 단추를 클릭하면 소유자 창에 BN_CLICKED 알림이 전송되고, 클릭한 라디오 단추의 상태를 BST_CHECKED 설정하고, 단추 그룹의 다른 모든 라디오 단추의 상태를 BST_UNCHECKED. 기본적으로 연결된 텍스트는 라디오 단추의 오른쪽에 표시됩니다. 라디오 단추의 왼쪽에 텍스트를 표시하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용합니다.|
|BS_CHECKBOX|BST_CHECKED BST_UNCHECKED 상태의 두 가지 상태로 확인란 단추를 만듭니다. 단추를 클릭하면 소유자 창에 BN_CLICKED 알림이 전송되지만 단추의 상태는 변경되지 않습니다. 기본적으로 연관된 텍스트는 확인란의 오른쪽에 표시됩니다. 확인란의 왼쪽에 텍스트를 표시하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용합니다.|
|BS_COMMANDLINK|명령 링크 단추를 만듭니다. 명령 링크 단추는 Windows Vista전용 명령 단추로, 기본 텍스트 왼쪽에 녹색 화살표와 주 텍스트 아래에 있는 메모를 표시합니다. [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote)를 사용하여 메모 텍스트를 설정할 수 있습니다.|
|BS_DEFCOMMANDLINK|명령 링크 단추를 만듭니다. 명령 링크 단추는 Windows Vista전용 명령 단추로, 기본 텍스트 왼쪽에 녹색 화살표와 주 텍스트 아래에 있는 메모를 표시합니다. [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote)를 사용하여 메모 텍스트를 설정할 수 있습니다. 단추에 대화 상자가 있는 경우 ENTER 키를 누르면 단추에 입력 포커스가 없는 경우에도 대화 상자로 BN_CLICKED 알림이 전송됩니다.|
|BS_DEFPUSHBUTTON|검은색 테두리가 무거운 명령 단추를 만듭니다. 단추에 대화 상자가 있는 경우 ENTER 키를 누르면 단추에 입력 포커스가 없는 경우에도 대화 상자로 BN_CLICKED 알림이 전송됩니다.|
|BS_DEFSPLITBUTTON|분할 단추를 만듭니다. 분할 단추는 드롭다운 화살표에 인접한 버튼이 포함된 Windows Vista전용 명령 단추입니다. 단추를 클릭하면 기본 명령이 실행됩니다. 드롭다운 화살표를 클릭하면 추가 명령 메뉴가 나타납니다. 분할 버튼이 대화 상자에 있는 경우 ENTER 키를 누르면 버튼에 입력 포커스가 없는 경우에도 대화 상자에 BN_CLICKED 알림이 전송됩니다.|
|BS_GROUPBOX|다른 단추를 그룹화할 수 있는 사각형을 만듭니다. 이 스타일과 연관된 텍스트는 사각형의 왼쪽 위 모서리에 표시됩니다.|
|BS_OWNERDRAW|소유자가 그린 단추를 만듭니다. 프레임워크는 단추의 시각적 측면이 `DrawItem` 변경된 경우 메서드를 호출합니다. `CBitmapButton` 클래스를 사용할 때 이 스타일을 설정해야 합니다.|
|BS_PUSHBUTTON|사용자가 단추를 클릭할 때 소유자 창에 BN_CLICKED 알림을 보내는 명령 단추를 만듭니다.|
|BS_RADIOBUTTON|BST_CHECKED BST_UNCHECKED 상태의 두 가지 상태로 라디오 단추를 만듭니다. 라디오 버튼은 일반적으로 그룹에서 사용되며 각 그룹은 한 번에 최대 하나의 선택 된 옵션을 가합니다. 단추를 클릭하면 소유자 창에 BN_CLICKED 알림이 전송되지만 그룹의 단추 상태가 자동으로 변경되지는 않습니다. 기본적으로 연결된 텍스트는 라디오 단추의 오른쪽에 표시됩니다. 라디오 단추의 왼쪽에 텍스트를 표시하려면 BS_LEFTTEXT 또는 BS_RIGHTBUTTON 스타일을 사용합니다.|
|BS_SPLITBUTTON|분할 단추를 만듭니다. 분할 단추는 드롭다운 화살표에 인접한 버튼이 포함된 Windows Vista전용 명령 단추입니다. 단추를 클릭하면 기본 명령이 실행됩니다. 드롭다운 화살표를 클릭하면 추가 명령 메뉴가 나타납니다.|
|BS_USERBUTTON|더 이상 사용되지 않지만 16비트 버전의 Windows와의 호환성을 위해 제공되었습니다. Win32 기반 응용 프로그램은 대신 BS_OWNERDRAW 사용해야 합니다.|

### <a name="radio-button-and-check-box-styles"></a>라디오 버튼 및 확인란 스타일

다음 표에는 라디오 단추 및 확인란과 관련된 스타일이 나열되어 있습니다. 이러한 스타일은 다른 모든 단추 유형에서 무시됩니다. 선택적으로 다음 중 하나 이상을 선택할 수 있습니다.

|스타일|Description|
|-----------|-----------------|
|BS_LEFTTEXT|라디오 단추 또는 확인란 스타일과 결합하면 텍스트가 라디오 단추 또는 확인란의 왼쪽에 나타납니다.|
|BS_RIGHTBUTTON|라디오 단추 또는 확인란 스타일과 결합하면 텍스트가 라디오 단추 또는 확인란의 왼쪽에 나타납니다. 이 스타일은 BS_LEFTTEXT 스타일과 동일합니다.|
|BS_PUSHLIKE|확인란 또는 라디오 단추를 명령 단추처럼 보이게 하고 행동합니다. 상태가 BST_CHECKED 때 단추를 누르고, BST_INDETERMINATE 때 누르고 흐리게 표시하고 상태가 BST_UNCHECKED 때 해제됩니다.|

### <a name="button-text-alignment-styles"></a>단추 텍스트 정렬 스타일

다음 표에는 가로 및 세로 텍스트 정렬 옵션이 나열되어 있습니다. 선택적으로 다음 중 하나를 선택할 수 있습니다.

|스타일|Description|
|-----------|-----------------|
|BS_LEFT|왼쪽은 단추 사각형의 텍스트를 정렬합니다. 그러나 단추에 BS_RIGHTBUTTON 스타일이 없는 확인란 또는 라디오 단추인 경우 텍스트는 확인란 또는 라디오 단추의 오른쪽에 정렬된 상태로 남아 있습니다.|
|BS_RIGHT|오른쪽은 단추 사각형의 텍스트를 정렬합니다. 그러나 단추에 BS_RIGHTBUTTON 스타일이 없는 확인란 또는 라디오 단추인 경우 텍스트가 확인란 또는 라디오 단추의 오른쪽에 정렬됩니다.|
|BS_CENTER|단추 사각형에 텍스트를 가로로 중심으로 합니다.|
|BS_TOP|단추 사각형 의 맨 위에 텍스트를 배치합니다.|
|BS_BOTTOM|단추 사각형 의 맨 아래에 텍스트를 배치합니다.|

## <a name="combo-box-styles"></a><a name="combo-box-styles"></a>콤보 박스 스타일

다음 콤보 상자 스타일을 MFC에서 사용할 수 있습니다. Windows의 콤보 박스 스타일에 대한 자세한 내용은 [콤보 상자 스타일(Windows)을](/windows/win32/Controls/combo-box-styles)참조하십시오.

|스타일|Description|
|-----------|-----------------|
|CBS_AUTOHSCROLL|사용자가 줄 끝에 문자를 입력할 때 편집 컨트롤의 텍스트를 오른쪽으로 자동으로 스크롤합니다. 이 스타일을 설정하지 않으면 사각형 경계에 포함되는 텍스트만 허용됩니다.|
|CBS_DISABLENOSCROLL|목록 상자에 스크롤할 항목이 충분하지 않은 경우 목록 상자에 비활성화된 세로 스크롤 막대가 표시됩니다. 이 스타일을 사용하지 않으면 목록 상자에 충분한 항목이 포함되지 않은 경우 스크롤 막대가 숨겨집니다.|
|CBS_DROPDOWN|사용자가 편집 컨트롤 옆에 있는 아이콘을 선택하지 않으면 목록 상자가 표시되지 않는다는 점을 제외하면 CBS_SIMPLE 비슷합니다.|
|CBS_DROPDOWNLIST|CBS_DROPDOWN 마찬가지로 편집 컨트롤이 목록 상자에 현재 선택 영역을 표시하는 정적 텍스트 항목으로 대체됩니다.|
|CBS_HASSTRINGS|소유자-그리기 콤보 상자에는 문자열로 구성된 항목이 포함되어 있습니다. 콤보 상자는 애플리케이션에서 `GetText` 멤버 함수를 사용하여 특정 항목에 대한 텍스트를 검색할 수 있도록 문자열에 대한 포인터 및 메모리를 유지합니다.|
|CBS_LOWERCASE|선택 필드와 목록 모두에서 모든 텍스트를 소문자로 변환합니다.|
|CBS_NOINTEGRALHEIGHT|콤보 상자의 크기가 콤보 상자를 만들 때 응용 프로그램에서 지정한 크기와 정확히 동일하다고 지정합니다. 일반적으로 Windows에서는 콤보 상자에 특정 항목이 표시되지 않도록 콤보 상자의 크기를 지정합니다.|
|CBS_OEMCONVERT|콤보 상자 편집 컨트롤에 입력한 텍스트는 ANSI 문자 집합에서 OEM 문자 집합으로 변환된 다음 ANSI로 다시 변환됩니다. 따라서 애플리케이션이 콤보 상자의 ANSI 문자열을 OEM 문자로 변환하기 위해 `AnsiToOem` Windows 함수를 호출할 때 문자가 적절히 변환됩니다. 이 스타일은 파일 이름을 포함하는 콤보 상자에 매우 유용하며 CBS_SIMPLE 또는 CBS_DROPDOWN 스타일로 만들어진 콤보 상자에만 적용됩니다.|
|CBS_OWNERDRAWFIXED|목록 상자의 소유자는 해당 내용을 그릴 책임이 있습니다. 목록 상자의 항목은 모두 동일한 높이입니다.|
|CBS_OWNERDRAWVARIABLE|목록 상자의 소유자는 해당 내용을 그릴 책임이 있습니다. 목록 상자의 항목은 높이가 가변적입니다.|
|CBS_SIMPLE|목록 상자는 항상 표시됩니다. 목록 상자에서 현재 선택된 항목이 편집 컨트롤에 표시됩니다.|
|CBS_SORT|목록 상자에 입력된 문자열을 자동으로 정렬합니다.|
|CBS_UPPERCASE|선택 필드와 목록 모두에서 모든 텍스트를 대문자로 변환합니다.|

## <a name="edit-styles"></a><a name="edit-styles"></a>스타일 편집

편집 스타일은 [CEdit 클래스](../../mfc/reference/cedit-class.md) 객체에 적용됩니다. [CEdit::Create](../../mfc/reference/cedit-class.md#create)의 *dwStyle* 매개변수에서 스타일 조합을 지정합니다. Windows에서 컨트롤 편집 스타일에 대한 자세한 내용은 [Windows(제어 스타일 편집)를](/windows/win32/Controls/edit-control-styles)참조하십시오.

|스타일|Description|
|-----------|-----------------|
|ES_AUTOHSCROLL|사용자가 줄 끝에 문자를 입력할 때 텍스트를 오른쪽으로 10자로 자동으로 스크롤합니다. 사용자가 ENTER 키를 누르면 컨트롤이 모든 텍스트를 위치 0으로 다시 스크롤합니다.|
|ES_AUTOVSCROLL|사용자가 마지막 줄에서 ENTER를 누르면 텍스트가 한 페이지 위로 자동으로 스크롤됩니다.|
|ES_CENTER|텍스트를 한 줄 또는 다중 줄 편집 컨트롤에 중심을 둡자 합니다.|
|ES_LEFT|한 줄 또는 다중 줄 편집 컨트롤에서 텍스트를 왼쪽 정렬합니다.|
|ES_LOWERCASE|편집 컨트롤에 입력할 때 모든 문자를 소문자로 변환합니다.|
|ES_MULTILINE|다중 라인 편집 컨트롤을 지정합니다. 기본값은 한 줄입니다. ES_AUTOVSCROLL 스타일을 지정하면 편집 컨트롤은 가능한 한 많은 줄을 표시하고 사용자가 ENTER 키를 누를 때 세로로 스크롤됩니다. ES_AUTOVSCROLL 부여되지 않으면 편집 컨트롤에 가능한 한 많은 줄이 표시되고 더 이상 줄을 표시할 수 없을 때 ENTER를 누르면 비프음이 울립니다. ES_AUTOHSCROLL 스타일을 지정하면 캐리트가 컨트롤의 오른쪽 가장자리를 지나가면 다중 선 편집 컨트롤이 자동으로 가로로 스크롤됩니다. 새 줄을 시작하려면 ENTER를 눌러야 합니다. ES_AUTOHSCROLL 부여되지 않으면 컨트롤은 필요할 때 다음 줄의 시작 부분으로 자동으로 둘러쌉을 바꿈합니다. ENTER를 누르면 새 줄도 시작됩니다. 단어 줄 바꿈의 위치는 창 크기에 따라 결정됩니다. 창 크기가 변경되면 wordwrap 위치가 변경되고 텍스트가 다시 표시됩니다. 다중 줄 편집 컨트롤에는 스크롤 막대가 있을 수 있습니다. 스크롤 막대가 있는 편집 컨트롤은 자체 스크롤 막대 메시지를 처리합니다. 위에서 설명한 대로 스크롤 막대가 없는 컨트롤 편집은 부모 창에서 보낸 스크롤 메시지를 처리합니다.|
|ES_NOHIDESEL|일반적으로 편집 컨트롤은 컨트롤이 입력 포커스를 잃고 컨트롤이 입력 포커스를 수신할 때 선택을 반전할 때 선택을 숨깁니다. ES_NOHIDESEL 지정하면 이 기본 작업이 삭제됩니다.|
|ES_NUMBER|편집 컨트롤에 숫자만 입력할 수 있습니다.|
|ES_OEMCONVERT|편집 컨트롤에 입력된 텍스트는 ANSI 문자 집합에서 OEM 문자 집합으로 변환된 다음 ANSI로 다시 변환됩니다. 이렇게 하면 응용 프로그램이 Windows 함수를 호출하여 편집 컨트롤의 `AnsiToOem` ANSI 문자열을 OEM 문자로 변환할 때 적절한 문자 변환이 가능합니다. 이 스타일은 파일 이름이 포함된 컨트롤 편집에 가장 유용합니다.|
|ES_PASSWORD|편집 컨트롤에 입력될 때__\*__ 모든 문자를 별표()로 표시합니다. 응용 프로그램은 멤버 `SetPasswordChar` 함수를 사용하여 표시되는 문자를 변경할 수 있습니다.|
|ES_READONLY|사용자가 편집 컨트롤에 텍스트를 입력하거나 편집하지 못하도록 합니다.|
|ES_RIGHT|한 줄 또는 다중 줄 편집 컨트롤에서 텍스트를 오른쪽으로 정렬합니다.|
|ES_UPPERCASE|편집 컨트롤에 입력할 때 모든 문자를 대문자로 변환합니다.|
|ES_WANTRETURN|사용자가 대화 상자의 다중 줄 편집 컨트롤에 텍스트를 입력하는 동안 ENTER 키를 누를 때 캐리지 리턴이 삽입되도록 지정합니다. 이 스타일이 없으면 ENTER 키를 누르면 대화 상자 기본 푸시 버튼을 누르는 것과 동일한 효과가 있습니다. 이 스타일은 한 줄 편집 컨트롤에는 영향을 주지 않습니다.|

## <a name="frame-window-styles"></a><a name="frame-window-styles-mfc"></a>프레임 창 스타일

프레임 창 스타일은 [CFrameWnd 클래스](../../mfc/reference/cframewnd-class.md) 개체에 적용됩니다. [CFrameWnd::Create의](../../mfc/reference/cframewnd-class.md#create) *dwStyle* 매개변수에서 스타일 조합을 지정합니다.

|FWS_ADDTOTITLE|프레임 창 제목의 끝에 부가할 정보를 지정합니다. 예를 들어 "Microsoft 그리기 - 문서1에 그리기"를 예로 들 수 있습니다. 응용 프로그램 마법사의 문서 템플릿 문자열 탭에 표시되는 문자열을 지정할 수 있습니다. 이 옵션을 해제해야 하는 경우 멤버 `CWnd::PreCreateWindow` 함수를 재정의합니다.|
|FWS_PREFIXTITLE|프레임 창 제목에 응용 프로그램 이름 앞에 문서 이름을 표시합니다. 예를 들어 "문서 - 워드 패드". 응용 프로그램 마법사의 문서 템플릿 문자열 탭에 표시되는 문자열을 지정할 수 있습니다. 이 옵션을 해제해야 하는 경우 멤버 `CWnd::PreCreateWindow` 함수를 재정의합니다.|
|FWS_SNAPTOBARS|컨트롤 창이 프레임 창에 도킹되지 않고 부동 창에 있을 때 컨트롤 막대를 둘러싸는 프레임 창의 크기를 제어합니다. 이 스타일은 컨트롤 막대에 맞게 창의 크기를 조정합니다.|

## <a name="list-box-styles"></a><a name="list-box-styles"></a>목록 상자 스타일

목록 상자 스타일은 [CListBox 클래스](../../mfc/reference/clistbox-class.md) 개체에 적용됩니다. [CListBox::Create의](../../mfc/reference/clistbox-class.md#create) *dwStyle* 매개변수에서 스타일 조합을 지정합니다. Windows의 목록 상자 스타일에 대한 자세한 내용은 [목록 상자 스타일(Windows)을](/windows/win32/Controls/list-box-styles)참조하십시오.

|스타일|Description|
|-----------|-----------------|
|LBS_DISABLENOSCROLL|목록 상자에 스크롤할 항목이 충분하지 않은 경우 목록 상자에 비활성화된 세로 스크롤 막대가 표시됩니다. 이 스타일을 사용하지 않으면 목록 상자에 충분한 항목이 포함되지 않은 경우 스크롤 막대가 숨겨집니다.|
|LBS_EXTENDEDSEL|사용자는 SHIFT 키와 마우스 또는 특수 키 조합을 사용하여 여러 항목을 선택할 수 있습니다.|
|LBS_HASSTRINGS|문자열로 구성된 항목이 포함된 소유자-그리기 목록 상자를 지정합니다. 목록 상자는 문자열에 대한 메모리와 포인터를 유지 관리하므로 `GetText` 응용 프로그램에서 멤버 함수를 사용하여 특정 항목에 대한 텍스트를 검색할 수 있습니다.|
|LBS_MULTICOLUMN|가로로 스크롤되는 다중 열 목록 상자를 지정합니다. 멤버 `SetColumnWidth` 함수는 열의 너비를 설정합니다.|
|LBS_MULTIPLESEL|문자열 선택은 사용자가 문자열을 클릭하거나 두 번 클릭할 때마다 전환됩니다. 임의의 수의 문자열을 선택할 수 있습니다.|
|LBS_NODATA|데이터 없음 목록 상자를 지정합니다. 목록 상자의 항목 수가 1,000개를 초과할 때 이 스타일을 지정합니다. 데이터 없음 목록 상자에는 LBS_OWNERDRAWFIXED 스타일이 있어야 하지만 LBS_SORT 또는 LBS_HASSTRINGS 스타일이 없어야 합니다.<br/><br/> 데이터 없음 목록 상자는 항목에 대한 문자열 또는 비트맵 데이터가 없다는 점을 제외하면 소유자가 그린 목록 상자와 유사합니다. 항목을 추가, 삽입 또는 삭제하는 명령은 항상 지정된 항목 데이터를 무시합니다. 목록 상자 내에서 문자열을 찾는 요청은 항상 실패합니다. 항목을 그려야 할 때 시스템에서 WM_DRAWITEM 메시지를 소유자 창으로 보냅니다. WM_DRAWITEM 메시지와 함께 `DRAWITEMSTRUCT` 전달된 구조체의 itemID 멤버는 그릴 항목의 줄 번호를 지정합니다. 데이터 없음 목록 상자는 WM_DELETEITEM 메시지를 보내지 않습니다.|
|LBS_NOINTEGRALHEIGHT|목록 상자의 크기는 목록 상자를 만들 때 응용 프로그램에서 지정한 크기와 정확히 같습니다. 일반적으로 Windows는 목록 상자에 부분 항목이 표시되지 않도록 목록 상자의 크기를 조정합니다.|
|LBS_NOREDRAW|변경이 완료되면 목록 상자 표시가 업데이트되지 않습니다. 이 스타일은 WM_SETREDRAW 메시지를 보내 언제든지 변경할 수 있습니다.|
|LBS_NOSEL|목록 상자에 볼 수 있지만 선택하지 않은 항목이 포함되어 있는지 지정합니다.|
|LBS_NOTIFY|부모 창은 사용자가 문자열을 클릭하거나 두 번 클릭할 때마다 입력 메시지를 받습니다.|
|LBS_OWNERDRAWFIXED|목록 상자의 소유자는 해당 내용을 그릴 책임이 있습니다. 목록 상자의 항목의 높이는 동일합니다.|
|LBS_OWNERDRAWVARIABLE|목록 상자의 소유자는 해당 내용을 그릴 책임이 있습니다. 목록 상자의 항목은 높이가 가변적입니다.|
|LBS_SORT|목록 상자의 문자열은 사전순으로 정렬됩니다.|
|LBS_STANDARD|목록 상자의 문자열은 사전순으로 정렬되며 사용자가 문자열을 클릭하거나 두 번 클릭할 때마다 부모 창에 입력 메시지가 표시됩니다. 목록 상자에는 모든 면의 테두리가 포함되어 있습니다.|
|LBS_USETABSTOPS|문자열을 그릴 때 목록 상자에서 탭 문자를 인식하고 확장할 수 있습니다. 기본 탭 위치는 32개의 대화 상자 단위입니다. 대화 상자는 수평 또는 수직 거리입니다. 하나의 가로 대화 상자 단위는 현재 대화 상자 기본 너비 단위의 4분의 1과 같습니다. 대화 상자 기본 단위는 현재 시스템 글꼴의 높이와 너비를 기준으로 계산됩니다. Windows `GetDialogBaseUnits` 함수는 현재 대화 상자 기본 단위를 픽셀 단위로 반환합니다. 이 스타일은 LBS_OWNERDRAWFIXED 함께 사용해서는 안 됩니다.|
|LBS_WANTKEYBOARDINPUT|목록 상자에 입력 포커스가 있는 동안 사용자가 키를 누를 때마다 목록 상자의 소유자는 WM_VKEYTOITEM 또는 WM_CHARTOITEM 메시지를 받습니다. 이렇게 하면 응용 프로그램이 키보드 입력에서 특수 처리를 수행할 수 있습니다.|

## <a name="scroll-bar-styles"></a><a name="scroll-bar-styles"></a>스크롤 막대 스타일

스크롤 막대 스타일은 [CScrollBar 클래스](../../mfc/reference/cscrollbar-class.md) 개체에 적용됩니다. [CScrollBar::Create의](../../mfc/reference/cscrollbar-class.md#create) *dwStyle* 매개변수에서 스타일 조합을 지정합니다. Windows에서 스크롤 막대 컨트롤 스타일에 대한 자세한 내용은 [스크롤 막대 컨트롤 스타일(Windows)을](/windows/win32/Controls/scroll-bar-control-styles)참조하십시오.

|스타일|Description|
|-----------|-----------------|
|SBS_BOTTOMALIGN|SBS_HORZ 스타일과 함께 사용. 스크롤 막대의 아래쪽 가장자리는 멤버 함수에 지정된 사각형의 아래쪽 가장자리와 `Create` 정렬됩니다. 스크롤 막대에는 시스템 스크롤 막대의 기본 높이가 있습니다.|
|SBS_HORZ|가로 스크롤 막대를 지정합니다. SBS_BOTTOMALIGN SBS_TOPALIGN 스타일을 지정하지 않으면 스크롤 막대에는 멤버 함수에 지정된 높이, 너비 및 위치가 `Create` 있습니다.|
|SBS_LEFTALIGN|SBS_VERT 스타일에 사용. 스크롤 막대의 왼쪽 가장자리는 `Create` 멤버 함수에 지정된 사각형의 왼쪽 가장자리와 정렬됩니다. 스크롤 막대에는 시스템 스크롤 막대의 기본 너비가 있습니다.|
|SBS_RIGHTALIGN|SBS_VERT 스타일에 사용. 스크롤 막대의 오른쪽 가장자리는 `Create` 멤버 함수에 지정된 사각형의 오른쪽 가장자리에 정렬됩니다. 스크롤 막대에는 시스템 스크롤 막대의 기본 너비가 있습니다.|
|SBS_SIZEBOX|크기 상자를 지정합니다. SBS_SIZEBOXBOTTOMRIGHTALIGN SBS_SIZEBOXTOPLEFTALIGN 스타일을 지정하지 않으면 크기 상자에 멤버 함수에 지정된 높이, `Create` 너비 및 위치가 있습니다.|
|SBS_SIZEBOXBOTTOMRIGHTALIGN|SBS_SIZEBOX 스타일과 함께 사용. 크기 상자의 오른쪽 아래 모서리는 멤버 함수에 지정된 사각형의 오른쪽 아래 `Create` 모서리에 정렬됩니다. 크기 상자에는 시스템 크기 상자의 기본 크기가 있습니다.|

## <a name="static-styles"></a><a name="static-styles"></a>정적 스타일

정적 스타일은 [CStatic 클래스](../../mfc/reference/cstatic-class.md) 개체에 적용됩니다. [CStatic::Create](../../mfc/reference/cstatic-class.md#create)의 *dwStyle* 매개변수에서 스타일 조합을 지정합니다. Windows의 정적 제어 스타일에 대한 자세한 내용은 [Windows(정적 제어 스타일)를](/windows/win32/Controls/static-control-styles)참조하십시오.

|스타일|Description|
|-----------|-----------------|
|SS_BITMAP|정적 컨트롤에 표시될 비트맵을 지정합니다. 지정된 텍스트는 리소스 파일의 다른 위치에 정의된 비트맵(파일 이름이 아님)의 이름입니다. 스타일은 nWidth 및 nHeight 매개 변수를 무시합니다. 컨트롤은 비트맵을 수용하기 위해 자동으로 크기를 조정합니다.|
|SS_BLACKFRAME|창 프레임과 동일한 색상으로 그려진 프레임이 있는 상자를 지정합니다. 기본값은 검정입니다.|
|SS_BLACKRECT|창 프레임을 그리는 데 사용되는 색상으로 채워진 사각형을 지정합니다. 기본값은 검정입니다.|
|SS_CENTER|간단한 사각형을 지정하고 사각형 가운데에 지정된 텍스트를 표시합니다. 텍스트가 표시되기 전에 서식이 지정됩니다. 줄의 끝을 지나 연장할 단어는 자동으로 다음 가운데 줄의 시작 부분으로 래핑됩니다.|
|SS_CENTERIMAGE|비트맵 또는 아이콘이 정적 컨트롤의 클라이언트 영역보다 작으면 클라이언트 영역의 나머지 영역은 비트맵 또는 아이콘의 왼쪽 상단 모서리에 있는 픽셀의 색상으로 채워지므로 지정합니다. 정적 컨트롤에 한 줄의 텍스트가 포함된 경우 텍스트는 컨트롤의 클라이언트 영역에서 세로로 가운데에 배치됩니다.|
|SS_ENDELLIPSIS|또는 SS_PATHELLIPSIS 결과가 지정된 사각형에 맞않도록 필요한 경우 지정된 문자열의 일부를 타원으로 바꿉습니다. <br /><br />문자열 끝에 있는 문자를 바꿀 SS_END_ELLIPSIS 지정하거나 문자열 중간에 있는 문자를 바꿀 SS_PATHELLIPSIS 수 있습니다. 문자열에 백슬래시 ()\\문자가 포함된 경우 SS_PATHELLIPSIS 마지막 백슬래시 후 의 텍스트를 최대한 많이 보존합니다.|
|SS_ENHMETAFILE|정적 컨트롤에 표시될 향상된 메타파일을 지정합니다. 지정된 텍스트는 메타파일의 이름입니다. 향상된 메타파일 정적 제어에는 고정된 크기가 있습니다. 메타파일은 정적 컨트롤의 클라이언트 영역에 맞게 배율이 조정됩니다.|
|SS_ETCHEDFRAME|EDGE_ETCHED 가장자리 스타일을 사용하여 정적 컨트롤의 프레임을 그립니다.|
|SS_ETCHEDHORZ|EDGE_ETCHED 가장자리 스타일을 사용하여 정적 컨트롤의 위쪽 및 아래쪽 가장자리를 그립니다.|
|SS_ETCHEDVERT|EDGE_ETCHED 가장자리 스타일을 사용하여 정적 컨트롤의 왼쪽 및 오른쪽 가장자리를 그립니다.|
|SS_GRAYFRAME|화면 배경(바탕 화면)과 동일한 색상으로 그려진 프레임이 있는 상자를 지정합니다. 기본값은 회색입니다.|
|SS_GRAYRECT|화면 배경을 채우는 데 사용되는 색상으로 채워진 사각형을 지정합니다. 기본값은 회색입니다.|
|SS_ICON|대화 상자에 표시되는 아이콘을 지정합니다. 지정된 텍스트는 리소스 파일의 다른 위치에 정의된 아이콘(파일 이름이 아님)의 이름입니다. *nWidth* 및 *nHeight* 매개 변수는 무시됩니다. 아이콘자체의 크기가 자동으로 조정됩니다.|
|SS_LEFT|간단한 사각형을 지정하고 지정된 텍스트를 왼쪽으로 표시합니다. 텍스트가 표시되기 전에 서식이 지정됩니다. 줄의 끝을 지나 확장되는 단어는 다음 플러시 왼쪽 줄의 시작 부분으로 자동으로 래핑됩니다.|
|SS_LEFTNOWORDWRAP|간단한 사각형을 지정하고 지정된 텍스트를 왼쪽으로 표시합니다. 탭은 확장되지만 단어는 래핑되지 않습니다. 줄의 끝을 지나 확장되는 텍스트는 잘립니다.|
|SS_NOPREFIX|이 스타일을 지정하지 않는 한 Windows는 컨트롤 텍스트의 앰퍼샌드(&) 문자를 가속기 접두사 문자로 해석합니다. 이 경우 앰퍼샌드가 제거되고 문자열의 다음 문자에 밑줄이 그어집니다. 정적 컨트롤이 이 기능을 원하지 않는 텍스트를 포함하는 경우 SS_NOPREFIX 추가할 수 있습니다. 이 정적 제어 스타일은 정의된 정적 컨트롤에 포함될 수 있습니다. 비트별 OR 연산자사용을 사용하여 SS_NOPREFIX 다른 스타일과 결합할 수 있습니다. 이는 앰퍼샌드를 포함할 수 있는 파일 이름 이나 다른 문자열대화 상자에 정적 컨트롤에 표시 해야 하는 경우에 가장 자주 사용 됩니다.|
|SS_NOTIFY|사용자가 컨트롤을 클릭하거나 두 번 클릭할 때 부모 창 STN_CLICKED, STN_DBLCLK, STN_DISABLE 및 STN_ENABLE 알림 메시지를 보냅니다.|
|SS_OWNERDRAW|정적 컨트롤의 소유자가 컨트롤을 그리는 책임이 있음을 지정합니다. 컨트롤을 그려야 할 때마다 소유자 창에 WM_DRAWITEM 메시지가 표시됩니다.|
|SS_REALSIZEIMAGE|정적 아이콘 또는 비트맵 컨트롤(즉, SS_ICON 또는 SS_BITMAP 스타일이 있는 정적 컨트롤)이 로드되거나 그려질 때 크기를 조정하지 못하도록 합니다. 아이콘 또는 비트맵이 대상 영역보다 크면 이미지가 잘립니다.|
|SS_RIGHT|간단한 사각형을 지정하고 지정된 텍스트를 직사각형에 바로 표시합니다. 텍스트가 표시되기 전에 서식이 지정됩니다. 줄의 끝을 지나 확장되는 단어는 다음 플러시 오른쪽 줄의 시작 부분으로 자동으로 래핑됩니다.|
|SS_RIGHTJUST|SS_BITMAP 또는 SS_ICON 스타일로 정적 컨트롤의 오른쪽 아래 모서리가 컨트롤크기를 조정할 때 고정된 상태로 유지되도록 지정합니다. 새 비트맵 또는 아이콘을 수용하도록 위쪽과 왼쪽면만 조정됩니다.|
|SS_SIMPLE|간단한 사각형을 지정하고 사각형에 왼쪽 텍스트의 한 줄을 표시합니다. 텍스트 줄을 줄이거나 변경할 수 없습니다. 컨트롤의 부모 창 또는 대화 상자가 WM_CTLCOLOR 메시지를 처리해서는 안 됩니다.|
|SS_SUNKEN|정적 컨트롤 주위에 반 가라앉은 테두리를 그립니다.|
|SS_USERITEM|사용자 정의 항목을 지정합니다.|
|SS_WHITEFRAME|창 배경과 동일한 색상으로 그려진 프레임이 있는 상자를 지정합니다. 기본값은 흰색입니다.|
|SS_WHITERECT|창 배경을 채우는 데 사용되는 색상으로 채워진 사각형을 지정합니다. 기본값은 흰색입니다.|
|SS_WORDELLIPSIS|맞지 않는 텍스트를 잘리고 타원을 추가합니다.|

## <a name="window-styles"></a><a name="window-styles"></a>창 스타일

창 스타일은 [CWnd 클래스](../../mfc/reference/cwnd-class.md) 개체에 적용됩니다. [CWnd:::만들기](../../mfc/reference/cwnd-class.md#create) 또는 [CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex)의 *dwStyle* 매개변수에서 스타일 조합을 지정합니다. Windows의 창 스타일에 대한 자세한 내용은 [창 스타일(Windows)을](/windows/win32/winmsg/window-styles)참조하십시오.

|스타일|Description|
|-----------|-----------------|
|WS_BORDER|테두리가 있는 창을 만듭니다.|
|WS_CAPTION|제목 표시줄이 있는 창을 만듭니다(WS_BORDER 스타일을 의미함). WS_DLGFRAME 스타일로는 사용할 수 없습니다.|
|WS_CHILD|하위 창을 만듭니다. WS_POPUP 스타일과 함께 사용할 수 없습니다.|
|WS_CHILDWINDOW|WS_CHILD 스타일과 동일합니다.|
|WS_CLIPCHILDREN|상위 창 내에서 그릴 때 자식 창이 차지하는 영역은 제외됩니다. 상위 창을 만들 때 사용됩니다.|
|WS_CLIPSIBLINGS|서로 를 기준으로 자식 창을 클립; 즉, 특정 자식 창이 페인트 메시지를 받으면 WS_CLIPSIBLINGS 스타일은 업데이트할 하위 창 영역에서 겹치는 다른 모든 하위 창을 클립합니다. (WS_CLIPSIBLINGS 부여되지 않고 자식 창이 겹치는 경우 자식 창의 클라이언트 영역 내에서 그릴 때 인접한 자식 창의 클라이언트 영역 내에서 그릴 수 있습니다. WS_CHILD 스타일에만 사용하십시오.|
|WS_DISABLED|처음에 비활성화된 창을 만듭니다.|
|WS_DLGFRAME|이중 테두리가 있지만 제목이 없는 창을 만듭니다.|
|WS_GROUP|사용자가 화살표 키를 사용하여 한 컨트롤에서 다음 컨트롤로 이동할 수 있는 컨트롤 그룹의 첫 번째 컨트롤을 지정합니다. 첫 번째 컨트롤 이후에 false 스타일 WS_GROUP 정의된 모든 컨트롤은 동일한 그룹에 속합니다. WS_GROUP 스타일로 다음 컨트롤은 다음 그룹을 시작합니다(즉, 한 그룹은 다음 그룹이 시작되는 지점).|
|WS_HSCROLL|가로 스크롤 막대가 있는 창을 만듭니다.|
|WS_ICONIC|처음에 최소화된 창을 만듭니다. WS_MINIMIZE 스타일과 동일합니다.|
|WS_MAXIMIZE|최대 크기의 창을 만듭니다.|
|WS_MAXIMIZEBOX|최대화 단추가 있는 창을 만듭니다.|
|WS_MINIMIZE|처음에 최소화된 창을 만듭니다. WS_OVERLAPPED 스타일에만 사용하십시오.|
|WS_MINIMIZEBOX|최소화 단추가 있는 창을 만듭니다.|
|WS_OVERLAPPED|겹친 창을 만듭니다. 겹친 창에는 일반적으로 캡션과 테두리가 있습니다.|
|WS_OVERLAPPEDWINDOW|WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX 및 WS_MAXIMIZEBOX 스타일로 겹친 창을 만듭니다.|
|WS_POPUP|팝업 창을 만듭니다. WS_CHILD 스타일로는 사용할 수 없습니다.|
|WS_POPUPWINDOW|WS_BORDER, WS_POPUP 및 WS_SYSMENU 스타일로 팝업 창을 만듭니다. 컨트롤 메뉴를 표시하려면 WS_CAPTION 스타일을 WS_POPUPWINDOW 스타일과 결합해야 합니다.|
|WS_SIZEBOX|크기 테두리가 있는 창을 만듭니다. WS_THICKFRAME 스타일과 동일합니다.|
|WS_SYSMENU|제목 표시줄에 컨트롤 메뉴 상자가 있는 창을 만듭니다. 제목 표시줄이 있는 창에만 사용됩니다.|
|WS_TABSTOP|사용자가 TAB 키를 사용하여 이동할 수 있는 컨트롤 수에 관계없이 지정합니다. TAB 키는 사용자를 WS_TABSTOP 스타일로 지정된 다음 컨트롤로 이동합니다.|
|WS_THICKFRAME|창 크기를 조정하는 데 사용할 수 있는 두꺼운 프레임이 있는 창을 만듭니다.|
|WS_TILED|겹친 창을 만듭니다. 겹친 창에는 제목 표시줄과 테두리가 있습니다. WS_OVERLAPPED 스타일과 동일합니다.|
|WS_TILEDWINDOW|WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX 및 WS_MAXIMIZEBOX 스타일로 겹친 창을 만듭니다. WS_OVERLAPPEDWINDOW 스타일과 동일합니다.|
|WS_VISIBLE|처음에 표시되는 창을 만듭니다.|
|WS_VSCROLL|세로 스크롤 막대가 있는 창을 만듭니다.|

## <a name="extended-window-styles"></a><a name="extended-window-styles"></a>확장 창 스타일

확장 창 스타일은 [CWnd 클래스](../../mfc/reference/cwnd-class.md) 개체에 적용됩니다. [CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex)의 *dwExStyle* 매개변수에서 스타일 조합을 지정합니다. Windows의 확장 창 스타일에 대한 자세한 내용은 [확장 창 스타일(Windows)을](/windows/win32/winmsg/extended-window-styles)참조하십시오.

|스타일|Description|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|이 스타일로 만든 창이 끌어서 놓기 파일을 허용되도록 지정합니다.|
|WS_EX_APPWINDOW|창이 표시될 때 작업 표시줄에 최상위 창을 강제로 합니다.|
|WS_EX_CLIENTEDGE|창에 3D 모양, 즉 가라앉은 가장자리가 있는 테두리가 있는지 지정합니다.|
|WS_EX_CONTEXTHELP|창의 제목 표시줄에 물음표를 포함합니다. 사용자가 물음표를 클릭하면 커서가 포인터가 있는 물음표로 변경됩니다. 그런 다음 사용자가 자식 창을 클릭하면 자식은 WM_HELP 메시지를 받습니다.|
|WS_EX_CONTROLPARENT|사용자가 TAB 키를 사용하여 창의 자식 창 사이를 탐색할 수 있습니다.|
|WS_EX_DLGMODALFRAME|*dwStyle* 매개변수에서 WS_CAPTION 스타일 플래그를 지정할 때 제목 표시줄로 만들 수 있는 이중 테두리가 있는 창을 지정합니다.|
|WS_EX_LAYERED|창은 [계층화된 창입니다.](/windows/win32/winmsg/window-features) 창에 CS_OWNDC 또는 CS_CLASSDC [클래스 스타일이](/windows/win32/winmsg/about-window-classes) 있는 경우 이 스타일을 사용할 수 없습니다. 그러나, 마이크로소프트 윈도 8 아이 창에 대 한 WS_EX_LAYERED 스타일을 지원 지 않습니다., 어디 이전 Windows 버전은 최상위 창에 대 한 그것을 지원.|
|WS_EX_LEFT|창 에 일반 왼쪽 정렬 속성을 제공합니다. 이것이 기본값입니다.|
|WS_EX_LEFTSCROLLBAR|클라이언트 영역의 왼쪽에 세로 스크롤 막대를 배치합니다.|
|WS_EX_LTRREADING|왼쪽에서 오른쪽으로 읽기 순서 속성을 사용하여 창 텍스트를 표시합니다. 이것이 기본값입니다.|
|WS_EX_MDICHILD|MDI 자식 창을 만듭니다.|
|WS_EX_NOPARENTNOTIFY|이 스타일로 만든 자식 창은 자식 창을 만들거나 소멸할 때 WM_PARENTNOTIFY 메시지를 부모 창으로 보내지 않도록 지정합니다.|
|WS_EX_OVERLAPPEDWINDOW|WS_EX_CLIENTEDGE 및 WS_EX_WINDOWEDGE 스타일을 결합|
|WS_EX_PALETTEWINDOW|WS_EX_WINDOWEDGE WS_EX_TOPMOST 스타일을 결합합니다.|
|WS_EX_RIGHT|창에 일반 오른쪽 정렬 속성을 제공합니다. 이는 창 클래스에 따라 다릅니다.|
|WS_EX_RIGHTSCROLLBAR|클라이언트 영역의 오른쪽에 세로 스크롤 막대(있는 경우)를 배치합니다. 이것이 기본값입니다.|
|WS_EX_RTLREADING|오른쪽에서 왼쪽 읽기 순서 속성을 사용하여 창 텍스트를 표시합니다.|
|WS_EX_STATICEDGE|사용자 입력을 허용하지 않는 항목에 사용할 3차원 테두리 스타일이 있는 창을 만듭니다.|
|WS_EX_TOOLWINDOW|부동 도구 모음으로 사용할 창인 도구 창을 만듭니다. 도구 창에는 일반 제목 표시줄보다 짧은 제목 표시줄이 있으며 창 제목은 더 작은 글꼴을 사용하여 그려집니다. 사용자가 Alt+Tab을 누를 때 나타나는 작업 표시줄이나 창에는 도구 창이 나타나지 않습니다.|
|WS_EX_TOPMOST|이 스타일로 만든 창을 맨 위에 없는 모든 창 위에 배치하고 창이 비활성화된 경우에도 창을 위에 두어야 합니다. 애플리케이션은 `SetWindowPos` 멤버 함수를 사용하여 이 특성을 추가 또는 제거할 수 있습니다.|
|WS_EX_TRANSPARENT|이 스타일로 만든 창이 투명하도록 지정합니다. 즉, 이 창 아래에 있는 모든 창은 이 창에 의해 가려지지 않습니다. 이 스타일로 만든 창은 WM_PAINT 메시지 아래에 있는 모든 형제 창이 업데이트된 후에만 메시지를 수신합니다.|
|WS_EX_WINDOWEDGE|창에 가장자리가 위온인 테두리가 있는지 지정합니다.|
