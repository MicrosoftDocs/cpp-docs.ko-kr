## <a name="remarks"></a>설명

"트리 뷰 컨트롤"에 디스크에 인덱스 또는 파일 및 디렉터리의 항목 머리글 문서 등의 항목의 계층적 목록을 표시 하는 창입니다. 각 항목 레이블 및 선택적 비트맵 이미지로 구성 되며 각 항목에 연결 된 하위 항목의 목록을 가질 수 있습니다. 항목을 클릭 하 여 사용자를 확장 하 고 연결 된 목록을 하위 항목 축소 수 있습니다.

이 컨트롤 (및 따라서는 `CTreeCtrl` 클래스) 이상 Windows 98 및 Windows NT 버전 4에서 실행 중인 프로그램에만 사용할 수 있습니다.

사용 하 여 대 한 자세한 내용은 `CTreeCtrl`를 참조 하세요.

- [컨트롤](../../mfc/controls-mfc.md)

##  <a name="create"></a>  CTreeCtrl::Create

대화 상자 템플릿에서 트리 컨트롤을 지정 하는 경우 또는 사용 중인 경우 [CTreeView](../../mfc/reference/ctreeview-class.md), 트리 컨트롤은 대화 상자 또는 보기를 만들 때 자동으로 생성 됩니다.

```
virtual BOOL Create(
    DWORD dwStyle,
    const RECT& rect,
    CWnd* pParentWnd,
    UINT nID);
```

### <a name="parameters"></a>매개 변수

*dwStyle*<br/>
트리 뷰 컨트롤의 스타일을 지정합니다. 에 설명 된 창 스타일을 적용 [CreateWindow](/windows/desktop/api/winuser/nf-winuser-createwindowa)와 원하는 조합의 [트리 뷰 컨트롤 스타일](/windows/desktop/Controls/tree-view-control-window-styles) Windows SDK에 설명 된 대로 합니다.

*rect*<br/>
트리 뷰 컨트롤의 크기와 위치를 지정합니다. 수 있습니다는 [CRect](../../atl-mfc-shared/reference/crect-class.md) 개체 또는 [RECT](https://msdn.microsoft.com/library/windows/desktop/dd162897) 구조입니다.

*pParentWnd*<br/>
트리 뷰 컨트롤의 부모 창에 일반적으로 지정 된 `CDialog`합니다. NULL이 아니어야 합니다.

*nID*<br/>
트리 뷰 컨트롤의 ID를 지정합니다.

### <a name="return-value"></a>반환 값

초기화에 성공 하면 0이 아닌 값 그렇지 않으면 0입니다.

### <a name="remarks"></a>설명

일부 다른 창의 자식 창으로 트리 컨트롤을 만들려는 경우 사용 된 `Create` 멤버 함수입니다. 사용 하 여 트리 컨트롤을 만드는 경우 `Create`, 다른 트리 뷰 스타일 외에도 WS_VISIBLE를 전달 해야 있습니다.

생성 된 `CTreeCtrl` 두 단계에서. 첫 번째 호출 생성자 호출 `Create`, 트리 뷰 컨트롤을 만들고 연결 하는 `CTreeCtrl` 개체입니다.

- TVGN_FIRSTVISIBLE 표시 되는 첫 번째 항목을 검색합니다.

- TVGN_NEXT 다음 형제 항목을 검색합니다.

- TVGN_NEXTVISIBLE 지정된 된 항목 뒤에 오는 다음 표시 가능 항목을 검색 합니다.

- TVGN_PREVIOUS 이전 형제 항목을 검색 합니다.

- TVGN_PREVIOUSVISIBLE 지정된 된 항목 앞에 오는 첫 번째 표시 가능 항목을 검색 합니다.

- TVGN_ROOT 지정된 된 항목은 일부 루트 항목의 첫 번째 자식 항목을 검색 합니다.
