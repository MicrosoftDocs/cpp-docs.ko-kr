# <a name="reference-windows-performance-analyzer-views"></a>참조: Windows 성능 분석기 보기

::: moniker range="<=vs-2017"

C++ 빌드 인사이트 도구는 Visual Studio 2019에서 사용할 수 있습니다. 이 버전에 대한 설명서를 보려면 이 문서의 Visual Studio **버전** 선택기 컨트롤을 Visual Studio 2019로 설정합니다. 이 페이지의 목조 테이블 맨 위에 있습니다.

::: moniker-end
::: moniker range="vs-2019"

이 문서에서는 WPA(Windows 성능 분석기)에서 사용할 수 있는 각 C++ 빌드 인사이트 보기에 대한 세부 정보를 제공합니다. 이 페이지를 사용하여 다음을 찾습니다.

- 데이터 열 설명; 및
- 의도된 사용 및 기본 보기 모드를 포함하여 각 보기에 사용할 수 있는 사전 설정이 있습니다.

WPA를 처음 접하는 경우 [먼저 C++ 빌드 인사이트를 위한 WPA의 기본 사항에](/cpp/build-insights/tutorials/wpa-basics)익숙해지는 것이 좋습니다.

## <a name="build-explorer"></a>빌드 탐색기

탐색기 빌드 보기는 다음과 같은 데 사용됩니다.

- 병렬 처리 문제를 진단하고,
- 구문 분석, 코드 생성 또는 연결에 의해 빌드 시간이 지배되는지 확인하고
- 병목 현상과 비정상적으로 긴 빌드 활동을 식별할 수 있습니다.

### <a name="build-explorer-view-data-columns"></a>탐색기 뷰 데이터 열 빌드

| 열 이름 | Description |
|-|-|
| 빌드 타임라인 설명 | 현재 활동 또는 속성이 발생하는 타임라인에 대한 텍스트 설명입니다. |
| 빌드타임라인ID          | 현재 활동 또는 속성이 발생하는 타임라인에 대한 0기반 식별자입니다. |
| 구성 요소                | 현재 이벤트가 내보내졌을 때 컴파일되거나 연결되는 구성 요소입니다. 이 열의 값은 * \<\> * 이 이벤트와 연관된 구성 요소가 없을 때 호출 X 정보입니다. X는 이벤트가 내보낼 때 실행되는 호출에 대한 고유한 숫자 식별자입니다. 이 식별자는 이 이벤트에 대한 호출Id 열의 식별자와 동일합니다. |
| 개수                    | 이 데이터 행으로 표시되는 활동 또는 속성 의 수입니다. 이 값은 항상 1이며 여러 행이 그룹화될 때 집계 시나리오에서만 유용합니다. |
| 익스클루시브CPU타임         | 이 활동에 사용되는 CPU 시간(밀리초)입니다. 하위 활동에 소요된 시간은 이 금액에 포함되지 않습니다. |
| 독점 기간        | 활동의 밀리초 기간입니다. 하위 활동의 기간은 이 금액에 포함되지 않습니다. |
| 포함CPU타임         | 이 활동 및 모든 자식 활동에 사용되는 CPU 시간(밀리초)입니다. |
| 포괄 기간        | 모든 하위 활동을 포함하여 본 활동의 밀리초 기간입니다. |
| 호출 설명    | 이 이벤트가 발생한 호출에 대한 텍스트 설명입니다. 설명에는 *cl.exe* 또는 *link.exe*인지 및 고유한 숫자 호출 식별자가 포함됩니다. 해당하는 경우 호출 중에 컴파일하거나 연결된 구성 요소에 대한 전체 경로를 포함합니다. 구성 요소를 빌드하지 않는 호출또는 여러 구성 요소를 빌드하는 호출의 경우 경로가 비어 있습니다. 호출 식별자는 호출Id 열의 식별자와 동일합니다. |
| 호출 ID             | 이 이벤트가 발생한 호출에 대한 고유한 숫자 식별자입니다. |
| 속성                     | 이 이벤트로 표시되는 활동 또는 속성의 이름입니다. |
| Time                     | 이벤트가 발생한 시기를 식별하는 타임스탬프입니다. |
| 도구                     | 이 이벤트가 발생했을 때 실행되는 도구입니다. 이 열의 값은 CL 또는 Link입니다. |
| Type                     | 현재 이벤트의 형식입니다. 이 값은 활동 또는 속성입니다. |
| 값                    | 현재 이벤트가 속성인 경우 이 열에는 해당 값이 포함됩니다. 이 열은 현재 이벤트가 활동일 때 비워 둡니까. |

### <a name="build-explorer-view-presets"></a>탐색기 뷰 사전 설정 빌드

| 사전 설정 이름           | 기본 설정 보기 모드 | 사용 방법 |
|-----------------------|---------------------|------------|
| 활동 통계   | 그래프 / 표       | 이 사전 설정을 사용하여 모든 Build 탐색기 활동에 대한 집계된 통계를 볼 수 있습니다. 테이블 모드에서 빌드가 구문 분석, 코드 생성 또는 링커에 의해 지배되는지 한눈에 알 수 있습니다. 각 활동에 대한 집계 된 기간은 내림차순으로 정렬됩니다. 최상위 노드를 확장하여 드릴인을 통해 이러한 상위 활동에 가장 많은 시간이 소요되는 호출을 쉽게 찾을 수 있습니다. 원하는 경우 WPA 설정을 조정하여 평균 또는 기타 유형의 집계를 표시할 수 있습니다. 그래프 모드에서 빌드 하는 동안 각 활동이 활성화 되는 경우를 참조 합니다. |
| 호출           | 그래프               | 시작 시간별로 정렬된 그래프 보기에서 호출 목록을 아래로 스크롤합니다. CPU(샘플) 보기와 함께 사용하여 낮은 CPU 사용률 영역에 맞는 호출을 찾을 수 있습니다. 병렬 처리 문제를 감지합니다. |
| 호출 속성 | 테이블               | 지정된 컴파일러 또는 링커 호출에 대한 주요 정보를 빠르게 찾을 수 있습니다. 버전, 작업 디렉토리 또는 해당 버전을 호출하는 데 사용되는 전체 명령줄을 결정합니다. |
| 타임라인             | 그래프               | 빌드가 병렬화된 방식에 대한 막대 그래프를 참조하세요. 병렬 처리 문제와 병목 현상을 한 눈에 파악할 수 있습니다. 필요에 따라 막대에 다른 의미를 할당하도록 WPA를 구성합니다. 호출 설명을 마지막으로 그룹화된 열로 선택하여 모든 호출의 색상 코드 막대 막대 그래프를 봅니다. 그것은 신속 하 게 시간이 많이 걸리는 범인을 식별 하는 데 도움이. 그런 다음 활동 이름을 마지막으로 그룹화된 열로 확대하여 선택하여 가장 긴 부분을 확인합니다. |

## <a name="files"></a>파일

파일 보기는 다음과 같은 데 사용됩니다.

- 가장 자주 포함되는 헤더를 결정하고
- 미리 컴파일된 헤더(PCH)에 포함할 내용을 결정하는 데 도움이 됩니다.

### <a name="files-view-data-columns"></a>파일은 데이터 열을 볼 수 있습니다.

| 열 이름              | Description |
|--------------------------|-------------|
| ActivityName             | 이 파일 이벤트가 내세되었을 때 진행 중인 활동입니다. 현재 이 값은 항상 *구문 분석입니다.* |
| 빌드 타임라인 설명 | * |
| 빌드타임라인ID          | * |
| 구성 요소                | * |
| 개수                    | * |
| 깊이                    | 이 파일이 발견되는 포함 트리의 0기반 위치입니다. 포함 트리의 루트에서 카운트가 시작됩니다. 값 0은 일반적으로 .c/cpp 파일에 해당합니다. |
| 독점 기간        | * |
| 포함된 바이               | 현재 파일을 포함하는 파일의 전체 경로입니다. |
| 포함 된 경로             | 현재 파일의 전체 경로입니다. |
| 포괄 기간        | * |
| 호출 ID             | * |
| StartTime                | 현재 파일 이벤트가 내보낸 시간을 나타내는 타임스탬프입니다. |
| 도구                     | * |

\*이 열의 값은 [탐색기 빌드](#build-explorer-view-data-columns) 보기와 동일합니다.

### <a name="files-view-presets"></a>파일보기 사전 설정

| 사전 설정 이름 | 기본 설정 보기 모드 | 사용 방법 |
|-------------|---------------------|------------|
| 통계  | 테이블               | 내림차순으로 목록을 확인하여 집계된 구문 분석 시간이 가장 높은 파일을 확인합니다. 이 정보를 사용하여 헤더를 재구성하거나 PCH에 포함할 내용을 결정할 수 있습니다. |

## <a name="functions"></a>Functions

Functions 뷰는 코드 생성 시간이 지나치게 긴 함수를 식별하는 데 사용됩니다.

### <a name="functions-view-data-columns"></a>함수는 데이터 열을 봅니다.

| 열 이름              | Description |
|--------------------------|-------------|
| ActivityName             | 이 함수 이벤트가 내세되었을 때 진행 중인 활동입니다. 현재 이 값은 항상 *CodeGeneration*입니다. |
| 빌드 타임라인 설명 | * |
| 빌드타임라인ID          | * |
| 구성 요소                | * |
| 개수                    | * |
| Duration                 | 이 함수에 대한 코드 생성 활동의 기간입니다. |
| FunctionName             | 코드 생성을 받는 함수의 이름입니다. |
| 호출 ID             | * |
| StartTime                | 현재 함수 이벤트가 내보내진 시기를 나타내는 타임스탬프입니다. |
| 도구                     | * |

\*이 열의 값은 [탐색기 빌드](#build-explorer-view-data-columns) 보기와 동일합니다.

### <a name="functions-view-presets"></a>함수보기 사전 설정

| 사전 설정 이름 | 기본 설정 보기 모드 | 사용 방법 |
|-------------|---------------------|------------|
| 통계  | 테이블               | 내림차순으로 목록을 확인하여 집계된 코드 생성 시간이 가장 높은 함수를 확인합니다. 코드가 **__forceinline** 키워드를 과도하게 사용하거나 일부 함수가 너무 클 수 있음을 암시할 수 있습니다. |
| 타임라인   | 그래프               | 이 막대 그래프를 보고 생성하는 데 가장 많은 시간이 소요되는 함수의 위치와 기간을 알아봅니다. 탐색기 빌드 보기에서 병목 현상과 일치하는지 확인합니다. 이 경우 코드 생성 시간을 줄이고 빌드 시간을 단축하기 위해 적절한 조치를 취합니다. |
