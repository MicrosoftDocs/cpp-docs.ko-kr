# <a name="previewing-resources"></a>리소스 미리 보기
리소스 미리 보기를 열지 않고 그래픽 리소스를 볼 수 있습니다. 리소스 식별자를 숫자로 변경 되므로 컴파일된 한 후 미리 보기는 실행 파일에 대 한 유용 합니다. 종종 이러한 숫자 식별자 충분 한 정보를 제공 하지 않습니다, 리소스 미리 보기를 사용 하면 신속 하 게 식별 합니다.  
  
 다음과 같은 리소스 종류의 시각적 레이아웃을 미리 볼 수 있습니다.  
  
-   Bitmap  
  
-   대화 상자  
  
-   아이콘  
  
-   메뉴  
  
-   Cursor  
  
-   Toolbar  
  
 시각적인 미리 보기 기능이 가속기, 매니페스트, 문자열 테이블 및 버전 정보 리소스에 적용 되지 않습니다.  
  
### <a name="to-preview-resources"></a>리소스를 미리 보려면  
  
1.  [리소스 뷰](../windows/resource-view-window.md) 또는 문서 창, 예를 들어 IDD_ABOUTBOX 리소스를 선택 합니다.  
  
     **참고** 프로젝트에 .rc 파일이 아직 없는 경우 [새 리소스 스크립트 파일 만들기](../windows/how-to-create-a-resource-script-file.md)를 참조하세요.  
  
2.  에 [속성 창](/visualstudio/ide/reference/properties-window), 클릭는 **속성 페이지** 단추입니다.  
  
     \- 또는 -  
  
3.  에 **보기** 메뉴를 클릭 하 여 **속성 페이지**합니다.  
  
     해당 리소스의 미리 보기를 표시 합니다. 리소스에 대 한 속성 페이지가 열립니다. 다음 위쪽을 사용 하 여 하 고 아래쪽 화살표 키를 계획할 때이 또는 문서 창에서 트리 컨트롤을 탐색할 수 있습니다. 속성 페이지를 열린 채로 되 고를 선택 하 여 미리 될 수 있으므로 리소스 됩니다.  
  
 관리 되는 프로젝트에 리소스를 추가 정보를 참조 하십시오 [데스크톱 응용 프로그램의 리소스](/dotnet/framework/resources/index) 에 *.NET Framework 개발자 가이드입니다.* 수동으로 관리 되는 프로젝트에 리소스 파일을 추가, 리소스 액세스, 정적 리소스 표시 및 속성에 리소스 문자열 할당에 대 한 정보를 참조 하십시오. [데스크톱 앱에 대 한 리소스 파일 만들기](/dotnet/framework/resources/creating-resource-files-for-desktop-apps)합니다. 전역화 및 지역화의 관리 되는 응용 프로그램의 리소스에 대 한 정보를 참조 하십시오. [전역화 및 지역화.NET Framework 응용 프로그램](/dotnet/standard/globalization-localization/index)합니다.
