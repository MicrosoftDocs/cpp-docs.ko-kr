# <a name="w64"></a>__w64

**Microsoft 전용**

이 키워드는 사용 되지 않습니다. Visual Studio 2013 이전 버전의 Visual Studio에서 이렇게 하면 변수를 표시할로 컴파일할 때 되도록 [/wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) 컴파일러는 64 비트 컴파일러를 통해 컴파일 중인 경우 보고 되는 모든 경고를 보고 합니다.

## <a name="syntax"></a>구문

> *type* **__w64** *identifier*

### <a name="parameters"></a>매개 변수

*type*<br/>
32 비트에서 64 비트 컴파일러로 이식 되는 코드에서 문제가 발생할 수 있는 세 가지 형식 중 하나: **int**를 **긴**, 또는 포인터입니다.

*identifier*<br/>
만들고 있는 변수에 대한 식별자입니다.

## <a name="remarks"></a>설명

> [!IMPORTANT]
>  합니다 [/wp64](../build/reference/wp64-detect-64-bit-portability-issues.md) 컴파일러 옵션 및 **__w64** 키워드는 Visual Studio 2010 및 Visual Studio 2013에서 사용 되지 않으며 Visual Studio 2013 부터는 제거 합니다. 사용 하는 경우는 `/Wp64` 컴파일러 옵션이 명령줄 컴파일러에서 명령줄 경고 d9002 합니다. 합니다 **__w64** 키워드는 자동으로 무시 됩니다. 이 옵션과 키워드를 사용 하 여 64 비트 이식성 문제를 감지, 대신 Microsoft C++ 64 비트 플랫폼을 대상으로 하는 컴파일러입니다. 자세한 내용은 [시각적 구성 C++ 64 비트 x64에 대 한 대상](../build/configuring-programs-for-64-bit-visual-cpp.md)합니다.

에 있는 형식 정의 **__w64** 32 비트 x86 및 x64에서는 64 비트가 되어야 합니다.

Microsoft의 버전을 사용 하 여 이식성 문제를 검색할 C++ Visual Studio 2010 이전의 컴파일러는 **__w64** 32 비트 및 64 비트 플랫폼 간에 크기를 변경 하는 형식 정의에 키워드를 지정 해야 합니다. 이러한 형식에 대 한 **__w64** typedef의 32 비트 정의에 표시 되어야 합니다.

이전 버전과 호환성에 대 한 **_w64** 에 대 한 동의어가 **__w64** 하지 않는 한 컴파일러 옵션 [/Za \(언어 확장을 사용 하지 않도록 설정)](../build/reference/za-ze-disable-language-extensions.md) 지정 됩니다.

합니다 **__w64** 키워드는 컴파일을 사용 하지 않는 경우 무시 됩니다 `/Wp64`합니다.

64비트로 이식하는 방법에 대한 자세한 내용은 다음 항목을 참조하세요.

- [MSVC 컴파일러 옵션](../build/reference/compiler-options.md)

- [32 비트 코드를 64 비트 코드로 이식](../build/common-visual-cpp-64-bit-migration-issues.md)

- [64비트, x64 대상을 위한 Visual C++ 구성](../build/configuring-programs-for-64-bit-visual-cpp.md)

## <a name="example"></a>예제

```cpp
// __w64.cpp
// compile with: /W3 /Wp64
typedef int Int_32;
#ifdef _WIN64
typedef __int64 Int_Native;
#else
typedef int __w64 Int_Native;
#endif

int main() {
   Int_32 i0 = 5;
   Int_Native i1 = 10;
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int

   // char __w64 c;  error, cannot use __w64 on char
}
```
