# <a name="displaying-assertions"></a>어설션 표시

서비스에 연결 된 클라이언트가 응답 하지 않는 것으로 표시 되는 경우 서비스는 표시 되지 않고 표시 될 수 없는 메시지 상자를 표시 했을 수 있습니다. Visual Studio 디버거를 사용 하 여 코드를 디버그 하 여이를 확인할 수 있습니다 (이 섹션의 앞부분에서 [작업 관리자 사용](../atl/using-task-manager.md) 참조).

서비스에 표시 되지 않는 메시지 상자가 표시 되는 것으로 확인 되 면 서비스를 다시 사용 하기 전에 **서비스를 데스크톱과 상호 작용 하도록 허용** 옵션을 설정 하는 것이 좋습니다. 이 옵션은 서비스에 표시 되는 모든 메시지 상자가 바탕 화면에 나타나도록 허용 하는 시작 매개 변수입니다. 이 옵션을 설정 하려면 서비스 제어판 응용 프로그램을 열고, 서비스를 선택 하 고, **시작** 을 클릭 한 후 **서비스에서 데스크톱과 상호 작용 하도록 허용** 옵션을 선택 합니다.
