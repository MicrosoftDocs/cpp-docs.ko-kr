# <a name="displaying-assertions"></a>어설션 표시
서비스에 연결 된 클라이언트는 응답 하지 않는 것 처럼, 서비스 어설션 있고 볼 수 없는 메시지 상자에 표시 될 수 있습니다. 코드를 디버깅 하려면 Visual c + +의 디버거를 사용 하 여이 확인할 수 있습니다 (참조 [작업 관리자를 사용 하 여](../atl/using-task-manager.md) 이 섹션의 앞부분에 나오는).  
  
 설정 하려는 서비스 볼 수 없는 메시지 상자를 표시 하 고 있는지 확인 한 경우는 **데스크톱과 상호 작용 하도록 허용 서비스** 서비스를 다시 사용 하기 전에 옵션입니다. 이 옵션은 바탕 화면에 표시 하기 위해 서비스에서 표시 되는 모든 메시지 상자를 허용 하는 시작 매개 변수입니다. 이 옵션을 설정 하려면 서비스 제어판 응용 프로그램을 열고, 서비스, 클릭 **시작**를 선택한 후는 **데스크톱과 상호 작용 하도록 허용 서비스** 옵션입니다.
