# <a name="c-runtime-error-r6002"></a>C 런타임 오류 R6002

부동 소수점 지원 로드 안 됨

필요한 부동 소수점 라이브러리가 연결 되지 않았습니다.

> [!NOTE]
> 앱을 실행 하는 동안이 오류 메시지가 표시 되 면 내부 문제가 있기 때문에 앱이 종료 되었습니다. 이 오류가 발생 하는 이유는 여러 가지가 있지만 앱 코드의 결함으로 인해 발생 하거나 특정 컴퓨터 프로세서용으로 빌드되지 않은 앱을 실행 하려고 시도 하는 경우가 종종 있습니다.
>
> 이 오류를 해결하려면 다음 단계를 시도할 수 있습니다.
>
> - **제어판** 의 **앱 및 기능** 또는 **프로그램 및 기능** 페이지를 사용 하 여 프로그램을 복구 하거나 다시 설치 합니다.
> - **제어판** 의 소프트웨어 업데이트에 대 한 **Windows 업데이트** 를 확인 합니다.
> - 앱의 업데이트 된 버전을 확인 합니다. 문제가 지속 되 면 앱 공급 업체에 문의 하세요.

**프로그래머를 위한 정보**

이 오류는 부동 소수점 라이브러리가 연결 되지 않은 경우 앱에서 발생할 수 있습니다. 다음 원인 중 하나를 확인 합니다.

- 또는 함수에 대 한 형식 문자열에 `printf_s` `scanf_s` 부동 소수점 형식 지정이 포함 되어 있고 프로그램에 부동 소수점 값 이나 변수가 포함 되어 있지 않습니다. 이 문제를 해결 하려면 부동 소수점 형식 지정에 해당 하는 부동 소수점 인수를 사용 하거나 프로그램의 다른 위치에서 부동 소수점 할당을 수행 합니다. 이로 인해 부동 소수점 지원이 로드 됩니다.

- 컴파일러는 필요한 경우에만 부동 소수점 지원을 로드 하 여 프로그램의 크기를 최소화 합니다. 컴파일러가 형식 문자열에서 부동 소수점 연산 또는 부동 소수점 형식 사양을 검색할 수 없으므로 필요한 부동 소수점 루틴이 로드 되지 않습니다. 이 문제를 해결 하려면 부동 소수점 형식 지정을 사용 하 고 부동 소수점 인수를 제공 하거나 프로그램의 다른 위치에서 부동 소수점 할당을 수행 합니다. 이로 인해 부동 소수점 지원이 로드 됩니다.

- 혼합 언어 프로그램에서는 프로그램이 연결 되었을 때 포트란 라이브러리 앞에 C 라이브러리가 지정 되었습니다. 다시 링크 하 고 C 라이브러리를 마지막에 지정 합니다.
