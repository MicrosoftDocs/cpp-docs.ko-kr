# <a name="c-runtime-error-r6002"></a>C 런타임 오류 R6002

로드 되지 않은 부동 소수점 지원

필요한 부동 소수점 라이브러리가 연결 되지 않았습니다.

> [!NOTE]
>  앱을 실행 하는 동안이 오류 메시지를 발생 하는 경우 앱 종료가 내부 문제가 있기 때문입니다. 이 오류에 대 한 몇 가지 가능한 이유는 앱의 코드에서 결함으로 또는 특정 컴퓨터 프로세서에 대 한 빌드하지 않은 앱을 실행 하 여 발생 한 경우가 많습니다.
>
>  이 오류를 해결하려면 다음 단계를 시도할 수 있습니다.
>
>  -   사용 하 여 합니다 **앱 및 기능** 또는 **프로그램 및 기능** 페이지를 **제어판** 를 복구 하거나 프로그램을 다시 설치 합니다.
> -   확인할 **Windows 업데이트** 에 **제어판** 소프트웨어 업데이트에 대 한 합니다.
> -   앱의 업데이트 된 버전을 확인 합니다. 문제가 지속 되는 경우 앱 공급 업체에 문의 합니다.

**프로그래머를 위한 정보**

이 오류는 부동 소수점 라이브러리 연결 되지 않았습니다 하는 경우 앱에서 발생할 수 있습니다. 이러한 원인 중 하나를 확인 합니다.

- 에 대 한 형식 문자열을 `printf_s` 또는 `scanf_s` 함수가 부동 소수점 형식 지정을 포함 하 고 모든 부동 소수점 값 또는 변수는 프로그램 들어 있지 않습니다. 이 문제를 해결 하려면, 부동 소수점 형식 지정에 해당 하는 부동 소수점 인수를 사용 하거나 프로그램에서 다른 곳에서 부동 소수점 할당을 수행 합니다. 그러면 로드할 부동 소수점 지원 됩니다.

- 컴파일러는 부동 소수점 지원이 필요한 경우에 로드 하 여 프로그램의 크기를 최소화 합니다. 컴파일러에서에서 감지할 수 없을 부동 소수점 연산 또는 부동 소수점 형식 사양 형식 문자열 하므로 필요한 부동 소수점 루틴을 로드 하지 않습니다. 이 문제를 해결 하려면 부동 소수점 형식 사양을 사용 하는 부동 소수점 인수를 지정 또는 프로그램에서 다른 곳에서 부동 소수점 할당을 수행 합니다. 그러면 로드할 부동 소수점 지원 됩니다.

- 혼합 언어 프로그램에서 프로그램이 링크 되는 경우 C 라이브러리 FORTRAN 라이브러리 전에 지정 되었습니다. 다시 연결 하 고 C 라이브러리를 마지막으로 지정 합니다.
