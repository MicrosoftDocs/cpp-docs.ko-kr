# <a name="compiler-warning-c5045"></a>컴파일러 경고 C5045

> 컴파일러는 메모리 /Qspectre 스위치 하는 경우 지정 된 로드에 대 한 스펙터 완화를 삽입

## <a name="remarks"></a>설명

경고 C5045 하면 코드에서 패턴을 LFENCE 같은 스펙터 완화 방법으로 발생 하는 것이 표시를 삽입할 때 합니다 [/Qspectre](../../build/reference/qspectre.md) 컴파일러 옵션을 지정 합니다. 이렇게 하면 보안 문제에 관련 된 코드 파일을 식별할 수 있습니다. 이 경고는 정보로: 완화를 사용 하 여 다시 컴파일할 때까지 삽입 되지 않고 합니다 **/Qspectre** 전환 합니다. C5045 기능은 무관 합니다 **/Qspectre** 스위치를 모두 동일한 컴파일에서 사용할 수 있도록 합니다.

이 경고는 Visual Studio 2017 버전 15.7의에서 새로운와 기본적으로 해제 되어 있습니다. 사용 하 여 [/wall](../../build/reference/compiler-option-warning-level.md) 기본적으로 해제 되어 있는 모든 경고를 사용 하도록 설정 하거나 __/w__*n*__5038__ C5045 수준으로 사용할 수 있도록 *n* 경고 합니다. IDE에서 기본 경고 수준이 **/w3** 있으며 프로젝트에서이 경고를 설정할 수 있습니다 **속성 페이지** 대화 합니다. 오픈 **구성 속성** > **C/c + +** > **명령줄** 고는 **추가 옵션** 상자에서 추가 */w35045*, 선택한 **확인**합니다. 자세한 내용은 [기본적으로 해제 되어 있는 컴파일러 경고](../../preprocessor/compiler-warnings-that-are-off-by-default.md)합니다. 컴파일러 버전별 경고를 사용 하지 않도록 설정 하는 방법에 대 한 자세한 내용은 [컴파일러 버전별 컴파일러 경고](compiler-warnings-by-compiler-version.md)합니다.

## <a name="example"></a>예제

다음 예제에서는 경고를 사용 하 여 Visual Studio 2017 버전 15.7에서 컴파일하면 C5045 합니다 **/wall** 또는 **/w35045** 하 고 **/w3** 옵션:

```cpp
// C5045.cpp
// Compile with: cl /EHsc /W3 /w35045 C5045.cpp

int G, G1, G2;

__forceinline
int * bar(int **p, int i)
{
    return p[i];
}

__forceinline
void bar1(int ** p, int i)
{
    if (i < G1) {
        auto x = p[i]; // C5045: mitigation here
        G = *x;
    }
}

__forceinline
void foo(int * p)
{
    G = *p;
}

void baz(int ** p, int i)
{
    if (i < G1) {
        foo(bar(p, i + G2));
    }
    bar1(p, i);
}

int main() { }
```

경고를 사용 하는 경우 컴파일러 출력이 같습니다.

```Output

```

경고 메시지는 완화 삽입 되었을 16 줄에 표시 합니다. 완화 인덱스의 줄 15 줄 17의 메모리 로드 피드 때문에 필요 하다는 사항도 소개 합니다. 추론에서 수행 됩니다 표시줄과 bar1 완화는 16 줄에 배치 될 때 유효 하지만 합니다.
