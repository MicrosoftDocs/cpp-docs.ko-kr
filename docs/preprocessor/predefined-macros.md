# <a name="predefined-macros"></a>미리 정의된 매크로

Microsoft C/C++ 컴파일러 (MSVC)는 언어 (C 또는 C++), 컴파일 대상 및 선택 된 컴파일러 옵션에 따라 특정 전처리기 매크로를 기본적으로 설정 합니다.

MSVC는 ANSI/ISO C99 표준 및 ISO c + + 14 및 c + + 17 표준에 필요한 미리 정의 된 전처리기 매크로를 지원 합니다. 구현에서는 몇 가지 Microsoft 전용 전처리기 매크로도 지원 합니다. 일부 매크로는 특정 빌드 환경 또는 컴파일러 옵션에 대해서만 정의 됩니다. 명시 된 경우를 제외 하 고 매크로는 변환 단위에서 **/d** 컴파일러 옵션 인수로 지정 된 것 처럼 정의 됩니다. 정의 된 경우 컴파일 전에 전처리기에 의해 매크로가 지정 된 값으로 확장 됩니다. 미리 정의 된 매크로는 인수를 사용 하지 않으며 다시 정의할 수 없습니다.

## <a name="standard-predefined-identifier"></a>미리 정의 된 표준 식별자

컴파일러는 ISO C99 및 ISO c + + 11에 지정 된이 미리 정의 된 식별자를 지원 합니다.

- 바깥쪽 함수의 정규화 되지 않은 이름 및 되지 않은 이름을 **char**의 함수-지역 **정적 const** 배열로 `__func__` 합니다.

    ```cpp
    void example(){
        printf("%s\n", __func__);
    } // prints "example"
    ```

## <a name="standard-predefined-macros"></a>미리 정의 된 표준 매크로

컴파일러는 ISO C99 및 ISO c + + 17 표준에 지정 된 이러한 미리 정의 된 매크로를 지원 합니다.

- 변환 단위가로 C++컴파일될 때 정수 리터럴 값으로 정의 `__cplusplus`. 그렇지 않으면 정의 되지 않습니다.

- 현재 소스 파일의 컴파일 날짜를 `__DATE__` 합니다. 날짜는 *Mmm dd yyyy*형식의 상수 길이 문자열 리터럴입니다. 월 이름 *Mmm* 은 CRT (C 런타임 라이브러리) [asctime](../c-runtime-library/reference/asctime-wasctime.md) 함수에서 생성 된 약식 월 이름과 동일 합니다. 값이 10 보다 작은 경우 첫 번째 날짜 *dd* 문자는 공백입니다. 이 매크로는 항상 정의 됩니다.

- 현재 소스 파일의 이름을 `__FILE__` 합니다. `__FILE__`은 문자열 리터럴로 확장 됩니다. 파일의 전체 경로가 표시 되도록 하려면 [/fc (진단 소스 코드 파일의 전체 경로)](../build/reference/fc-full-path-of-source-code-file-in-diagnostics.md)를 사용 합니다. 이 매크로는 항상 정의 됩니다.

- `__LINE__` 현재 소스 파일의 정수 줄 번호로 정의 됩니다. `__LINE__` 매크로의 값은 `#line` 지시어를 사용 하 여 변경할 수 있습니다. 이 매크로는 항상 정의 됩니다.

- C로 컴파일된 경우에만 1로 정의 되 고 [/za](../build/reference/za-ze-disable-language-extensions.md) 컴파일러 옵션이 지정 된 경우에만 `__STDC__` 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- 구현은 전체 필수 표준 라이브러리를 지 원하는 *호스팅된 구현인*경우 1로 정의 `__STDC_HOSTED__`. 그렇지 않으면 0으로 정의 됩니다.

- 프로그램에서 실행 스레드를 두 개 이상 포함할 수 있고로 C++컴파일할 수 있는 경우에만 1로 정의 `__STDCPP_THREADS__`. 그렇지 않으면 정의 되지 않습니다.

- 전처리 된 변환 단위를 변환 하는 시간을 `__TIME__` 합니다. 시간은 *hh: mm: ss*형식의 문자열 리터럴인 CRT [asctime](../c-runtime-library/reference/asctime-wasctime.md) 함수에서 반환 된 시간과 동일 합니다. 이 매크로는 항상 정의 됩니다.

## <a name="microsoft-specific-predefined-macros"></a>Microsoft 전용 미리 정의 된 매크로

MSVC는 미리 정의 된 추가 매크로를 지원 합니다.

- [/favor: ATOM](../build/reference/favor-optimize-for-architecture-specifics.md) 컴파일러 옵션이 설정 되 고 컴파일러 대상이 x86 또는 x 64 인 경우 `__ATOM__` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- [/arch: AVX](../build/reference/arch-x86.md), [/arch: AVX2](../build/reference/arch-x86.md) 또는 [/arch: AVX512](../build/reference/arch-x86.md) 컴파일러 옵션이 설정 되 고 컴파일러 대상이 x86 또는 x 64 인 경우 `__AVX__` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- [/arch: AVX2](../build/reference/arch-x86.md) 또는 [/arch: AVX512](../build/reference/arch-x86.md) 컴파일러 옵션을 설정 하 고 컴파일러 대상이 x86 또는 x 64 인 경우 `__AVX2__` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- [/arch: AVX512](../build/reference/arch-x86.md) 컴파일러 옵션이 설정 되 고 컴파일러 대상이 x86 또는 x 64 인 경우 `__AVX512BW__` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- [/arch: AVX512](../build/reference/arch-x86.md) 컴파일러 옵션이 설정 되 고 컴파일러 대상이 x86 또는 x 64 인 경우 `__AVX512CD__` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- [/arch: AVX512](../build/reference/arch-x86.md) 컴파일러 옵션이 설정 되 고 컴파일러 대상이 x86 또는 x 64 인 경우 `__AVX512DQ__` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- [/arch: AVX512](../build/reference/arch-x86.md) 컴파일러 옵션이 설정 되 고 컴파일러 대상이 x86 또는 x 64 인 경우 `__AVX512F__` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- [/arch: AVX512](../build/reference/arch-x86.md) 컴파일러 옵션이 설정 되 고 컴파일러 대상이 x86 또는 x 64 인 경우 `__AVX512VL__` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- 기본 **문자** 형식이 unsigned 인 경우 `_CHAR_UNSIGNED` 1로 정의 됩니다. 이 값은 [/j (기본 문자 형식이 Unsigned 인 경우)](../build/reference/j-default-char-type-is-unsigned.md) 컴파일러 옵션이 설정 된 경우에 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- 응용 프로그램을 컴파일하는 데 사용 되는 CLR (공용 언어 런타임) 버전을 나타내는 정수 리터럴로 정의 `__CLR_VER`. 값은 `Mmmbbbbb`형식으로 인코딩됩니다. 여기서 `M`는 런타임의 주 버전이 고, `mm`는 런타임의 부 버전 이며 `bbbbb`는 빌드 번호입니다. [/clr](../build/reference/clr-common-language-runtime-compilation.md) 컴파일러 옵션이 설정 된 경우 `__CLR_VER` 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

    ```cpp
    // clr_ver.cpp
    // compile with: /clr
    using namespace System;
    int main() {
       Console::WriteLine(__CLR_VER);
    }
    ```

- [/wguard: cf (제어 흐름 보호 사용)](../build/reference/guard-enable-control-flow-guard.md) 컴파일러 옵션이 설정 된 경우 `_CONTROL_FLOW_GUARD` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- `__COUNTER__`는 0부터 시작 하는 정수 리터럴로 확장 됩니다. 원본 파일 또는 소스 파일의 포함 된 헤더에서 사용 될 때마다 값이 1 씩 증가 합니다. `__COUNTER__`는 미리 컴파일된 헤더를 사용 하는 경우 해당 상태를 기억 합니다. 이 매크로는 항상 정의 됩니다.

  이 예제에서는 `__COUNTER__`를 사용 하 여 고유 식별자를 동일한 형식의 세 가지 다른 개체에 할당 합니다. `exampleClass` 생성자는 정수를 매개 변수로 사용 합니다. `main`에서 응용 프로그램은 `__COUNTER__`을 고유 식별자 매개 변수로 사용 하 여 `exampleClass`형식의 세 개체를 선언 합니다.

    ```cpp
    // macro__COUNTER__.cpp
    // Demonstration of __COUNTER__, assigns unique identifiers to
    // different objects of the same type.
    // Compile by using: cl /EHsc /W4 macro__COUNTER__.cpp
    #include <stdio.h>

    class exampleClass {
        int m_nID;
    public:
        // initialize object with a read-only unique ID
        exampleClass(int nID) : m_nID(nID) {}
        int GetID(void) { return m_nID; }
    };

    int main()
    {
        // __COUNTER__ is initially defined as 0
        exampleClass e1(__COUNTER__);

        // On the second reference, __COUNTER__ is now defined as 1
        exampleClass e2(__COUNTER__);

        // __COUNTER__ is now defined as 2
        exampleClass e3(__COUNTER__);

        printf("e1 ID: %i\n", e1.GetID());
        printf("e2 ID: %i\n", e2.GetID());
        printf("e3 ID: %i\n", e3.GetID());

        // Output
        // ------------------------------
        // e1 ID: 0
        // e2 ID: 1
        // e3 ID: 2

        return 0;
    }
    ```

- 로 C++ 컴파일될 때 `__cplusplus_cli` 정수 리터럴 값 200406으로 정의 되 고 [/clr](../build/reference/clr-common-language-runtime-compilation.md) 컴파일러 옵션이 설정 됩니다. 그렇지 않으면 정의 되지 않습니다. 정의 된 경우 `__cplusplus_cli` 변환 단위 전체에 적용 됩니다.

    ```cpp
    // cplusplus_cli.cpp
    // compile by using /clr
    #include "stdio.h"
    int main() {
       #ifdef __cplusplus_cli
          printf("%d\n", __cplusplus_cli);
       #else
          printf("not defined\n");
       #endif
    }
    ```

- 로 C++ 컴파일될 때 `__cplusplus_winrt` 정수 리터럴 값 201009으로 정의 되 고 [/zw (Windows 런타임 컴파일)](../build/reference/zw-windows-runtime-compilation.md) 컴파일러 옵션이 설정 됩니다. 그렇지 않으면 정의 되지 않습니다.

- [/gr (런타임 형식 정보 사용)](../build/reference/gr-enable-run-time-type-information.md) 컴파일러 옵션이 설정 된 경우 `_CPPRTTI` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- 하나 이상의 [/gx (예외 처리 사용)](../build/reference/gx-enable-exception-handling.md), [/Clr (공용 언어 런타임 컴파일)](../build/reference/clr-common-language-runtime-compilation.md)또는 [/Eh (예외 처리 모델)](../build/reference/eh-exception-handling-model.md) 컴파일러 옵션이 설정 된 경우 `_CPPUNWIND` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- [/Ldd](../build/reference/md-mt-ld-use-run-time-library.md), [/mdd](../build/reference/md-mt-ld-use-run-time-library.md)또는 [/MTd](../build/reference/md-mt-ld-use-run-time-library.md) 컴파일러 옵션이 설정 된 경우 1로 정의 됩니다. `_DEBUG` 그렇지 않으면 정의 되지 않습니다.

- [/md](../build/reference/md-mt-ld-use-run-time-library.md) 또는 [/Mdd](../build/reference/md-mt-ld-use-run-time-library.md) (다중 스레드 DLL) 컴파일러 옵션이 설정 된 경우 `_DLL` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- 바깥쪽 함수의 [데코레이팅된 이름을](../build/reference/decorated-names.md) 포함 하는 문자열 리터럴로 정의 `__FUNCDNAME__`. 매크로는 함수 내 에서만 정의 됩니다. [/Ep](../build/reference/ep-preprocess-to-stdout-without-hash-line-directives.md) 또는 [/p](../build/reference/p-preprocess-to-a-file.md) 컴파일러 옵션을 사용 하는 경우 `__FUNCDNAME__` 매크로가 확장 되지 않습니다.

   이 예에서는 `__FUNCDNAME__`, `__FUNCSIG__`및 `__FUNCTION__` 매크로를 사용 하 여 함수 정보를 표시 합니다.

- 기본 [/zc: wchar_t](../build/reference/zc-wchar-t-wchar-t-is-native-type.md) 컴파일러 옵션이 설정 된 경우 `_WCHAR_T_DEFINED` 1로 정의 됩니다. `_WCHAR_T_DEFINED` 매크로가 정의 되어 있지만 `/Zc:wchar_t-` 컴파일러 옵션을 설정 하 고 **wchar_t** 프로젝트에 포함 된 시스템 헤더 파일에 정의 되어 있는 경우에는 값이 없습니다. 그렇지 않으면 정의 되지 않습니다.

- 컴파일 대상이 32 비트 ARM, 64 비트 ARM, x86 또는 x 64 인 경우 `_WIN32` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- 컴파일 대상이 64 비트 ARM 또는 x 64 인 경우 `_WIN64` 1로 정의 됩니다. 그렇지 않으면 정의 되지 않습니다.

- 로 C++ 컴파일될 때 `_WINRT_DLL` 1로 정의 되 고, [/zw (Windows 런타임 컴파일)](../build/reference/zw-windows-runtime-compilation.md) 및 [/LD 또는/ldd](../build/reference/md-mt-ld-use-run-time-library.md) 컴파일러 옵션이 설정 됩니다. 그렇지 않으면 정의 되지 않습니다.

ATL 또는 MFC 라이브러리 버전을 식별 하는 전처리기 매크로는 컴파일러에 의해 미리 정의 되어 있지 않습니다. ATL 및 MFC 라이브러리 헤더는 내부적으로 이러한 버전 매크로를 정의 합니다. 필수 헤더를 포함 하기 전에 만들어진 전처리기 지시문에서 정의 되지 않습니다.

- \<`_ATL_VER`에 정의 된 ATL 버전 번호를 인코딩하는 정수 리터럴로 >.

- \<afxver_에 정의 된 `_MFC_VER` MFC 버전 번호를 인코딩하는 정수 리터럴로 >.
