# <a name="c26110"></a>C26110

> 경고 C 26110: 호출자가 \<lock> 함수를 호출 하기 전에 잠금을 유지 하지 못했습니다 \<func> .

잠금이 필요한 경우 함수 자체 또는 해당 호출자가 잠금을 획득해야 하는지 여부를 확인합니다. 경고 C 26110은 `_Requires_lock_held_` 주석이 나 기타 잠금 관련 주석을 위반 하는 경우에 실행 됩니다. 자세한 내용은 [잠금 동작에 주석](annotating-locking-behavior.md) 추가를 참조 하세요.

## <a name="example"></a>예제

다음 예제에서는 `_Requires_lock_held_` 함수의 `LockRequired` 주석에 `LockRequired` 호출자가 `LockRequired`를 호출하기 전에 잠금을 획득하도록 지정되었기 때문에 경고 C26110이 생성되었습니다. 이 주석이 없으면 `LockRequired`가 잠금으로 보호되는 공유 데이터에 액세스하기 전에 잠금을 획득해야 합니다.

```cpp
typedef struct _DATA
{
    CRITICAL_SECTION cs;
    int d;
} DATA;

_Requires_lock_held_(p->cs)

void LockRequired(DATA* p)
{
    p->d = 0;
}

void LockNotHeld(DATA* p)
{
    LockRequired(p); // Warning C26110
}
```
