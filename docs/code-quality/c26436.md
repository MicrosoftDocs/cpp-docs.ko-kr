# <a name="c26436-need_virtual_dtor"></a>C26436 NEED_VIRTUAL_DTOR

"가상 함수가 있는 형식에는 공용 가상 또는 보호된 비가상 소멸자가 필요합니다."

**C++ 핵심 지침**: C.35: 기본 클래스 소멸자는 공용 및 가상 또는 보호 및 비가상이어야 합니다.

클래스가 가상 함수를 정의하면 파생 클래스가 리소스 관리 및 소멸 논리를 포함하여 동작을 변경할 수 있음을 의미하는 다형성이 됩니다. 클라이언트 코드는 기본 클래스에 대한 포인터를 통해 다형성 형식을 호출할 수 있으므로 클라이언트가 다운캐스팅없이 적절한 동작을 명시적으로 선택할 수 있는 방법은 없습니다. 리소스가 일관되게 관리되고 실제 형식의 규칙에 따라 소멸이 발생하는지 확인하려면 공용 가상 소멸기를 정의하는 것이 좋습니다. 형식 계층 구조가 클라이언트 코드가 개체를 직접 삭제하도록 허용하지 않도록 설계된 경우 소멸자는 보호된 비가상으로 정의되어야 합니다.

## <a name="remarks"></a>설명

- 경고는 형식의 첫 번째 가상 함수 정의에 표시됩니다(공개가 아닌 경우 가상 소멸자가 될 수 있음) 형식당 한 번씩 표시됩니다.
  - 정의는 선언과 별도로 배치할 수 있으므로 항상 가상 지정자가 없는 것은 아닙니다. 그러나 경고는 여전히 유효합니다 - 함수의 실제 '가상성'을 확인합니다.
